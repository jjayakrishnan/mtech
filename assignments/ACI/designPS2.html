1. PEAS Analysis

PEAS (Performance measure, Environment, Actuators, Sensors) characterises the submarine as a rational agent operating in a partially-hazardous ocean grid.

PEAS ElementDescription
Performance Measure Minimise total movement cost from start to goal. Secondary objective: avoid Sonar Blackout zones whenever an alternative path exists. Tertiary: minimise the number of grid cells traversed (path length).
Environment A fully observable, deterministic, static, discrete ocean grid containing six terrain types: Open Water (O), Coral Reef (R), Current Corridor (C), Deep Trench (D), Sonar Blackout Zone (B), and Thermal Vent (T). Movement is 8-directional (orthogonal + diagonal).
Actuators Submarine propulsion system — moves the agent one cell per action in any of the eight cardinal/diagonal directions. Speed is reduced automatically in Coral Reef cells and assisted in Current Corridors.
Sensors Sonar and depth sensors that provide the terrain type and cost of any cell in the grid (fully observable). Sonar is degraded inside Blackout Zones, reflected in the avoidance constraint.

2. Problem Modelling

The ocean grid is modelled as a weighted directed graph where each cell is a node and each valid move is a directed edge whose weight equals the entry cost of the destination cell.

State Space

  • State: current cell (row, col)
  • Initial state: Start cell (0, 0)
  • Goal state: Goal cell (5, 5) for Case 1
  • Actions: move to any of up to 8 neighbours
  • Transition: deterministic — new state is (row+Δr, col+Δc)

3. Heuristic Function

Chosen Heuristic: Scaled Chebyshev Distance

h(n) = max( |row(n) − row(goal)|, |col(n) − col(goal)| ) × 0.5

The submarine can move in 8 directions (orthogonal and diagonal). The Chebyshev distance — the maximum of the absolute row and column differences — equals the minimum number of steps required to reach the goal when diagonal movement is allowed, regardless of intermediate obstacles. Multiplying by 0.5 (the lowest possible step cost, achieved on Current Corridor cells) yields an estimate that never exceeds the actual cost.

Admissibility

A heuristic is admissible if h(n) ≤ h*(n) for all nodes n, where h*(n) is the true optimal cost from n to the goal. Since every cell entry costs at least 0.5 and the minimum number of moves is the Chebyshev distance, we have: 

h*(n) ≥ chebyshev(n, goal) × 0.5 = h(n)   ✓ (never overestimates)

Consistency (Monotonicity)

A heuristic is consistent if for every move from n to successor n': h(n) ≤ cost(n, n') + h(n'). Chebyshev distance satisfies the triangle inequality, so scaling by a constant preserves consistency. This guarantees that A* expands each node at most once, making the implementation correct and efficient.

Justification in Context

  • Better than Manhattan distance for 8-directional grids. Manhattan counts orthogonal steps only and would overcount diagonal moves, violating admissibility if the scaling factor were too large.
  • Tighter than constant 0 (Dijkstra's heuristic). The non-zero estimate directs the search towards the goal, dramatically reducing the number of nodes expanded compared to uniform-cost search.
  • The scale factor 0.5 is tight: any factor > 0.5 could overestimate (if a direct path through Current Corridors exists) and lose admissibility. The factor 0.5 is therefore both safe and maximally informative.

4. Cost Function and Its Influence on A* Search

Terraing(n) contributionEffect on A* Search
Open Water (O) +1.0 per step Baseline. A* treats these as neutral transitions.
Coral Reef (R) +2.0 per step Doubles the g-cost increment. A* detours around reefs unless avoiding them adds more than 1 extra step.
Current Corridor (C) +0.5 per step Halves the g-cost increment. A* actively seeks Current Corridors because they lower both g(n) and the f-score, pulling them to the front of the priority queue.
Deep Trench (D) +3.0 per step Highest traversal cost. A* avoids Trenches aggressively; a path bypassing a Trench with up to 2 extra Open Water steps is still cheaper.
Sonar Blackout (B) ∞ (Pass 1) / +1.0 (Pass 2) Two-pass strategy: treated as a wall in Pass 1 (safe routing). Cost reverts to 1.0 in Pass 2 only when no B-free path exists, reflecting the "avoid unless absolutely necessary" constraint.
Thermal Vent (T) ∞ (always) Never entered. These cells are pruned from the neighbour list before any cost check, acting as hard walls regardless of f-score.

The cost function directly determines the g(n) component of A*'s evaluation f(n) = g(n) + h(n). Low-cost cells (Current Corridors) raise f(n) slowly, keeping nodes in the open list longer and prioritising paths through them. High-cost cells (Trenches, Reefs) cause rapid f(n) growth, pushing those paths towards the back of the heap. Impassable cells are filtered entirely. This asymmetry naturally steers the search along energetically optimal routes without any domain-specific hard-coding.

5. Why A* is Suitable for This Problem

The submarine navigation problem is a single-agent, fully observable, deterministic, static, discrete environment with varying edge weights. A* is the canonical informed search algorithm for exactly this class of problems. The table below compares candidate algorithms:

AlgorithmOptimal?Complete?Terrain Costs?Verdict
BFS Only for unit cost Yes No Ignores variable costs — incorrect here
DFS No No (cycles) No Neither optimal nor complete
UCS (Dijkstra) Yes Yes Yes Optimal but explores blindly in all directions
Greedy Best-First No Yes Yes Fast but may miss cheaper routes through detours
A* Yes Yes Yes Optimal and efficient — best fit

A* combines the cost-awareness of UCS (g(n)) with the directional focus of Greedy search (h(n)). With an admissible and consistent heuristic, A* is optimally efficient: no other optimal algorithm expands fewer nodes (Russell & Norvig, AIMA 4th ed., §3.5.2). This is crucial when the grid contains expensive detour terrain (Trenches, Reefs) and we must find the globally cheapest path, not just any path.

Key Properties in This Domain

  • Optimality: Because the heuristic is admissible, A* is guaranteed to return the minimum-cost path from start to goal.
  • Completeness: The grid is finite and all edge costs are positive. A* will always find a solution if one exists and correctly report failure otherwise.
  • Efficiency: The Chebyshev heuristic prunes large portions of the search space that cannot lead to a better solution. On a 6×6 grid, A* typically expands only a fraction of the 36 cells, converging quickly.
  • Constraint Handling: Terrain constraints (impassable Thermal Vents, Sonar Blackout avoidance) are enforced cleanly at the neighbour-generation step, keeping the core search loop simple and correct.

6. Alternate Problem Modelling Approach

Approach: Multi-Objective Search with Pareto Frontier

Instead of collapsing cost and safety into a single scalar, the problem can be modelled as a bi-objective shortest-path problem with two independent objectives:

  1. Objective 1 — Minimise total movement cost (fuel/time)
  2. Objective 2 — Minimise total number of Sonar Blackout cells entered (safety risk)

A Pareto-optimal frontier of non-dominated paths is maintained. Algorithms such as NAMOA* (multi-objective A*) or label-setting with lexicographic ordering can enumerate solutions where no path is simultaneously cheaper and safer than another.

Performance Implications vs. Single-Objective A*

AspectSingle-Objective A* (chosen)Multi-Objective A*
Time complexity O(bd) guided by heuristic; fast in practice O(b2d) worst case; exponential Pareto set
Memory O(bd) — one best g-value per node O(|Pareto set| × nodes) — multiple labels per node
Output Single optimal path Set of trade-off paths — requires human/policy decision
Implementation Standard priority queue, straightforward Complex dominance checks, vector labels
Suitability Suitable — problem has a clear primary objective Overkill for a 6×6 grid; useful for large uncertain grids

The multi-objective approach is theoretically richer and gives a decision-maker a choice of risk vs. efficiency trade-offs. However, for this problem the assignment specifies a clear priority — avoid Sonar Blackout if possible (a lexicographic preference, not a true trade-off) — which is handled exactly by the two-pass A* strategy at negligible extra cost. Multi-objective search would add substantial complexity without yielding a better final answer.

7. Implementation Notes

Data Structure: Min-Heap Priority Queue

A* uses Python's heapq module (binary min-heap) keyed on f(n) = g(n) + h(n). A monotone counter breaks ties to avoid comparing path lists. Stale heap entries (where a cheaper path to the same node was already found) are detected and skipped using a best_g dictionary — this implements lazy deletion without a decrease-key operation.

Sonar Blackout: Two-Pass Strategy

Pass 1:  treat B cells as impassable (cost = ∞)
         → run A*  →  if path found, return it  (sonar-safe)

Pass 2:  treat B cells as Open Water (cost = 1.0)
         → run A*  →  return this path (sonar unavoidable)
         → flag warning in output

Input/Output Format

The program reads from inputPS2.txt and writes to outputPS2.txt. No values are hard-coded. The input format is: 

GridSize: <rows> <cols>
Source: <row> <col>
Destination: <row> <col>
Grid:
S O C T O D
R O B O T C
...
