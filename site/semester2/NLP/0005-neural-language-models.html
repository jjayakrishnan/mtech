1. From N-grams to Neural LMs

N-gram language models (covered in Lesson 3) estimate \(P(w_n | w_1, \ldots, w_{n-1})\) by counting sequences. Their critical weaknesses:

  • Sparsity: Most n-grams are never observed in training data
  • No generalization: seeing "the cat sat on the" gives zero help predicting "the dog sat on the" unless that exact sequence was also observed
  • Exponential memory: storing all n-gram counts is prohibitive for large n
Analogy

An N-gram model is like a student who memorizes every sentence in a textbook. A neural LM is like a student who understands the grammar and vocabulary — they can produce and evaluate sentences they have never seen before.

Neural LMs solve this by:

  1. Representing words as dense vectors (embeddings) — similar words share similar representations
  2. Using continuous functions (neural networks) that generalize smoothly to unseen combinations
  3. Sharing parameters — one weight matrix applies to all positions, enabling generalization

2. Feedforward Neural LM (Bengio et al., 2003)

The pioneering neural language model. Architecture:

w_{t-3}, w_{t-2}, w_{t-1}
→
Lookup: C(w)
→
Concatenate embeddings
→
Hidden (tanh)
→
Softmax (|V|)

Step by step:

  1. Embedding lookup: Map each of the previous \(n-1\) words to a \(d\)-dimensional vector using shared matrix \(\mathbf{C} \in \mathbb{R}^{|V| \times d}\)
  2. Concatenation: Concatenate the \(n-1\) vectors to form input \(\mathbf{x} \in \mathbb{R}^{(n-1) \cdot d}\)
  3. Hidden layer: \(\mathbf{h} = \tanh(\mathbf{W}\mathbf{x} + \mathbf{b})\) where \(\mathbf{W} \in \mathbb{R}^{H \times (n-1)d}\)
  4. Output layer: \(\mathbf{y} = \text{softmax}(\mathbf{U}\mathbf{h} + \mathbf{d})\) where \(\mathbf{U} \in \mathbb{R}^{|V| \times H}\)
Concrete Example

If \(|V| = 5{,}000\), \(d = 100\), context window \(n-1 = 3\):
• Embedding matrix \(\mathbf{C}\): \(5{,}000 \times 100 = 500{,}000\) params
• Input to hidden: \(3 \times 100 = 300\) dimensional
• Hidden size \(H = 200\): weight matrix = \(200 \times 300 = 60{,}000\) params
• Output layer: \(5{,}000 \times 200 = 1{,}000{,}000\) params

Exam Tip

The key innovation: the embedding matrix \(\mathbf{C}\) is shared across all positions and learned jointly with the rest of the network. This is what gives generalization — words with similar contexts get similar embeddings automatically.

3. Activation Functions

Non-linear functions applied element-wise after the linear transformation:

FunctionFormulaRangeKey Property
Sigmoid\(\sigma(z) = \frac{1}{1+e^{-z}}\)(0, 1)Squashes to probability-like output
Tanh\(\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}\)(-1, 1)Zero-centered; preferred in hidden layers
ReLU\(\text{ReLU}(z) = \max(0, z)\)[0, +inf)No vanishing gradient; fast to compute
Important values to memorize

\(\sigma(0) = 0.5\),   \(\tanh(0) = 0\),   \(\text{ReLU}(0) = 0\)
\(\sigma(z) = 1 - \sigma(-z)\)   (symmetry property of sigmoid)

Worked example: Sigmoid

Compute \(\sigma(2.0)\):

\(\sigma(2.0) = \frac{1}{1 + e^{-2.0}} = \frac{1}{1 + 0.135} = \frac{1}{1.135} \approx 0.881\)

Why we need non-linearity

Without activation functions, stacking linear layers just produces another linear function: \(\mathbf{W}_2(\mathbf{W}_1 \mathbf{x}) = (\mathbf{W}_2\mathbf{W}_1)\mathbf{x}\). Non-linearity is what gives neural nets their expressive power.

4. Softmax Output Layer

Converts a vector of raw scores (logits) into a valid probability distribution:

\[P(w_i \mid \text{context}) = \frac{e^{z_i}}{\sum_{j=1}^{|V|} e^{z_j}}\]

Properties: all outputs in (0,1) and they sum to 1.

Worked Example

Logits for 3 words: \(\mathbf{z} = [2.0, \; 1.0, \; 0.1]\)

  • \(e^{2.0} = 7.389\)
  • \(e^{1.0} = 2.718\)
  • \(e^{0.1} = 1.105\)
  • Sum = \(7.389 + 2.718 + 1.105 = 11.212\)

Probabilities:

  • \(P(w_A) = 7.389 / 11.212 \approx \mathbf{0.659}\)
  • \(P(w_B) = 2.718 / 11.212 \approx 0.242\)
  • \(P(w_C) = 1.105 / 11.212 \approx 0.099\)
Exam Tip

The word with the highest logit gets the highest probability, but softmax never assigns exactly 0 or 1. The exponential amplifies differences: a logit difference of 1.0 becomes roughly a 2.7x probability ratio.

5. Cross-Entropy Loss

The training objective for next-word prediction:

\[\mathcal{L} = -\sum_{i=1}^{|V|} y_i \log(\hat{y}_i)\]

Since \(y\) is one-hot (only the correct word has \(y_i = 1\)), this simplifies to:

\[\mathcal{L} = -\log P(\text{correct word} \mid \text{context}) = -\log \hat{y}_{\text{correct}}\]
Plain English

Cross-entropy loss = "how surprised are we by the correct answer?" If the model assigns probability 0.9 to the correct word, loss = \(-\log(0.9) = 0.105\) (low surprise). If it assigns 0.01, loss = \(-\log(0.01) = 4.605\) (very surprised).

Connection to Perplexity

Perplexity = \(2^{\text{avg cross-entropy}}\) (or \(e^{\text{avg CE}}\) if using natural log). Lower perplexity = better model.

Key relationship

If average cross-entropy loss over a test corpus is \(H\), then perplexity = \(e^H\). A perplexity of 50 means the model is "as confused as if it were choosing uniformly among 50 words" at each step.

6. Parameter Counting

A common exam question: "How many parameters does this network have?"

Recipe for the Bengio Neural LM:

ComponentShapeParameters
Embedding matrix C\(|V| \times d\)\(|V| \cdot d\)
Hidden weights W\(H \times (n-1) \cdot d\)\(H \cdot (n-1) \cdot d\)
Hidden bias b\(H \times 1\)\(H\)
Output weights U\(|V| \times H\)\(|V| \cdot H\)
Output bias d\(|V| \times 1\)\(|V|\)

Total = \(|V| \cdot d + H(n-1)d + H + |V| \cdot H + |V|\)

Worked Example

Given: \(|V| = 10{,}000\), \(d = 50\), context \(n-1 = 4\), hidden \(H = 100\)

  • C: \(10{,}000 \times 50 = 500{,}000\)
  • W: \(100 \times 200 = 20{,}000\)
  • b: \(100\)
  • U: \(10{,}000 \times 100 = 1{,}000{,}000\)
  • d: \(10{,}000\)
  • Total: 1,530,100 parameters
Exam Tip

The output layer \(\mathbf{U}\) dominates — it is \(|V| \times H\). This is why large vocabularies make neural LMs expensive. Solutions: hierarchical softmax, negative sampling, subword tokenization.

7. Introduction to LLMs

Large Language Models scale the neural LM idea to billions of parameters and massive datasets:

Autoregressive Models (GPT family)

  • Predict the next token left-to-right: \(P(w_t | w_1, \ldots, w_{t-1})\)
  • Use the Transformer architecture (self-attention, not covered in depth for mid-sem)
  • Good at: text generation, completion, zero-shot reasoning
  • Training: maximize likelihood of next token over huge corpora

Masked Language Models (BERT family)

  • Randomly mask 15% of tokens; predict them using bidirectional context
  • Objective: \(P(w_{\text{mask}} | w_1, \ldots, w_{i-1}, \text{[MASK]}, w_{i+1}, \ldots, w_n)\)
  • Good at: classification, NER, question answering (understanding tasks)
  • Cannot generate text naturally (not autoregressive)
Key Difference

GPT = left-to-right (causal), good at generation.
BERT = bidirectional (masked), good at understanding.
Both use the same underlying prediction framework as Bengio's LM, just scaled massively.

AspectGPT (Autoregressive)BERT (Masked LM)
DirectionLeft-to-right onlyBidirectional
Training taskPredict next tokenPredict masked token
GenerationNatural (one token at a time)Not designed for it
UnderstandingGood but unidirectionalExcellent (sees both sides)
Use caseChatbots, writing, codeClassification, NER, QA

8. Prompt Engineering Basics

Using LLMs effectively without fine-tuning — just by carefully crafting the input:

Zero-shot

Ask the model directly with no examples:

Example

"Classify the sentiment of this review as positive or negative: 'The movie was fantastic!'"

Few-shot

Provide a few input-output examples before the actual query:

Example

"great movie → positive
terrible food → negative
amazing service → positive
boring lecture → ???"

Chain-of-Thought (CoT)

Ask the model to show its reasoning step by step:

Example

"Q: Roger has 5 tennis balls. He buys 2 more cans of 3. How many does he have now?
A: Let's think step by step. Roger started with 5 balls. He bought 2 cans of 3 = 6. Total = 5 + 6 = 11."

Exam Tip

Know the difference: zero-shot (no examples), few-shot (some examples), CoT (step-by-step reasoning). If asked "how to improve LLM performance without training," the answer is prompt engineering.

Formula Cards — Memorize for Exam

Card 1: Softmax

\[P(w_i) = \frac{e^{z_i}}{\sum_{j=1}^{|V|} e^{z_j}}\]
Converts logits to probabilities. All positive, sums to 1. Exponential amplifies differences.

Card 2: Cross-Entropy Loss

\[\mathcal{L} = -\sum_{i=1}^{|V|} y_i \log(\hat{y}_i) = -\log P(\text{correct word})\]
For one-hot targets, simplifies to negative log probability of the true word. Lower = better.

Card 3: Sigmoid

\[\sigma(z) = \frac{1}{1+e^{-z}}, \quad \sigma'(z) = \sigma(z)(1-\sigma(z))\]
Maps any real number to (0,1). Derivative is elegant: output times (1 - output).

Card 4: Neural LM Parameters

\[\text{Total} = |V| \cdot d + H \cdot (n-1) \cdot d + H + |V| \cdot H + |V|\]
Embed + Hidden_weights + Hidden_bias + Output_weights + Output_bias. Output layer dominates for large V.

Card 5: Perplexity

\[\text{PP}(W) = P(w_1 w_2 \ldots w_N)^{-1/N} = e^{H(W)}\]
H = average cross-entropy. Perplexity K means model is "as confused as choosing from K words uniformly."

Try It Yourself

Exercise 1: Softmax Computation

Given logits \(\mathbf{z} = [3.0, \; 1.0, \; 0.5, \; -1.0]\) for a 4-word vocabulary {apple, banana, cherry, date}, compute the softmax probabilities.

Solution:

\(e^{3.0} = 20.086, \;\; e^{1.0} = 2.718, \;\; e^{0.5} = 1.649, \;\; e^{-1.0} = 0.368\)

Sum = \(20.086 + 2.718 + 1.649 + 0.368 = 24.821\)

\(P(\text{apple}) = 20.086 / 24.821 = \mathbf{0.809}\)

\(P(\text{banana}) = 2.718 / 24.821 = 0.110\)

\(P(\text{cherry}) = 1.649 / 24.821 = 0.066\)

\(P(\text{date}) = 0.368 / 24.821 = 0.015\)

Check: \(0.809 + 0.110 + 0.066 + 0.015 = 1.000\)

Exercise 2: Cross-Entropy Loss

The model assigns these probabilities: P(the)=0.4, P(a)=0.3, P(cat)=0.2, P(dog)=0.1. The correct next word is "cat". What is the cross-entropy loss?

Solution:

Loss = \(-\log P(\text{cat}) = -\log(0.2) = -(-1.609) = \mathbf{1.609}\)

If the model had assigned P(cat) = 0.8: loss = \(-\log(0.8) = 0.223\) (much lower = better).

If P(cat) = 0.01: loss = \(-\log(0.01) = 4.605\) (very high = terrible prediction).

Exercise 3: Parameter Counting

A neural LM has: vocabulary |V| = 8,000, embedding dimension d = 64, context window n-1 = 5, hidden layer size H = 128. Count all parameters.

Solution:

Embedding C: \(8{,}000 \times 64 = 512{,}000\)

Hidden W: \(128 \times (5 \times 64) = 128 \times 320 = 40{,}960\)

Hidden bias b: \(128\)

Output U: \(8{,}000 \times 128 = 1{,}024{,}000\)

Output bias d: \(8{,}000\)

Total: \(512{,}000 + 40{,}960 + 128 + 1{,}024{,}000 + 8{,}000 = \mathbf{1{,}585{,}088}\) parameters

Notice: the output layer alone is 65% of total parameters.

Exercise 4: Sigmoid Values

Without a calculator, determine: (a) \(\sigma(0)\), (b) \(\sigma(\infty)\), (c) \(\sigma(-\infty)\), (d) If \(\sigma(x) = 0.73\), what is \(\sigma(-x)\)?

Solution:

(a) \(\sigma(0) = \frac{1}{1+e^0} = \frac{1}{2} = \mathbf{0.5}\)

(b) \(\sigma(\infty) = \frac{1}{1+0} = \mathbf{1.0}\)

(c) \(\sigma(-\infty) = \frac{1}{1+\infty} = \mathbf{0.0}\)

(d) Using symmetry \(\sigma(-x) = 1 - \sigma(x) = 1 - 0.73 = \mathbf{0.27}\)

Sources: Bengio et al. (2003) "A Neural Probabilistic Language Model"; Jurafsky & Martin, Speech and Language Processing, Ch. 7; Vaswani et al. (2017) "Attention Is All You Need"; Brown et al. (2020) GPT-3.
