1. What is Dynamic Programming?

Dynamic Programming (DP) refers to a collection of algorithms that compute optimal policies given a perfect model of the environment as an MDP. The key requirement: you must know \(p(s', r \mid s, a)\) for all states, actions, and transitions.

Definition

DP = Planning, not learning. The agent has the complete transition model and reward function. It computes the value function (and hence the optimal policy) by sweeping through states using Bellman equations.

Plain English

Think of DP like having the complete map of a maze, including every dead end. You can plan the optimal route by working backwards from the goal — you never need to actually walk through the maze to figure things out. In contrast, Monte Carlo and TD methods are like exploring the maze without a map.

Why "Dynamic Programming"?

Richard Bellman coined the term. "Dynamic" refers to sequential, multi-stage decision problems. "Programming" means optimisation (like linear programming), not coding. The core idea: break a complex problem into simpler subproblems, solve each once, store and reuse.

Exam Tip

If a question says "given the model" or provides a full transition table, it is asking for a DP solution. If it says "episodes" or "experience," that is model-free (MC/TD).

2. Policy Evaluation (Iterative)

Given a fixed policy \(\pi\), compute its value function \(V^\pi(s)\) for all states. This is also called the prediction problem.

Core Update Rule

Iterative policy evaluation turns the Bellman expectation equation into an update rule:

Policy Evaluation Update

\[V_{k+1}(s) = \sum_a \pi(a|s) \sum_{s',r} p(s',r|s,a)\bigl[r + \gamma V_k(s')\bigr]\]
Start with \(V_0(s) = 0\) for all \(s\). Sweep all states. Repeat until \(\max_s |V_{k+1}(s) - V_k(s)| < \theta\).
Plain English

Imagine asking: "If I follow this specific strategy, how much reward will I get from each state?" You start by assuming all states are worth 0. Then you repeatedly update each state's value by looking one step ahead: what does my policy do, where might I end up, and how good are those next states? After many sweeps, the values converge to the true answer.

Algorithm: Iterative Policy Evaluation

  1. Initialise \(V(s) = 0\) for all \(s\)
  2. Repeat until convergence:
    1. \(\Delta \leftarrow 0\)
    2. For each state \(s\):
      • \(v \leftarrow V(s)\)
      • \(V(s) \leftarrow \sum_a \pi(a|s) \sum_{s',r} p(s',r|s,a)[r + \gamma V(s')]\)
      • \(\Delta \leftarrow \max(\Delta, |v - V(s)|)\)
  3. Stop when \(\Delta < \theta\) (a small threshold)
Synchronous vs In-place

Synchronous: use old values \(V_k\) to compute all new values \(V_{k+1}\) simultaneously. In-place: update \(V(s)\) immediately and use it for subsequent states in the same sweep. In-place converges faster in practice but order matters.

3. Policy Improvement

Once you have \(V^\pi\), you can ask: "Is there a better action than what \(\pi\) currently prescribes?" For each state, compute the Q-value for every action and pick the best.

Q-value Computation

\[Q^\pi(s,a) = \sum_{s'} p(s'|s,a)\bigl[r(s,a,s') + \gamma V^\pi(s')\bigr]\]
This tells you: "What is the value of taking action \(a\) in state \(s\), then following \(\pi\) afterwards?"

Greedy Policy (Improvement Step)

\[\pi'(s) = \arg\max_a Q^\pi(s,a) = \arg\max_a \sum_{s'} p(s'|s,a)\bigl[r(s,a,s') + \gamma V^\pi(s')\bigr]\]
The improved policy \(\pi'\) is greedy with respect to \(V^\pi\). The Policy Improvement Theorem guarantees \(V^{\pi'}(s) \geq V^\pi(s)\) for all \(s\).
Plain English

You have evaluated your current strategy. Now for each situation, you check: "Could I do better by deviating from my strategy for just one step, then going back to it?" If yes, switch to that better action. This always improves (or maintains) the policy — never makes it worse.

Exam Tip

Policy improvement stops when \(\pi' = \pi\) (no action changes). At that point, the Bellman optimality equation is satisfied, meaning you have found \(\pi^*\).

4. Policy Iteration

Combine evaluation and improvement into an alternating loop:

Algorithm: Policy Iteration

  1. Start with arbitrary policy \(\pi_0\)
  2. Evaluate: Compute \(V^{\pi_k}\) using iterative policy evaluation (many sweeps until convergence)
  3. Improve: Set \(\pi_{k+1}(s) = \arg\max_a Q^{\pi_k}(s,a)\) for all \(s\)
  4. If \(\pi_{k+1} = \pi_k\), stop (optimal). Otherwise go to step 2.

The sequence of policies is guaranteed to improve monotonically:

\(\pi_0 \xrightarrow{E} V^{\pi_0} \xrightarrow{I} \pi_1 \xrightarrow{E} V^{\pi_1} \xrightarrow{I} \pi_2 \xrightarrow{E} \cdots \xrightarrow{I} \pi^* \xrightarrow{E} V^*\)

Key Property

For finite MDPs, policy iteration converges in a finite number of steps because there are finitely many deterministic policies. In practice, it often converges surprisingly quickly (5-10 iterations).

Drawback

Each iteration requires a full policy evaluation (many sweeps to convergence). This can be expensive for large state spaces. Value Iteration avoids this by doing only one sweep per iteration.

5. Value Iteration

Value Iteration combines the evaluation and improvement steps into a single update. Instead of fully evaluating a policy, it takes one sweep of the Bellman optimality equation:

Value Iteration Update

\[V_{k+1}(s) = \max_a \sum_{s'} p(s'|s,a)\bigl[r(s,a,s') + \gamma V_k(s')\bigr]\]
Notice the \(\max\) — we pick the best action at each step, combining improvement into evaluation.

Algorithm: Value Iteration

  1. Initialise \(V_0(s) = 0\) for all \(s\)
  2. For each sweep \(k = 0, 1, 2, \ldots\):
    • For each state \(s\): \(V_{k+1}(s) = \max_a \sum_{s',r} p(s',r|s,a)[r + \gamma V_k(s')]\)
  3. Stop when \(\max_s |V_{k+1}(s) - V_k(s)| < \theta\)
  4. Extract final policy: \(\pi^*(s) = \arg\max_a \sum_{s'} p(s'|s,a)[r(s,a,s') + \gamma V^*(s')]\)
Plain English

Value Iteration is impatient. Instead of fully evaluating a mediocre policy, it says: "At every state, just assume I will pick the best action." After enough sweeps, the values stabilise and you read off the optimal policy at the end.

Policy Iteration vs Value Iteration

AspectPolicy IterationValue Iteration
Per iterationFull evaluation (many sweeps) + 1 improvement1 sweep (max over actions)
ConvergenceFewer outer iterationsMore iterations but each is cheap
Equation usedBellman Expectation + greedyBellman Optimality directly
When to useSmaller state spacesLarger state spaces

6. Worked Example: Value Iteration (EC-2, Q1c — 4 marks)

Exam Question 4 marks

"For the MDP given below, calculate the Values of states = {Resting, Moderate Activity, High Activity} using synchronous Value Iteration. Actions = {Mode A, Mode B}. Use \(\gamma = 0.1\). Solve for 1 iteration."

MDP Transition Table

From StateActionTo StateProbabilityReward
RestingMode AResting1.0+1
RestingMode BModerate0.6+1
RestingMode BResting0.4+1
ModerateMode AResting0.4+2
ModerateMode AModerate0.6+1
ModerateMode BResting0.1+1
ModerateMode BModerate0.4+1
ModerateMode BHigh0.5+1
HighMode AModerate0.5+1
HighMode AHigh0.5+1
HighMode BHigh1.0+1

Solution: Synchronous Value Iteration (1 Iteration)

Initialise: \(V_0(\text{Resting}) = V_0(\text{Moderate}) = V_0(\text{High}) = 0\)

The Value Iteration update: \(V_1(s) = \max_a \sum_{s'} p(s'|s,a)[r(s,a,s') + \gamma V_0(s')]\)

Since \(V_0 = 0\) everywhere and \(\gamma = 0.1\), the \(\gamma V_0(s')\) terms vanish. We just need the expected immediate reward for each action.

V1(Resting)

Mode A: \(1.0 \times [1 + 0.1 \times 0] = 1.0\)

Mode B: \(0.6 \times [1 + 0.1 \times 0] + 0.4 \times [1 + 0.1 \times 0] = 0.6 + 0.4 = 1.0\)

\[V_1(\text{Resting}) = \max\{1.0,\; 1.0\} = \boxed{1.0}\]

V1(Moderate Activity)

Mode A: \(0.4 \times [2 + 0.1 \times 0] + 0.6 \times [1 + 0.1 \times 0] = 0.4(2) + 0.6(1) = 0.8 + 0.6 = 1.4\)

Mode B: \(0.1 \times [1 + 0] + 0.4 \times [1 + 0] + 0.5 \times [1 + 0] = 0.1 + 0.4 + 0.5 = 1.0\)

\[V_1(\text{Moderate}) = \max\{1.4,\; 1.0\} = \boxed{1.4}\]

V1(High Activity)

Mode A: \(0.5 \times [1 + 0.1 \times 0] + 0.5 \times [1 + 0.1 \times 0] = 0.5 + 0.5 = 1.0\)

Mode B: \(1.0 \times [1 + 0.1 \times 0] = 1.0\)

\[V_1(\text{High}) = \max\{1.0,\; 1.0\} = \boxed{1.0}\]

Exam Tip

For the first iteration with \(V_0 = 0\), the discount term disappears. The value equals the expected immediate reward under the best action. Always check: does one action give higher expected reward? Here Mode A gives 1.4 for Moderate (due to the +2 reward transition), making it the maximising action.

Try it yourself: What would V_2 be?

Now use \(V_1(\text{R})=1.0, V_1(\text{M})=1.4, V_1(\text{H})=1.0\) and compute \(V_2\). The \(\gamma V_1(s')\) terms no longer vanish!

V2(Resting) with Mode A: \(1.0 \times [1 + 0.1 \times 1.0] = 1.1\)

V2(Resting) with Mode B: \(0.6[1 + 0.1 \times 1.4] + 0.4[1 + 0.1 \times 1.0] = 0.6(1.14) + 0.4(1.10) = 0.684 + 0.44 = 1.124\)

So \(V_2(\text{Resting}) = \max\{1.1, 1.124\} = 1.124\) — Mode B becomes slightly better in iteration 2!

7. Worked Example: Policy Evaluation (EC-3, Q2a — 2 marks)

Exam Question 2 marks

"Warehouse robot with states \(\{L, M, H\}\), Actions \(\{Continue, Recharge\}\), \(\gamma=0.9\). Current estimates \(V(L)=1, V(M)=1, V(H)=1\). Policy \(\pi\): \(\pi(Continue|M)=1\). Write the Bellman expectation equation for \(V_\pi(M)\) and calculate its value."

Given Transitions from Medium with Continue:

FromActionToProbabilityReward
MContinueM0.6+1
MContinueL0.4+1

Solution

Step 1: Write the Bellman Expectation Equation

Since \(\pi(Continue|M) = 1\), the policy sums over a single action:

\[V^\pi(M) = \pi(Continue|M) \cdot \bigl[P(M|M,C)(r + \gamma V(M)) + P(L|M,C)(r + \gamma V(L))\bigr]\]

\[V^\pi(M) = 1 \cdot \bigl[0.6(1 + 0.9 \times V(M)) + 0.4(1 + 0.9 \times V(L))\bigr]\]

Step 2: Substitute Values and Compute

Using current estimates \(V(M) = 1\) and \(V(L) = 1\):

\[V^\pi(M) = 0.6(1 + 0.9 \times 1) + 0.4(1 + 0.9 \times 1)\]

\[= 0.6(1.9) + 0.4(1.9)\]

\[= 1.14 + 0.76\]

\[\boxed{V^\pi(M) = 1.9}\]

Key Insight

Notice that when both next-state values are equal (\(V(M) = V(L) = 1\)) and both transition rewards are equal (+1), the probabilities do not matter — the answer simplifies to \(r + \gamma V = 1 + 0.9(1) = 1.9\) regardless of the probability split.

Try it yourself: What if V(L) = 0 instead?

Recalculate \(V^\pi(M)\) if \(V(L) = 0, V(M) = 1\).

\(V^\pi(M) = 0.6(1 + 0.9 \times 1) + 0.4(1 + 0.9 \times 0) = 0.6(1.9) + 0.4(1.0) = 1.14 + 0.40 = 1.54\)

Now the probability split matters because the two next-states have different values.

8. Worked Example: Policy Improvement (EC-3, Q2b — 4 marks)

Exam Question 4 marks

"Policy improvement at state M. Initial policy \(\pi_0\): \(\pi_0(Continue|H)=1.0\), \(\pi_0(Continue|M)=1.0\), \(\pi_0(Recharge|L)=1.0\). Given \(V^{\pi_0}(L)=-10\), \(V^{\pi_0}(M)=5\), \(V^{\pi_0}(H)=8\). \(\gamma = 0.9\)."

Compute \(Q^{\pi_0}(M, Continue)\) and \(Q^{\pi_0}(M, Recharge)\). Should the policy change at state M?

Given Transitions:

FromActionToProbabilityReward
MContinueM0.6+1
MContinueL0.4+1
MRechargeH1.0-2

Solution

Q(M, Continue)

\[Q^{\pi_0}(M, Continue) = \sum_{s'} p(s'|M, C)\bigl[r + \gamma V^{\pi_0}(s')\bigr]\]

\[= 0.6\bigl[1 + 0.9 \times 5\bigr] + 0.4\bigl[1 + 0.9 \times (-10)\bigr]\]

\[= 0.6\bigl[1 + 4.5\bigr] + 0.4\bigl[1 - 9\bigr]\]

\[= 0.6(5.5) + 0.4(-8)\]

\[= 3.3 - 3.2\]

\[\boxed{Q^{\pi_0}(M, Continue) = 0.1}\]

Q(M, Recharge)

\[Q^{\pi_0}(M, Recharge) = \sum_{s'} p(s'|M, R)\bigl[r + \gamma V^{\pi_0}(s')\bigr]\]

\[= 1.0\bigl[-2 + 0.9 \times 8\bigr]\]

\[= -2 + 7.2\]

\[\boxed{Q^{\pi_0}(M, Recharge) = 5.2}\]

Policy Improvement Decision

\[\pi_1(M) = \arg\max_a Q^{\pi_0}(M, a) = \arg\max\{0.1,\; 5.2\}\]

Since \(Q(M, Recharge) = 5.2 > Q(M, Continue) = 0.1\):

\[\boxed{\pi_1(Recharge|M) = 1.0 \quad \text{— Policy CHANGES!}}\]

Intuition

Why does the policy switch to Recharge? Because Continuing risks landing in the Low state (\(V(L) = -10\), terrible!). Even though Recharge costs -2 immediately, it sends you to High (\(V(H) = 8\), excellent!). The agent learns: "Take a small hit now to avoid the catastrophic Low state."

Try it yourself: Would the policy change at state H?

Suppose from H: Continue goes to H with P=0.8, r=+2 and to M with P=0.2, r=+1. Recharge goes to H with P=1.0, r=-2. Using the same values \(V(H)=8, V(M)=5\). Which action wins?

\(Q(H, Continue) = 0.8(2 + 0.9 \times 8) + 0.2(1 + 0.9 \times 5) = 0.8(9.2) + 0.2(5.5) = 7.36 + 1.10 = 8.46\)

\(Q(H, Recharge) = 1.0(-2 + 0.9 \times 8) = 5.2\)

Since \(8.46 > 5.2\), the policy stays: \(\pi_1(Continue|H) = 1.0\). No change at H.

9. Formula Cards — MEMORISE (Closed Book)

Closed Book Exam

These four formulas form the backbone of all DP methods. You must reproduce them from memory. Pay attention to: (1) \(\sum\) vs \(\max\), (2) what you sum over, (3) where \(\pi\) appears.

1. Bellman Expectation — Policy Evaluation

\[V^\pi(s) = \sum_a \pi(a|s) \sum_{s'} p(s'|s,a)\bigl[r + \gamma V^\pi(s')\bigr]\]
Weighted average over actions (using policy probabilities) and transitions. Used to evaluate a given policy.

2. Bellman Optimality — Value Iteration

\[V_{k+1}(s) = \max_a \sum_{s'} p(s'|s,a)\bigl[r + \gamma V_k(s')\bigr]\]
Replace \(\sum_a \pi(a|s)\) with \(\max_a\). The optimal value at \(s\) equals the best one-step-ahead value. No policy needed — greediness is built in.

3. Q-value — Policy Improvement

\[Q^\pi(s,a) = \sum_{s'} p(s'|s,a)\bigl[r(s,a,s') + \gamma V^\pi(s')\bigr]\]
Value of taking action \(a\) once, then following \(\pi\). Computed for each action to find the best one.

4. Greedy Policy — Policy Improvement

\[\pi'(s) = \arg\max_a Q^\pi(s,a)\]
Pick the action with highest Q-value. Guaranteed to be at least as good as the old policy (\(V^{\pi'} \geq V^\pi\) everywhere).

Quick Mnemonics

FormulaRemember
Policy Eval\(\sum \pi \cdot \sum p[\ldots]\) — "sum policy, sum transitions"
Value Iter\(\max \sum p[\ldots]\) — "max over actions, sum transitions"
Q-value\(\sum p[r + \gamma V]\) — "just the inner sum, one specific action"
Improve\(\arg\max Q\) — "pick the best Q"
Self-test: Cover the formulas and write them from memory

On paper, write all four formulas. Check:

  • Policy Eval: Did you include \(\pi(a|s)\) weighting?
  • Value Iter: Did you use \(\max_a\) instead of \(\sum_a \pi(a|s)\)?
  • Q-value: Did you sum over \(s'\) only (action \(a\) is fixed)?
  • Improve: Is it \(\arg\max\) (returns the action) not \(\max\) (returns the value)?

Common mistakes: (1) Forgetting \(\pi(a|s)\) in policy eval. (2) Writing \(\max\) when you mean \(\arg\max\) for the policy. (3) Confusing which sums appear in Q vs V formulas.
