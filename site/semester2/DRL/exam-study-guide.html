2. Topic Reference

Topic 1

Multi-Armed Bandit

  • No state — just actions + rewards
  • UCB: Q(a) + c·√(ln t / N(a))
  • EWMA: Q(a) ← Q(a) + α·[r − Q(a)]
  • ε-greedy: explore with prob ε
  • ε-case: random action chosen
  • α=1 → only last reward matters
Very High Frequency
Topic 2

MDP Formulation

  • Tuple: (S, A, P, R, γ)
  • Bellman: Vπ(s) = Σₐ π(a|s) · Σₛ' P·[R + γV(s')]
  • Value Iteration: V ← max over actions
  • Synchronous VI: update all states at once
  • Episodic vs continuing tasks
  • γ≈0: myopic; γ≈1: far-sighted
Very High Frequency
Topic 3

Monte Carlo Methods

  • First-visit: only first occurrence in episode
  • G = r₁ + γr₂ + γ²r₃ + … (work backwards)
  • V(s) ← V(s) + (1/n)[G − V(s)]
  • Q update → greedy policy improvement
  • Off-policy: importance sampling ratio ρ
  • Weighted IS: ρ·G / Σρ
Very High Frequency
Topic 4

Policy Gradient

  • REINFORCE: θ ← θ + α·G·∇ln π(a|s)
  • Softmax policy: π(a|s) = σ(θᵀφ(s,a))
  • Baseline: reduces variance (subtract b from G)
  • Without baseline: high variance updates
  • A2C: Actor picks actions; Critic estimates V
  • Advantage: A(s,a) = Q(s,a) − V(s)
High Frequency
Topic 5

MAB vs MDP Classification

  • MAB: no state transitions, i.i.d. rewards
  • MDP: current action affects future states
  • Key test: does today's action change tomorrow's context?
  • Episodic: clear start/end (e.g., ICU stays)
  • Continuous: no terminal state
Medium Frequency
Topic 6

DQN

  • Experience replay: breaks correlation
  • Target network: separate frozen network for TD target
  • Loss: (r + γ·max Q_target(s') − Q(s,a))²
  • Deadly triad: FA + bootstrapping + off-policy
Medium Frequency
Topic 7

MCTS & AlphaGo/Zero

  • UCB for trees: Q(s,a) + c·√(N(s)/N(s,a))
  • 4 phases: Select → Expand → Simulate → Backprop
  • AlphaGo: trained on human games + self-play
  • AlphaZero: self-play only → more general
Medium Frequency
Topic 8

Imitation Learning

  • BC loss: L = −Σ log π_θ(aᴱ|s)
  • Failure: covariate shift (unseen states at test)
  • DAGGER: aggregate expert labels on policy's own states
  • Advantage: policy learns to recover from mistakes
Lower Frequency

3. Key Formulas — Cheat Sheet

UCB Action Selection (MAB)
$$ a^* = \arg\max_a \left[ Q(a) + c \cdot \sqrt{\frac{\ln t}{N(a)}} \right] $$ \(t\) = total time steps so far  ·  \(N(a)\) = times \(a\) was chosen
Exponential Recency-Weighted Average (EWMA)
$$ Q_{n+1}(a) = Q_n(a) + \alpha \left[ R_n - Q_n(a) \right] $$ \(\alpha \in (0,1]\)  ·  \(\alpha = 0.5\) common in exams
Bellman Expectation (Policy Evaluation)
$$ V_\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\left[ R(s,a,s') + \gamma V_\pi(s') \right] $$
Bellman Optimality (Value Iteration)
$$ V^*(s) = \max_a \sum_{s'} P(s'|s,a)\left[ R + \gamma V^*(s') \right] $$
Discounted Return (MC)
$$ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots $$ Compute backwards from terminal state
First-Visit MC Update
$$ V(s) \leftarrow V(s) + \frac{1}{n}\left[ G - V(s) \right] $$ \(n\) = number of times \(s\) has been visited so far
Importance Sampling Ratio
$$ \rho = \prod_t \frac{\pi(a_t|s_t)}{b(a_t|s_t)} $$ \(\pi\) = target policy  ·  \(b\) = behaviour policy
Policy Gradient Update (REINFORCE)
$$ \theta \leftarrow \theta + \alpha (G - b)\, \nabla_\theta \ln \pi_\theta(a|s) $$ \(b\) = baseline (often \(V(s)\) or a constant)
Linear Softmax Policy
$$ \pi(a|s) = \frac{\exp(\theta^\top \phi(s,a))}{\sum_{a'} \exp(\theta^\top \phi(s,a'))} $$ \(\phi(s)\) = feature vector of state \(s\)
UCB for MCTS (Tree)
$$ \text{UCB}(s,a) = Q(s,a) + c\sqrt{\frac{\ln N(s)}{N(s,a)}} $$ \(N(s)\) = parent visits  ·  \(N(s,a)\) = child visits
DQN Loss
$$ \mathcal{L}(\theta) = \mathbb{E}\!\left[\left(r + \gamma \max_{a'} Q_{\theta^-}(s',a') - Q_\theta(s,a)\right)^2\right] $$ \(\theta^-\) = frozen target network weights
Behavior Cloning Loss
$$ \mathcal{L}_{\text{BC}}(\theta) = -\frac{1}{N}\sum_i \log \pi_\theta(a_i^E \mid s_i) $$ \(a^E\) = expert action at state \(s\)

Part (c) — Comparison [2 Marks]

  • From (a): UCB selects C over B only if r < −3.92 — the agent's choice is robust unless Zone B performs catastrophically.
  • From (b): EWMA with α=0.5 gives equal weight to all past rewards (exponentially decaying). Each older observation's weight = 0.5ⁿ.
  • If α=1.0, only the last reward matters — highly sensitive to r. If α→0, old rewards dominate — very insensitive to r.
  • α=0.5 balances recent and historical performance, making dispatch moderately sensitive to r.

Solution — Question 2 (MDP: HVAC Control)

Part (a) — Bellman Equation for V_π(M) [2 Marks]

Policy π: π(Maintain | M) = 1. From M under Maintain: goes to M with P=0.8, r=+2; goes to H with P=0.2, r=+2.

V_π(M) = Σ_s' P(s'|M, Maintain) · [R + γ·V_π(s')] = 0.8 · [2 + 0.9·V(M)] + 0.2 · [2 + 0.9·V(H)] = 0.8·[2 + 0] + 0.2·[2 + 0] (using V⁰ = 0 for all) = 1.6 + 0.4 = 2.0

V_π(M) ≈ 2.0 (first iteration from V⁰ = 0)

The full convergence needs iteration until stable. In the exam — one iteration is usually sufficient unless asked otherwise.

Part (b) — Policy Improvement at M [4 Marks]

Given: V⁰(C)=1, V⁰(M)=2, V⁰(H)=0. π₀: Maintain everywhere.

Compute Q_π(M, Increase):

Q(M, I) = Σ_s' P(s'|M,I)·[R + γ·V(s')] = 0.5·[1 + 0.9·1] + 0.5·[1 + 0.9·2] (goes to C or M) = 0.5·[1 + 0.9] + 0.5·[1 + 1.8] = 0.5·1.9 + 0.5·2.8 = 0.95 + 1.40 = 2.35

Compute Q_π(M, Maintain):

Q(M, K) = 0.8·[2 + 0.9·2] + 0.2·[2 + 0.9·0] (goes to M or H) = 0.8·[2 + 1.8] + 0.2·[2 + 0] = 0.8·3.8 + 0.2·2.0 = 3.04 + 0.40 = 3.44

Maintain (K)3.44✓ Higher

Policy does NOT change: π₁(Maintain | M) = 1 (Maintain is still better).

Part (c) — Episodic or Continuous? [1 Mark]

This is a continuous task. The HVAC system runs indefinitely with no terminal state — thermal management never "ends".

Part (d) — Off-Policy MC with Importance Sampling [3 Marks]

Episode: (M, Maintain, +2, M) → (M, Increase, +1, C) → (C, Maintain, +1, Terminal)

Target policy π: π(Maintain|M) = 1, π(Maintain|C) = 1

Behavior policy b: b(Increase|M) = 0.6, b(Maintain|M) = 0.4, b(Maintain|C) = 1.0

d-1: Compute ρ (IS ratio from first M onwards)

StepStateAction Takenπ(a|s)b(a|s)Ratio
1MMaintain1.00.41.0/0.4 = 2.5
2MIncrease0.00.60.0/0.6 = 0
3CMaintain1.01.01.0

ρ = 2.5 × 0 × 1.0 = 0

d-2: Return G from M (γ = 0.9)

G = 2 + 0.9·1 + 0.9²·1 = 2 + 0.9 + 0.81 = 3.71

d-3: Weighted IS update

V(M) ← V(M) + (ρ / Σρ) · (ρ·G − V(M)) Since ρ = 0, this episode contributes nothing to V(M). The target policy would never take "Increase" at M — so this trajectory is incompatible with π and gets zero weight.
Key insight: off-policy IS can produce zero updates when behavior and target policies diverge. This is why off-policy methods need diverse exploration in b.

Part (d) — Without Baseline [1 Mark]

Without b=1, update = α·G·∇ln π. All positive returns (G>0) always push probability up for the chosen action — even if G=0.5 is poor. The tutor could overfit to recommending Hard tasks that gave marginally positive returns, ignoring that Medium tasks gave much higher returns. High variance in updates makes learning unstable.

Solution — Question 4 (MCTS + DQN + Imitation Learning)

Part (a) — UCB Scores in MCTS [3 Marks]

N(s) = 25, c = 1.5. UCB(s,a) = Q(s,a) + 1.5·√(ln N(s) / N(s,a))

ln(25) = 3.219
ActionN(s,a)Q(s,a)Exploration term: 1.5·√(3.219/N)UCB Score
a₁50.551.5·√(3.219/5) = 1.5·√0.644 = 1.5·0.802 = 1.2040.55 + 1.204 = 1.754
a₂200.621.5·√(3.219/20) = 1.5·√0.161 = 1.5·0.401 = 0.6010.62 + 0.601 = 1.221

UCB(a₁) = 1.754 > UCB(a₂) = 1.221 → Select a₁

Although a₂ has a higher mean Q value (0.62 > 0.55), a₁ has been visited far fewer times (5 vs 20), giving it a much larger exploration bonus. MCTS prioritizes exploring under-tried actions.

Part (b) — DQN Stabilization Techniques [2 Marks]

TechniqueWhat it doesWhy it addresses instability
Experience Replay Stores transitions (s,a,r,s') in a replay buffer; samples random mini-batches for training Breaks temporal correlation between consecutive updates. Without it, consecutive correlated samples cause the network to "forget" old patterns, leading to oscillation.
Target Network Maintains a separate frozen copy of Q (Q_θ⁻) used only to compute TD targets; updated only every C steps Prevents the TD target from moving every step. Without it, the target and estimated values chase each other in a feedback loop — the "deadly triad" of instability.

Part (c) — AlphaGo vs AlphaZero [2 Marks]

AspectAlphaGoAlphaZero
Training data sourceHuman expert games + self-playSelf-play only (no human data)
GeneralizabilityLimited — biased towards human strategiesHigher — discovers strategies unconstrained by human play patterns
Domain specificityGo-specific human openings baked inSame algorithm applied to Chess, Shogi, Go without modification

AlphaZero is more generalizable because it learns purely from the game rules, not from human demonstrations. Human data introduces a prior bias — AlphaZero's self-play discovers strategies humans never attempted.

Part (d-1) — Behavior Cloning Loss [1.5 Marks]

L_BC(θ) = −(1/N) · Σᵢ log π_θ(aᵢᴱ | sᵢ) where aᵢᴱ = expert action at state sᵢ N = number of expert demonstration transitions

Failure mode — Covariate Shift: At training time, the policy only sees states the expert visited. At test time, the policy's own actions lead to states not in the expert distribution. The policy has no experience in these unseen states and makes compounding errors (error → new unseen state → worse error).

Part (d-2) — DAGGER [1.5 Marks]

AspectBehavior CloningDAGGER
Data collectionFixed expert demonstrations onlyIterative: run current policy, collect states; ask expert to label those states
Training distributionExpert state distribution onlyMixture of expert + policy's own visited states
Handles own mistakes?No — unseen recovery states cause driftYes — policy learns to recover from states it actually visits

Why DAGGER helps: Since the expert labels states the policy actually reaches (including mistake states), the policy learns the correct action even from off-track situations. This directly solves covariate shift by progressively including the policy's own trajectory distribution in the training data.

2. First-visit MC: Only use the first time a (state, action) appears in the episode for the update.
