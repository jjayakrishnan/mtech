The Agent-Environment Interface

Think of an RL agent as a loop. At each time step, the agent looks at the world, picks an action, then the world responds with a reward and a new state. This repeats forever (or until a terminal state).

Analogy

Like a chess player: look at the board (state), move a piece (action), observe the opponent's response and whether you captured something (new state + reward).

Formally, at each step \(t\):

  1. Agent observes state \(S_t\)
  2. Agent selects action \(A_t\)
  3. Environment returns reward \(R_{t+1}\) and next state \(S_{t+1}\)
Trajectory
\[ S_0, A_0, R_1, S_1, A_1, R_2, S_2, A_2, R_3, \ldots \]

Notice: the reward at time \(t+1\) is a consequence of the action at time \(t\). The subscripts matter for exam questions.

┌───────────────────────────────────────┐ │ ENVIRONMENT │ │ │ │ receives A_t produces R_{t+1} │ │ ↓ ↓ │ │ ┌─────────┐ ┌──────────┐ │ │ │ dynamics │────→│ new state│ │ │ │ p(s',r|s,a) │ S_{t+1} │ │ │ └─────────┘ └──────────┘ │ └──────┬──────────────────┬────────────┘ │ │ A_t │ S_t, R_t │ S_{t+1}, R_{t+1} ↑ │ ↓ │ ↓ ┌──────┴──────────────────┴────────────┐ │ AGENT │ │ │ │ observes S_t → policy π(a|s) → A_t │ └───────────────────────────────────────┘

Elements of a Finite MDP

A finite MDP is "finite" because S, A, and R all have a finite number of elements. Every piece fits together like this:

ElementSymbolMeaning
State set\(\mathcal{S}\)All possible states the environment can be in
Action set\(\mathcal{A}(s)\)Actions available in state \(s\)
Reward set\(\mathcal{R} \subset \mathbb{R}\)Finite set of possible rewards
Dynamics\(p(s',r \mid s,a)\)Probability of landing in \(s'\) with reward \(r\) given \(s, a\)
Policy\(\pi(a \mid s)\)Probability of selecting action \(a\) in state \(s\)
Discount factor\(\gamma \in [0,1]\)How much future rewards are worth today
Definition — Finite MDP

A tuple \((\mathcal{S}, \mathcal{A}, \mathcal{R}, p, \gamma)\) where the dynamics function \(p\) fully specifies the environment's behaviour.

The Markov Property

In plain English: the future depends only on where you are now, not how you got here. History is irrelevant once you know the current state.

Analogy

A chess position tells you everything you need to plan your next move. You do not need to replay the entire game from move 1.

Markov Property (formal)
\[ \Pr\{S_{t+1}=s', R_{t+1}=r \mid S_0, A_0, R_1, \ldots, S_{t-1}, A_{t-1}, R_t, S_t, A_t\} = \Pr\{S_{t+1}=s', R_{t+1}=r \mid S_t, A_t\} \]

This is what makes MDPs tractable. If the Markov property did not hold, we would need the entire history to make decisions — computationally impossible for long episodes.

Exam tip

If a question asks "what assumption does an MDP make?" the answer is the Markov property. If it asks "when might this fail?" think of partially observable problems (e.g., a robot that cannot see behind a wall).

The Dynamics Function and Derived Quantities

The dynamics function is the master equation — everything else can be derived from it.

Dynamics function (four-argument)
\[ p(s',r \mid s,a) = \Pr\{S_t = s',\; R_t = r \mid S_{t-1} = s,\; A_{t-1} = a\} \]

It must satisfy: \(\sum_{s' \in \mathcal{S}} \sum_{r \in \mathcal{R}} p(s',r \mid s,a) = 1\) for all \(s \in \mathcal{S}, a \in \mathcal{A}(s)\).

Derived: State-Transition Probability

If we only care about where we end up (not the reward):

State-transition probability
\[ p(s' \mid s,a) = \sum_{r \in \mathcal{R}} p(s',r \mid s,a) \]

Derived: Expected Reward

If we only care about the expected reward for a state-action pair:

Expected reward for state-action pair
\[ r(s,a) = \mathbb{E}[R_t \mid S_{t-1}=s, A_{t-1}=a] = \sum_{r \in \mathcal{R}} r \sum_{s' \in \mathcal{S}} p(s',r \mid s,a) \]
Key insight

The four-argument \(p(s',r|s,a)\) is the most general form. The state-transition probability and expected reward are just marginals/moments of this distribution. On exams, always start from the four-argument form and derive what you need.

The Agent-Environment Boundary

A common confusion: the boundary is about what the agent can control, not what it can see or know.

Robot example

A robot's motors and joints are part of the environment, not the agent. The agent sends signals ("rotate joint 30 degrees"), but the physics of the motor executing that rotation is the environment's job. The agent cannot control whether the motor slips.

Traffic signal example

An RL-controlled traffic light: the agent decides which lights to turn green (action). But the cars, pedestrians, weather — all environment. The agent might know how many cars are waiting (that is the state), but it cannot directly move the cars.

Rule of thumb: anything the agent cannot arbitrarily change is part of the environment. The boundary is the limit of the agent's direct control, not the limit of its knowledge.

Goals and Rewards — The Reward Hypothesis

How do we tell an agent what we want? Not by programming behaviour directly, but by defining a reward signal.

The Reward Hypothesis

All of what we mean by goals and purposes can be well thought of as the maximization of the expected value of the cumulative sum of a received scalar signal (called reward).

Warning — reward design pitfall

The reward must signal what you want achieved, not how to achieve it. If you reward a chess agent for capturing pieces, it might sacrifice its queen to capture two pawns. Reward checkmate, not captures.

The reward is always a single scalar number. Even if the goal is complex (drive safely AND arrive quickly), it must be collapsed into one number per time step. This is both the power and the limitation of the framework.

Returns and the Discount Factor

The agent does not just want high immediate reward — it wants high cumulative reward. The return is how we measure that.

Episodic Return (no discounting)

Episodic (undiscounted) return
\[ G_t = R_{t+1} + R_{t+2} + R_{t+3} + \cdots + R_T \]

Used when episodes have a clear terminal state (games, mazes). The sum is finite because \(T\) is finite.

Discounted Return

Discounted return
\[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]
Recursive form (critical for Bellman equations)
\[ G_t = R_{t+1} + \gamma G_{t+1} \]
Plain English

A reward received \(k\) steps in the future is worth \(\gamma^k\) times what it would be worth right now. It is like compound interest in reverse: future money is worth less today.

The Discount Factor Table

\(\gamma\) valueBehaviourAnalogy
\(\gamma = 0\)Completely myopic — only cares about immediate \(R_{t+1}\)A toddler grabbing the nearest candy
\(0 < \gamma < 1\)Balances short-term and long-termA normal adult planning a career
\(\gamma \to 1\)Far-sighted — future rewards almost as valuable as presentA long-term investor

Numerical Example

Given: \(R_{t+1} = 2,\; R_{t+2} = 3,\; R_{t+3} = 5,\; \gamma = 0.9\)

Calculation
\[ G_t = 2 + (0.9)(3) + (0.9)^2(5) = 2 + 2.7 + 4.05 = 8.75 \]

Without discounting (\(\gamma=1\)): \(G_t = 2+3+5 = 10\). Discounting penalises the later rewards — \(R_{t+3}=5\) contributes only 4.05 instead of 5.

Memorise This for the Exam

Core equations you must be able to write from memory:

p(s',r|s,a) = Pr{S_t=s', R_t=r | S_{t-1}=s, A_{t-1}=a}
p(s'|s,a) = sum_r p(s',r|s,a)
r(s,a) = sum_r r * sum_{s'} p(s',r|s,a)
G_t = R_{t+1} + gamma * G_{t+1}
G_t = sum_{k=0}^{inf} gamma^k * R_{t+k+1}

Key facts:

Markov property: future depends only on (S_t, A_t), not history
Boundary: limit of CONTROL, not knowledge
Reward hypothesis: all goals = max cumulative scalar reward
gamma=0 myopic | gamma→1 far-sighted | gamma in (0,1) balanced

Quick-Fire Quiz

Test yourself before moving on.

Q1: In the trajectory \(S_0, A_0, R_1, S_1, \ldots\), the reward \(R_{t+1}\) is a consequence of which action?
Correct: R_{t+1} results from taking action A_t in state S_t.
Q2: The Markov property states that the future is independent of _____ given the present state and action.
Correct: The entire past trajectory is irrelevant once you condition on the current state.
Q3: With \(\gamma=0.9\), rewards \(R_{t+1}=2, R_{t+2}=3, R_{t+3}=5\), what is \(G_t\)?
Correct: G_t = 2 + 0.9(3) + 0.81(5) = 2 + 2.7 + 4.05 = 8.75
Q4: The agent-environment boundary represents the limit of the agent's ___.
Correct: The boundary is about what the agent can directly control, not what it knows or observes.
Q5: Which of these is NOT derivable from \(p(s',r|s,a)\)?
Correct: The policy is the agent's choice, not part of the environment dynamics. It cannot be derived from p(s',r|s,a).

+12 Analogy — Finite MDP

Think of an MDP as a board game. Each square on the board is a state. Rolling dice and moving is an action + transition. Landing on a coloured square gives a reward (or penalty). The rule book says exactly which square you go to and what you gain — that's \(p(s',r|s,a)\). An episodic MDP is like a board game that ends (someone wins). A continuing MDP is like playing forever with \(\gamma\) making future turns worth less than today's move.

Try It Yourself

S&B Ex 3.8 — Compute Returns G₀–G₅ [6 marks]

Suppose \(\gamma = 0.5\) and the reward sequence is \(R_1 = -1,\; R_2 = 2,\; R_3 = 6,\; R_4 = 3,\; R_5 = 2\), with \(T = 5\). Compute \(G_0, G_1, G_2, G_3, G_4, G_5\). Hint: work backwards from \(G_5 = 0\).

Solution:

  • \(G_5 = 0\) (terminal state)
  • \(G_4 = R_5 + \gamma G_5 = 2 + 0.5(0) = \mathbf{2}\)
  • \(G_3 = R_4 + \gamma G_4 = 3 + 0.5(2) = \mathbf{4}\)
  • \(G_2 = R_3 + \gamma G_3 = 6 + 0.5(4) = \mathbf{8}\)
  • \(G_1 = R_2 + \gamma G_2 = 2 + 0.5(8) = \mathbf{6}\)
  • \(G_0 = R_1 + \gamma G_1 = -1 + 0.5(6) = \mathbf{2}\)

Note: the large reward at \(t=3\) (\(R_3=6\)) propagates backward and counteracts the negative \(R_1\), resulting in \(G_0 = 2\).

S&B Ex 3.9 — Infinite Reward Sequence [4 marks]

Suppose \(\gamma = 0.9\) and the reward sequence is \(R_1 = 2\) followed by an infinite sequence of 7s (\(R_2 = R_3 = \cdots = 7\)). Compute \(G_1\) and \(G_0\).

Solution:

\(G_1\) is an infinite geometric series of constant reward \(r=7\):

$$ G_1 = \sum_{k=0}^{\infty} (0.9)^k \cdot 7 = \frac{7}{1-0.9} = \frac{7}{0.1} = \mathbf{70} $$

Then using the recursive form:

$$ G_0 = R_1 + \gamma G_1 = 2 + 0.9 \times 70 = 2 + 63 = \mathbf{65} $$
S&B Ex 3.17 — Bellman Equation for \(q_\pi\) [5 marks]

Write the Bellman equation for the action-value function \(q_\pi(s,a)\). Express it in terms of the four-argument dynamics \(p(s',r|s,a)\) and the state-value function \(v_\pi(s')\). Use the backup diagram for \(q_\pi\) as a guide.

Solution:

$$ q_\pi(s,a) = \sum_{s',r} p(s',r|s,a)\!\left[r + \gamma\, v_\pi(s')\right] $$

The value of taking action \(a\) in state \(s\) equals the expected immediate reward plus the discounted value of the next state, averaged over all \((s', r)\) under the dynamics.

Compare with the Bellman equation for \(v_\pi\): it goes one level deeper, also averaging over the policy \(\pi(a|s)\) before taking the action.
