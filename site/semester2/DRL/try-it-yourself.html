L1 RL Framework

▶ Exercise 1.1 — Identify RL Components [2 marks]

A drone delivery system learns to navigate city streets to minimise delivery time while avoiding collisions. At each timestep the controller chooses a direction; the GPS system measures arrival time and collision events.

Identify:

  1. Agent, Environment, State, Action, Reward
  2. Is the task episodic or continuing? Justify.
✓ Solution
ComponentInstantiation
AgentDrone navigation controller
EnvironmentCity streets, GPS system, weather, other obstacles
State \(S_t\)Current position, speed, nearby obstacles, remaining battery
Action \(A_t\)Direction to fly (N / S / E / W / hover) + speed setting
Reward \(R_{t+1}\)\(+\)large on successful delivery, \(-\)small per timestep (time cost), \(-\)large on collision

Episodic. Each delivery is one episode — it starts when the drone picks up the package and ends when it delivers or crashes. Returns reset at episode boundaries. If the drone flew indefinitely without episode resets, it would be continuing.

▶ Exercise 1.2 — RL vs Supervised Learning [1.5 marks]

A hospital wants to learn a ventilator control policy from patient data. Engineer A proposes supervised learning on historical records; Engineer B proposes RL.

Give two reasons why RL is the better fit.

✓ Solution
  1. No correct labels — only evaluative feedback. Historical records show what doctors did, not what was optimal. RL learns from a reward signal (patient outcome); supervised learning requires instructive labels telling the agent the correct action — these do not exist here.
  2. Sequential decisions with delayed consequences. A ventilator setting at hour 1 may not affect patient stability until hour 6. RL explicitly handles long-horizon credit assignment via the return \( G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \). Supervised learning treats each timestep independently and cannot capture this temporal dependency.

L2 MAB vs Finite MDP

▶ Exercise 2.1 — Classify Three Scenarios [3 marks]

For each scenario state MAB or Finite MDP and give one justification.

  1. A website tests three ad variants. Each page load one variant is shown; a click (reward = 1) or no-click (reward = 0) is recorded. Each page load is independent of the previous.
  2. A robot arm picks objects from a conveyor. After each pick the arm position changes, and the next pick depends on where the arm currently is.
  3. A blood glucose monitor chooses one of three insulin doses per meal. The same dose tables are used regardless of previous meals (no state carryover).
✓ Solution
ScenarioClassificationJustification
AMABNo state transition — each page load is independent. The action (variant shown) affects only that round's reward, not future states.
BFinite MDPActions change the state (arm position). The next observation depends on the current action, satisfying the Markov property with state transitions.
CMABNo carryover state across meals — each meal is an independent bandit pull. Actions affect only immediate reward, not future state.

Key diagnostic: Does taking an action change the state that influences future rewards? If yes → MDP. If no → MAB.

L4 MAB Action-Value Methods

▶ Exercise 3.1 — Compute Q(a) and Apply ε-greedy [3 marks]

A 3-armed bandit has the following reward history:

ArmRewards observed so far
\(A_1\)4, 2, 6
\(A_2\)5, 5
\(A_3\)1

Using the sample-average method:

  1. Compute \( Q(A_1),\; Q(A_2),\; Q(A_3) \).
  2. With \( \varepsilon = 0.1 \), state the probability of selecting each arm on the next step.
  3. Which arm does a pure greedy agent select?
✓ Solution

1. Sample averages:

$$Q(A_1) = \frac{4+2+6}{3} = \frac{12}{3} = 4.0 \qquad Q(A_2) = \frac{5+5}{2} = 5.0 \qquad Q(A_3) = \frac{1}{1} = 1.0$$

2. ε-greedy probabilities (\(\varepsilon = 0.1\), 3 arms):

Greedy arm = \(A_2\). With probability \(1-\varepsilon\) the greedy arm is chosen; with probability \(\varepsilon\) one of the 3 arms is chosen uniformly at random.

Arm\(Q(a)\)\(P(\text{select})\)Why
\(A_1\)4.0\(\varepsilon/3 = 0.1/3 \approx 0.033\)Non-greedy, random share only
\(A_2\)5.0\((1-\varepsilon) + \varepsilon/3 = 0.9 + 0.033 = 0.933\)Greedy arm + random share
\(A_3\)1.0\(\varepsilon/3 \approx 0.033\)Non-greedy, random share only

3. Pure greedy selects \(A_2\) (\(Q = 5.0\), highest).

▶ Exercise 3.2 — Optimistic Initial Values [1.5 marks]

A 3-armed bandit has true expected rewards: \(q^*(A_1)=2,\; q^*(A_2)=5,\; q^*(A_3)=3\). Both agents start with \(\varepsilon=0\) (greedy) but differ in initial estimates:

  • Agent X: \(Q_1(a) = 0\) for all arms.
  • Agent Y: \(Q_1(a) = 10\) for all arms (optimistic).

Explain how Agent Y explores more than Agent X in early steps, despite both being greedy.

✓ Solution

Agent X is greedy from the start. On step 1 it picks any arm (ties broken arbitrarily, say \(A_1\)). After observing reward ≈ 2, it updates \(Q(A_1) \approx 2\), which is still the highest (since all others are 0 until pulled). It may never explore other arms systematically.

Agent Y starts with \(Q = 10\) for all arms — much higher than any real reward. After pulling \(A_1\) and seeing reward ≈ 2, \(Q(A_1)\) drops to ≈ 6 (average of 10 and 2). Now \(A_2\) and \(A_3\) still have \(Q = 10\), so Agent Y greedily switches to one of them. Disappointment drives exploration: every arm gets pulled until its estimate falls in line with reality. This forces early exploration without requiring \(\varepsilon > 0\).

Downside: optimistic initialisation only drives exploration in early steps — it cannot adapt to non-stationary problems where the optimal arm shifts later.

L5 Incremental Updates & Tracking Non-Stationary

▶ Exercise 4.1 — Incremental Q Update [2 marks]

An arm has been pulled 4 times, giving current estimate \( Q_4 = 3.5 \). On pull 5, the reward observed is \( R_5 = 7.0 \).

Using the incremental sample-average update rule, compute \( Q_5 \). Show the prediction-error term.

✓ Solution

Incremental update rule:

$$Q_{n+1} = Q_n + \frac{1}{n}\bigl[R_n - Q_n\bigr]$$

Here \(n = 5\) (the 5th pull just occurred), \(Q_4 = 3.5\), \(R_5 = 7.0\):

$$Q_5 = 3.5 + \frac{1}{5}\bigl[7.0 - 3.5\bigr] = 3.5 + \frac{1}{5}(3.5) = 3.5 + 0.70 = \mathbf{4.2}$$

Prediction error \([R_n - Q_n] = 3.5\) — the estimate was too low. We move a fraction \(1/n = 0.2\) of the way toward the new observation. As \(n\) grows this step shrinks, so old rewards carry increasing weight.

▶ Exercise 4.2 — Constant-α Tracking [2 marks]

An arm's true reward distribution changes over time (non-stationary). You use constant step size \(\alpha = 0.3\). Current estimate \(Q = 2.0\). Observe three consecutive rewards: \(R_1 = 5,\; R_2 = 5,\; R_3 = 5\).

  1. Compute \(Q\) after each of the 3 updates.
  2. Why is constant-\(\alpha\) preferred over \(1/n\) for non-stationary problems?
✓ Solution

1. Updates — rule: \(Q \leftarrow Q + \alpha[R - Q]\)

Step\(R\)Error \([R-Q]\)Update \(Q + 0.3 \times \text{error}\)New \(Q\)
15\(5 - 2.0 = 3.0\)\(2.0 + 0.3 \times 3.0\)2.90
25\(5 - 2.9 = 2.1\)\(2.9 + 0.3 \times 2.1\)3.53
35\(5 - 3.53 = 1.47\)\(3.53 + 0.3 \times 1.47\)3.97

2. Why constant-\(\alpha\)? With \(1/n\), the step size shrinks toward zero — old and new rewards eventually get equal weight and the estimate stops tracking changes. With constant \(\alpha\), the most recent reward always has the most influence (exponential recency weighting): \( Q_{n+1} = (1-\alpha)^n Q_1 + \sum_{i=1}^{n} \alpha(1-\alpha)^{n-i} R_i \). For a non-stationary arm whose true value shifts, constant-\(\alpha\) keeps the estimate agile; \(1/n\) would converge to an outdated average.

L5 UCB Action Selection

▶ Exercise 5.1 — Compute UCB Scores and Select Action [3 marks]

At time step \(t = 10\) a 3-armed bandit has the following history:

ArmTimes pulled \(N_t(a)\)Estimated value \(Q_t(a)\)
\(A_1\)54.0
\(A_2\)36.0
\(A_3\)25.5

UCB formula with \(c = 2\):

$$A_t = \arg\max_a \left[ Q_t(a) + c\sqrt{\frac{\ln t}{N_t(a)}} \right]$$

Compute the UCB score for each arm and state which action is selected. (\(\ln 10 \approx 2.303\))

✓ Solution
Arm\(Q_t(a)\)\(N_t(a)\)\(\sqrt{\ln 10\,/\,N}\)Bonus \(2 \times \sqrt{\cdots}\)UCB score
\(A_1\)4.05\(\sqrt{2.303/5}=\sqrt{0.461}=0.679\)\(1.357\)\(4.0 + 1.357 = \mathbf{5.36}\)
\(A_2\)6.03\(\sqrt{2.303/3}=\sqrt{0.768}=0.876\)\(1.753\)\(6.0 + 1.753 = \mathbf{7.75}\)
\(A_3\)5.52\(\sqrt{2.303/2}=\sqrt{1.152}=1.073\)\(2.146\)\(5.5 + 2.146 = \mathbf{7.65}\)

Select \(A_2\) (UCB score = 7.75, highest). Note: \(A_3\) has a larger uncertainty bonus (pulled only twice) but \(A_2\)'s high Q still wins. If \(A_2\) is pulled again and its Q drops, \(A_3\) may become the UCB choice next.

Intuition: \(c\) controls the width of the confidence interval. \(c = 0\) → pure greedy; larger \(c\) → more exploration of uncertain arms.

L6 Finite Markov Decision Processes

▶ Exercise 6.1 — Compute Discounted Return \(G_t\) [2 marks]

An agent receives rewards: \(R_1 = 2,\; R_2 = 4,\; R_3 = 6\), then the episode terminates (\(R_4 = 0\)). Discount factor \(\gamma = 0.9\).

Compute:

  1. \(G_0\) — return from \(t = 0\)
  2. \(G_1\) — return from \(t = 1\)
  3. Verify the recursive property: \(G_0 = R_1 + \gamma G_1\)
✓ Solution

1. \(G_0\):

$$G_0 = R_1 + \gamma R_2 + \gamma^2 R_3 = 2 + 0.9(4) + 0.81(6) = 2 + 3.6 + 4.86 = \mathbf{10.46}$$

2. \(G_1\):

$$G_1 = R_2 + \gamma R_3 = 4 + 0.9(6) = 4 + 5.4 = \mathbf{9.4}$$

3. Recursive check:

$$R_1 + \gamma G_1 = 2 + 0.9 \times 9.4 = 2 + 8.46 = 10.46 \checkmark$$

This recursive relation \(G_t = R_{t+1} + \gamma G_{t+1}\) is the backbone of DP, TD, and MC methods — it lets us compute returns backward through time.

▶ Exercise 6.2 — Infinite Discounted Return [1.5 marks]

A continuing task generates a constant reward of \(R = 3\) at every timestep forever. Discount factor \(\gamma = 0.8\). Compute \(G_0\).

✓ Solution

Constant-reward geometric series:

$$G_0 = \sum_{k=0}^{\infty} \gamma^k R = R \cdot \frac{1}{1-\gamma} = 3 \cdot \frac{1}{1 - 0.8} = 3 \cdot 5 = \mathbf{15}$$

The series converges because \(\gamma < 1\). Without discounting (\(\gamma = 1\)) the sum diverges to infinity — we could not rank policies. Discounting makes the infinite-horizon problem tractable and models the intuition that future rewards are worth less than immediate ones.

L7 Bellman Equations & Optimality

▶ Exercise 7.1 — Solve Vπ from Bellman Equations [3 marks]

A 3-state MDP: \(\{A,\, B,\, T\}\) where \(T\) is terminal (\(V(T) = 0\)).

Policy \(\pi\) always takes action "right". Transitions under \(\pi\):

FromTo \(s'\)ProbabilityReward \(r\)
\(A\)\(B\)1.0+2
\(B\)\(T\)0.8+5
\(B\)\(A\)0.2−1

Discount \(\gamma = 0.9\). Solve the system of Bellman equations for \(V_\pi(A)\) and \(V_\pi(B)\).

✓ Solution

Write the two Bellman equations:

$$V_\pi(A) = 2 + 0.9\,V_\pi(B) \tag{1}$$
$$V_\pi(B) = 0.8\bigl(5 + 0.9 \cdot 0\bigr) + 0.2\bigl(-1 + 0.9\,V_\pi(A)\bigr) \tag{2}$$

Expand (2):

$$V_\pi(B) = 4.0 - 0.2 + 0.18\,V_\pi(A) = 3.8 + 0.18\,V_\pi(A)$$

Substitute into (1):

$$V_\pi(A) = 2 + 0.9\bigl(3.8 + 0.18\,V_\pi(A)\bigr) = 2 + 3.42 + 0.162\,V_\pi(A)$$
$$V_\pi(A)\,(1 - 0.162) = 5.42 \implies V_\pi(A) = \frac{5.42}{0.838} \approx \mathbf{6.47}$$
$$V_\pi(B) = 3.8 + 0.18 \times 6.47 = 3.8 + 1.16 \approx \mathbf{4.96}$$
State\(V_\pi(s)\)
\(A\)6.47
\(B\)4.96
\(T\)0 (terminal)

Notice \(V_\pi(A) > V_\pi(B)\): even though \(B\) is one step closer to the high-reward terminal transition, the possibility of looping back to \(A\) with \(-1\) reward pulls \(V(B)\) down.

L8 Dynamic Programming

▶ Exercise 8.1 — Policy Evaluation Sweeps [3 marks]

A 1×3 gridworld: states \(\{s_1, s_2, s_3\}\). \(s_3\) is terminal (\(V = 0\) always). Policy: always move right. Transitions deterministic: \(s_1 \xrightarrow{-1} s_2 \xrightarrow{-1} s_3\). \(\gamma = 1\).

Initial values: \(V_0(s_1) = V_0(s_2) = 0\).

Perform two full sweeps of the Bellman evaluation update. Show \(V_1\) and \(V_2\) for each state.

✓ Solution

Update rule: \(V_{k+1}(s) = R + \gamma\,V_k(s')\)

Sweep 1 (using \(V_0\) values throughout):

StateCalculation\(V_1(s)\)
\(s_1\)\(-1 + 1 \cdot V_0(s_2) = -1 + 0\)−1
\(s_2\)\(-1 + 1 \cdot V_0(s_3) = -1 + 0\)−1
\(s_3\)terminal0

Sweep 2 (using \(V_1\) values):

StateCalculation\(V_2(s)\)
\(s_1\)\(-1 + 1 \cdot V_1(s_2) = -1 + (-1)\)−2
\(s_2\)\(-1 + 1 \cdot V_1(s_3) = -1 + 0\)−1
\(s_3\)terminal0

Converged. True values: \(V^*(s_1) = -2,\; V^*(s_2) = -1,\; V^*(s_3) = 0\). After exactly 2 sweeps under \(\gamma = 1\), values have propagated fully backward from the terminal state.

▶ Exercise 8.2 — Policy Improvement Step [2 marks]

At state \(s\) there are two available actions with known action-values:

Action\(Q_\pi(s, a)\)
Left−3.0
Right+1.5

Current policy \(\pi(s)\): Left with probability 0.6, Right with probability 0.4.

  1. Compute \(V_\pi(s)\).
  2. State the greedy improved policy \(\pi'(s)\).
  3. Compute \(V_{\pi'}(s)\). Does the Policy Improvement Theorem hold?
✓ Solution

1. \(V_\pi(s)\):

$$V_\pi(s) = \sum_a \pi(a|s)\,Q_\pi(s,a) = 0.6 \times (-3.0) + 0.4 \times 1.5 = -1.8 + 0.6 = \mathbf{-1.2}$$

2. Improved policy \(\pi'\): Take Right with probability 1 (greedy: highest \(Q = 1.5\)).

3. \(V_{\pi'}(s) = Q_\pi(s, \text{Right}) = \mathbf{1.5}\)

Policy Improvement Theorem: \(V_{\pi'}(s) = 1.5 \geq V_\pi(s) = -1.2\) ✓ — greedy improvement always produces a policy at least as good at every state.

L9 Monte Carlo Methods

▶ Exercise 9.1 — First-Visit MC Returns [2 marks]

An episode visits states in order: \(s_1,\, s_2,\, s_1,\, s_3\) (terminal). Rewards received: \(R_1=2,\; R_2=4,\; R_3=6\). Discount \(\gamma = 0.9\).

Using first-visit MC:

  1. Identify the first visit to each non-terminal state.
  2. Compute the return \(G\) used to update each state's value estimate.
✓ Solution

1. First visits:

StateFirst visit (timestep)Note
\(s_1\)\(t = 0\)Visited again at \(t=2\) — ignored by first-visit rule
\(s_2\)\(t = 1\)Only visit

2. Returns from first visits:

\(G(s_1,\; t=0)\) — full episode from \(t=0\):

$$G = R_1 + \gamma R_2 + \gamma^2 R_3 = 2 + 0.9(4) + 0.81(6) = 2 + 3.6 + 4.86 = \mathbf{10.46}$$

\(G(s_2,\; t=1)\) — from \(t=1\) onward:

$$G = R_2 + \gamma R_3 = 4 + 0.9(6) = 4 + 5.4 = \mathbf{9.4}$$

These returns are appended to running lists. After many episodes, \(V(s) \approx \text{mean of all } G\text{ values collected for that state}\).

▶ Exercise 9.2 — Update V(s) Running Average [2 marks]

State \(s_A\) has been visited in 3 previous episodes with returns 5.0, 7.0, 6.0. Current estimate: \(V(s_A) = 6.0\).

A 4th episode visits \(s_A\) and gives return \(G = 10.0\).

  1. Update \(V(s_A)\) using the incremental sample-average formula (\(N=4\) after this visit).
  2. What would the updated value be using constant \(\alpha = 0.25\)?
  3. For which problem type is each method better?
✓ Solution

1. Incremental average (\(N = 4\)):

$$V(s_A) \leftarrow 6.0 + \frac{1}{4}\bigl[10.0 - 6.0\bigr] = 6.0 + 1.0 = \mathbf{7.0}$$

Verify: \((5 + 7 + 6 + 10)/4 = 28/4 = 7.0\; \checkmark\)

2. Constant \(\alpha = 0.25\):

$$V(s_A) \leftarrow 6.0 + 0.25\bigl[10.0 - 6.0\bigr] = 6.0 + 1.0 = \mathbf{7.0}$$

(Same result here because \(\alpha = 1/N = 0.25\) coincidentally.)

3. For \(N > 4\): \(1/N\) step shrinks below 0.25, so the sample-average weights older episodes more — good for stationary environments where the true value does not change. Constant-\(\alpha\) always weights recent returns more strongly (exponential recency) — better for non-stationary environments where the optimal policy shifts over time.
