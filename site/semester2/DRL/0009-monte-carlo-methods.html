1. Why Monte Carlo?

Dynamic Programming requires the full model — transition probabilities \(p(s', r \mid s, a)\). But in most real problems, we do not know the model. We only have the ability to interact with the environment and observe what happens.

Analogy

DP is like having the full map of a city and computing the best route on paper. Monte Carlo is like actually driving through the city many times and averaging how long each route took. You learn from experience, not blueprints.

Monte Carlo methods are model-free: they learn directly from episodes of experience. An episode is a complete sequence from start to terminal state:

\[ S_0, A_0, R_1, S_1, A_1, R_2, \ldots, S_{T-1}, A_{T-1}, R_T, S_T(\text{terminal}) \]
Key Requirement

MC methods require episodic tasks — the environment must eventually terminate. They learn from complete returns (must reach terminal state before updating).

2. The Key Idea

The value of a state is the expected return from that state. Monte Carlo simply estimates this by averaging the actual returns observed after visiting that state across many episodes.

Core Principle

Run policy many times. Each time you visit state \(s\), record the return \(G_t\) you got from that point onward. Average all those returns. By the law of large numbers, this converges to \(v_\pi(s)\).

Compare with DP which uses the Bellman equation to bootstrap from neighboring state values. MC uses no bootstrapping — it waits until the end of the episode and uses the actual, observed return.

3. First-Visit MC Policy Evaluation

The most common MC prediction algorithm. Goal: estimate \(V(s) \approx v_\pi(s)\) for all states.

Algorithm

  1. Initialize \(V(s)\) arbitrarily for all \(s\), and \(\text{Returns}(s) \leftarrow \) empty list for all \(s\).
  2. Generate an episode using policy \(\pi\): \(S_0, A_0, R_1, S_1, \ldots, S_T\).
  3. For each state \(s\) appearing in the episode (first time only):
    • \(G \leftarrow\) return following the first occurrence of \(s\)
    • Append \(G\) to \(\text{Returns}(s)\)
    • \(V(s) \leftarrow \text{average}(\text{Returns}(s))\)
  4. Repeat (generate more episodes) until convergence.
Exam Tip

In exam questions, you typically get one episode and must compute returns. "First-visit" means if a state appears multiple times, you only use the first occurrence for that state's return calculation.

4. Computing Returns from an Episode

Given an episode with rewards \(R_1, R_2, \ldots, R_T\) and discount factor \(\gamma\), the return from time step \(t\) is:

\[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots + \gamma^{T-t-1} R_T \]

Efficient backward computation: Start from the end and work backward:

\[ G_T = 0 \quad \text{(terminal)} \] \[ G_t = R_{t+1} + \gamma \, G_{t+1} \]
Why Backward?

Think of it like computing a running total from the back of a receipt. Each line's total is its own value plus a discounted version of the running total below it. One pass, \(O(T)\), gives you returns for every time step.

Example: Backward Return Calculation

Episode: rewards = [+1, +1, -3, +1], \(\gamma = 0.9\)

Step 1 (from terminal)

\(G_4 = 0\) (terminal)

Step 2

\(G_3 = R_4 + \gamma G_4 = 1 + 0.9(0) = 1.0\)

Step 3

\(G_2 = R_3 + \gamma G_3 = -3 + 0.9(1) = -2.1\)

Step 4

\(G_1 = R_2 + \gamma G_2 = 1 + 0.9(-2.1) = -0.89\)

Step 5

\(G_0 = R_1 + \gamma G_1 = 1 + 0.9(-0.89) = 0.199\)

5. First-Visit vs Every-Visit MC

AspectFirst-Visit MCEvery-Visit MC
Which visits count?Only the first time \(s\) appears in an episodeEvery time \(s\) appears
BiasUnbiased estimatorBiased (but consistent)
VarianceHigher (fewer samples)Lower (more samples)
ConvergenceConverges to \(v_\pi(s)\)Also converges to \(v_\pi(s)\)
Exam defaultUnless stated otherwise, assume First-Visit
Watch Out

If a state appears at \(t=0\) and \(t=3\) in the same episode, first-visit only uses the return from \(t=0\). Every-visit uses returns from both \(t=0\) and \(t=3\) as separate samples.

6. MC for Action-Values (Q-values)

For control (policy improvement), we need \(Q(s,a)\) not just \(V(s)\) — because without a model we cannot do the one-step lookahead that converts \(V\) into a policy.

Same idea, but track state-action pairs instead of just states:

  1. Initialize \(Q(s,a)\) for all \((s,a)\), \(\text{Returns}(s,a) \leftarrow \) empty list.
  2. Generate episode using policy \(\pi\).
  3. For each \((s,a)\) pair appearing in the episode (first time only):
    • \(G \leftarrow\) return following first occurrence of \((s,a)\)
    • Append \(G\) to \(\text{Returns}(s,a)\)
    • \(Q(s,a) \leftarrow \text{average}(\text{Returns}(s,a))\)
Key Difference

A "visit" to \((s,a)\) means the agent was in state \(s\) AND took action \(a\). Being in state \(s\) but taking a different action does not count as a visit to \((s,a)\).

7. On-Policy MC Control with \(\varepsilon\)-Soft Policies

The exploration problem: to estimate \(Q(s,a)\) accurately, we need to visit every \((s,a)\) pair. A deterministic greedy policy will never try suboptimal actions. Solution: use an \(\varepsilon\)-soft (e.g., \(\varepsilon\)-greedy) policy.

\(\varepsilon\)-Greedy Policy

\[ \pi(a \mid s) = \begin{cases} 1 - \varepsilon + \frac{\varepsilon}{|\mathcal{A}(s)|} & \text{if } a = \arg\max_a Q(s,a) \\ \frac{\varepsilon}{|\mathcal{A}(s)|} & \text{otherwise} \end{cases} \]

This ensures every action has at least probability \(\frac{\varepsilon}{|\mathcal{A}|}\), guaranteeing continued exploration.

On-Policy MC Control Algorithm

  1. Initialize \(Q(s,a)\) arbitrarily, \(\pi \leftarrow \varepsilon\)-greedy w.r.t. \(Q\).
  2. Generate episode using \(\pi\).
  3. For each \((s,a)\) in episode (first-visit): update \(Q(s,a)\) with observed return.
  4. Policy improvement: For each \(s\) visited, set \(\pi\) to be \(\varepsilon\)-greedy w.r.t. updated \(Q\).
  5. Repeat.
Exam Tip

After updating Q-values, the new \(\varepsilon\)-greedy policy gives \(1 - \varepsilon + \varepsilon/|A|\) to the greedy action and \(\varepsilon/|A|\) to each non-greedy action. Make sure probabilities sum to 1.

8. Off-Policy Monte Carlo

On-policy learns the value of the policy it is following (which must be exploratory, hence suboptimal). Off-policy separates the two concerns:

PolicyRoleCharacter
Behavior policy \(b\)Generates episodes (collects data)Exploratory (e.g., \(\varepsilon\)-greedy or uniform)
Target policy \(\pi\)Policy being evaluated/improvedCan be greedy (deterministic, optimal)
Coverage Assumption

For off-policy to work, we need \(b(a|s) > 0\) whenever \(\pi(a|s) > 0\). That is, the behavior policy must sometimes take every action the target would take.

The challenge: episodes come from \(b\), but we want to estimate values under \(\pi\). The solution is importance sampling.

9. Importance Sampling

The importance sampling ratio corrects for the difference between the behavior and target policies. For a trajectory from time \(t\) to termination at \(T\):

\[ \rho_{t:T-1} = \prod_{k=t}^{T-1} \frac{\pi(A_k \mid S_k)}{b(A_k \mid S_k)} \]

This ratio tells us: "How much more (or less) likely would this trajectory have been under \(\pi\) compared to \(b\)?"

Ordinary Importance Sampling

\[ V(s) = \frac{\sum_{i=1}^{n} \rho_i \, G_i}{n} \]

Unbiased but can have very high variance (ratios can be huge).

Weighted Importance Sampling

\[ V(s) = \frac{\sum_{i=1}^{n} \rho_i \, G_i}{\sum_{i=1}^{n} \rho_i} \]

Biased (for finite \(n\)) but much lower variance. Preferred in practice.

Critical Insight

If the target policy \(\pi\) would never take some action that the behavior policy took (i.e., \(\pi(a|s) = 0\) for some action in the trajectory), the entire ratio \(\rho = 0\). That episode gets zero weight — it is irrelevant to the target policy's value.

Worked Exam Question: First-Visit MC Value

EC-3, Question 2c 1 Mark

The robot executes policy \(\pi\) (Continue at M, Recharge at L) and generates the following episode with \(\gamma = 0.9\):

Episode: \((M, \text{Continue}, +1, M) \to (M, \text{Continue}, +1, L) \to (L, \text{Recharge}, -3, M) \to (M, \text{Continue}, +1, \text{Terminal})\)

Using First-Visit Monte Carlo, calculate the estimated value \(V(M)\) from this single episode. Show the return calculation with proper discounting.

Solution

Step 1: Identify first visit of M

State M appears at \(t=0\), \(t=1\), and \(t=3\). First visit is at \(t=0\).

Step 2: List rewards after t=0

Rewards: \(R_1 = +1,\ R_2 = +1,\ R_3 = -3,\ R_4 = +1\)

Step 3: Compute return from t=0

\[ G_0 = R_1 + \gamma R_2 + \gamma^2 R_3 + \gamma^3 R_4 \] \[ = 1 + 0.9(1) + 0.81(-3) + 0.729(1) \] \[ = 1 + 0.9 - 2.43 + 0.729 \] \[ = 0.199 \]

Answer

\(V(M) \approx 0.199\) from this single episode.

Try It Yourself: What is V(L) from this same episode?

First (and only) visit to L is at \(t=2\). Rewards after: \(R_3 = -3,\ R_4 = +1\).

\(G_2 = R_3 + \gamma R_4 = -3 + 0.9(1) = -3 + 0.9 = -2.1\)

So \(V(L) = -2.1\) from this episode.

Worked Exam Question: Importance Sampling

EC-3, Question 2d 3 Marks

The robot uses exploratory behaviour policy \(b\): \(b(\text{Continue}|M) = 0.7\), \(b(\text{Recharge}|M) = 0.3\). Target policy \(\pi\) is deterministic: \(\pi(\text{Continue}|M) = 1.0\).

Episode under \(b\) with \(\gamma = 0.9\):

\((M, \text{Continue}, +1, M) \to (M, \text{Recharge}, -2, H) \to (H, \text{Continue}, +2, \text{Terminal})\)

Calculate:

d-1) The importance sampling ratio \(\rho\) from first occurrence of M until termination.

d-2) The return \(G\) from state M.

d-3) Explain how this ratio would be used in weighted importance sampling to update \(V(M)\).

Solution: Part d-1 — Importance Sampling Ratio

Identify the trajectory from first visit of M to terminal

First visit of M at \(t=0\). Actions taken from \(t=0\) onward: Continue (at M), Recharge (at M), Continue (at H).

Compute the ratio

\[ \rho_{0:T-1} = \frac{\pi(\text{Continue}|M)}{b(\text{Continue}|M)} \times \frac{\pi(\text{Recharge}|M)}{b(\text{Recharge}|M)} \times \frac{\pi(\text{Continue}|H)}{b(\text{Continue}|H)} \]

Now, \(\pi(\text{Recharge}|M) = 0\) because the target policy never recharges from M. Therefore:

\[ \rho = \frac{1.0}{0.7} \times \frac{0}{0.3} \times \ldots = 0 \]
Key Insight

\(\rho = 0\) because the behavior policy took an action (Recharge at M) that the target policy would never take. This entire episode is irrelevant for estimating \(V(M)\) under \(\pi\).

Solution: Part d-2 — Return from M

Compute G backward

\[ G_2 = R_3 = +2 \quad (\text{last step before terminal}) \] \[ G_1 = R_2 + \gamma G_2 = -2 + 0.9(2) = -0.2 \] \[ G_0 = R_1 + \gamma G_1 = 1 + 0.9(-0.2) = 0.82 \]

Return from state M: \(G = 0.82\).

Solution: Part d-3 — Weighted Importance Sampling

Weighted IS formula

\[ V(M) = \frac{\sum_{i=1}^{n} \rho_i \, G_i}{\sum_{i=1}^{n} \rho_i} \]

Since \(\rho = 0\) for this episode, it contributes zero to both numerator and denominator. This episode is effectively discarded — the behavior policy followed a path the target policy would never take, so it provides no information about what \(\pi\) would experience from M.

Only episodes where \(b\) happens to follow the same actions \(\pi\) would choose (or at least actions with \(\pi(a|s) > 0\)) will have \(\rho > 0\) and contribute to the estimate.

Try It Yourself: What if the episode were (M, Continue, +1, M) then (M, Continue, +3, Terminal)?

Both actions are Continue at M, and \(\pi(\text{Continue}|M) = 1.0\).

\(\rho = \frac{1.0}{0.7} \times \frac{1.0}{0.7} = \frac{1}{0.49} \approx 2.04\)

\(G_0 = 1 + 0.9(3) = 3.7\)

Weighted contribution: \(\rho \cdot G = 2.04 \times 3.7 = 7.55\), weight = 2.04.

The ratio > 1 because \(\pi\) always picks Continue (probability 1.0) but \(b\) only picks it with probability 0.7 — so trajectories where \(b\) happened to always Continue are up-weighted to reflect that \(\pi\) would do this every time.

Worked Exam Question: On-Policy MC Control

Chatbot with states \(s_0\) (engaged), \(s_1\) (disengaged). Actions: \(a_1\) (open question), \(a_2\) (recommend resource), \(a_3\) (fun fact).

Current Q-values:

\(Q(s_1, a_1)=0,\ Q(s_1, a_2)=0.5,\ Q(s_1, a_3)=1\)

With \(\varepsilon = 0.3\) (for 3 actions, \(\varepsilon/|A| = 0.1\) per action):

For state \(s_0\)

Greedy action: \(a_1\) (has highest Q = 3.408)

Try It Yourself: What if we used Every-Visit MC for \((s_1, a_2)\)?

Every-visit would use both visits to \((s_1, a_2)\) — at \(t=2\) and \(t=3\).

Returns: \(G_2 = 2.2\) and \(G_3 = -1.0\).

\(Q(s_1, a_2) = \frac{2.2 + (-1.0)}{2} = 0.6\)

The greedy action at \(s_1\) would then be \(a_3\) (Q=1 > 0.6), changing the policy!

Formula Cards — MEMORISE (Closed Book)

1. Return (Backward Recursion)

\[ G_t = R_{t+1} + \gamma \, G_{t+1} \quad \text{with } G_T = 0 \]
Start from terminal and work backward. Each step: immediate reward + discounted future.

2. First-Visit MC Update (Incremental Form)

\[ V(s) \leftarrow V(s) + \frac{1}{N(s)}\bigl[G_t - V(s)\bigr] \]
\(N(s)\) = number of first-visits to \(s\) so far. Moves \(V(s)\) toward the latest return, weighted by 1/count.

3. Importance Sampling Ratio

\[ \rho_{t:T-1} = \prod_{k=t}^{T-1} \frac{\pi(A_k \mid S_k)}{b(A_k \mid S_k)} \]
Product of policy ratios for every action from time \(t\) to end. If any \(\pi(A_k|S_k) = 0\), entire ratio = 0.

4. Weighted Importance Sampling

\[ V(s) = \frac{\sum_{i=1}^{n} \rho_i \, G_i}{\sum_{i=1}^{n} \rho_i} \]
Self-normalizing: episodes where \(b\) diverges from \(\pi\) get low/zero weight. Lower variance than ordinary IS.

5. \(\varepsilon\)-Greedy Policy (for |A| actions)

\[ \pi(a|s) = \begin{cases} 1 - \varepsilon + \frac{\varepsilon}{|\mathcal{A}|} & a = a^* \\ \frac{\varepsilon}{|\mathcal{A}|} & a \neq a^* \end{cases} \]
Greedy action \(a^* = \arg\max_a Q(s,a)\). All probabilities sum to 1. Ensures exploration.
