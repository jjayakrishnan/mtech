The k-Armed Bandit Problem

Imagine you walk into a casino and face a row of k slot machines (bandits). Each pull of a lever gives you a random reward drawn from some unknown distribution. Your goal: maximize your total reward over many pulls.

Analogy

You are a doctor choosing between k experimental treatments for a patient. Each treatment has an unknown success rate. You want to discover and use the best treatment, but every trial on a worse treatment is a lost opportunity.

Three defining features

  1. Evaluative feedback — you learn how good your chosen action was, but not whether it was the best possible action (no "correct answer" is revealed).
  2. One situation (no states) — unlike a full MDP, the bandit has no state transitions. You face the same decision repeatedly.
  3. k actions — at each time step, you choose one of k possible actions (arms).
Formal Setup

At each time step \(t = 1, 2, 3, \ldots\), the agent selects action \(A_t \in \{1, 2, \ldots, k\}\) and receives reward \(R_t\) whose distribution depends on the action selected.

True vs Estimated Action Values

Each action has a true value — the expected (average) reward you would get if you always picked that action:

\[ q_*(a) \;=\; \mathbb{E}[R_t \mid A_t = a] \]

This is what we want to know but don't. Instead, we maintain estimates:

\[ Q_t(a) \;\approx\; q_*(a) \]
Plain English

\(q_*(a)\) is the true average rating of a restaurant you have never been to. \(Q_t(a)\) is your personal estimate after eating there a few times. The more visits, the closer your estimate gets to the truth.

If we knew \(q_*(a)\) for every action, the problem would be trivial — always pick the action with the highest true value. The challenge is that we only learn through experience.

Sample-Average Estimation

The simplest way to estimate an action's value: just average all the rewards you have received from it so far.

\[ Q_t(a) \;=\; \frac{\text{sum of rewards when } a \text{ was taken before time } t}{\text{number of times } a \text{ was taken before time } t} \;=\; \frac{\sum_{i=1}^{t-1} R_i \cdot \mathbf{1}_{A_i=a}}{\sum_{i=1}^{t-1} \mathbf{1}_{A_i=a}} \]

By the law of large numbers, as the count goes to infinity, \(Q_t(a) \to q_*(a)\). The estimate converges to the truth — but only if you keep sampling that action.

Exam tip

If they ask "what method guarantees convergence to \(q_*(a)\)?" — it is sample averaging, provided every action is selected infinitely often.

The Repeated Estimation Cycle

At every time step, the agent performs a four-stage loop:

StageWhat happens
1. Maintain estimatesKeep running averages \(Q_t(a)\) for each of the \(k\) actions.
2. Select actionUse a policy (greedy, epsilon-greedy, etc.) to choose \(A_t\).
3. Observe rewardPull the lever, receive reward \(R_t\).
4. Update estimateIncorporate the new reward into \(Q_{t+1}(A_t)\).

This loop repeats indefinitely. The key design question is stage 2: how do you select?

Greedy Action Selection

The greedy strategy always picks the action with the highest current estimate:

\[ A_t \;=\; \arg\max_a \; Q_t(a) \]

This is pure exploitation — it uses what you currently believe is best, without ever testing alternatives.

The problem with pure greedy

If your early estimates are wrong (and they usually are, because you have few samples), greedy locks in on a suboptimal action and never discovers the truly best one. It settles too early.

Analogy

You try three restaurants once each. Restaurant B happened to serve you a great meal on a lucky night. Greedy says: "B is best, eat there forever." You never discover that Restaurant A is actually better on average.

Epsilon-Greedy Selection

The epsilon-greedy strategy introduces controlled randomness to ensure exploration:

  • With probability \(1 - \varepsilon\): select the greedy action (exploit).
  • With probability \(\varepsilon\): select a random action uniformly from all \(k\) actions (explore).

Note that "random from all \(k\) actions" includes the greedy action itself. So the greedy action can be selected in both branches.

Total probability of selecting the greedy action

\[ P(\text{greedy action chosen}) = (1 - \varepsilon) + \frac{\varepsilon}{k} \]

The first term is from the exploit branch. The second is the chance of randomly landing on the greedy action during the explore branch.

Why this works

Every action is tried infinitely often (since every action has at least probability \(\varepsilon / k > 0\) of being selected). So all estimates converge to their true values, and eventually the agent does exploit the truly best action most of the time.

Numerical Example: 4-Armed Bandit, epsilon = 0.20

Problem: What is the total probability that the greedy action is selected?

Given: \(k = 4\) arms, \(\varepsilon = 0.20\).

Step 1: Probability from the exploit branch: \(1 - \varepsilon = 1 - 0.20 = 0.80\).
Step 2: Probability from the explore branch: \(\varepsilon / k = 0.20 / 4 = 0.05\).
Step 3: Total probability: \(0.80 + 0.05 = \mathbf{0.85}\).

The greedy action is chosen 85% of the time. Each non-greedy action is chosen with probability \(\varepsilon / k = 0.05\), i.e., 5% each.

Try it yourself

A 10-armed bandit uses \(\varepsilon = 0.1\). What is the probability that the greedy action is selected on any given step? What is the probability that a specific non-greedy action is selected?

Greedy action probability: \((1 - 0.1) + 0.1/10 = 0.9 + 0.01 = 0.91\)

Any specific non-greedy action: \(\varepsilon / k = 0.1 / 10 = 0.01\) (1% each)

Sanity check: \(0.91 + 9 \times 0.01 = 0.91 + 0.09 = 1.0\) — probabilities sum to 1.

Try it yourself

With \(k = 5\) and \(\varepsilon = 0.4\), what fraction of the time is the agent exploiting (choosing greedily from the exploit branch)? What is the total probability of the greedy action being chosen?

Exploiting fraction: \(1 - \varepsilon = 0.6\) (60% of the time the agent deliberately exploits).

Total greedy probability: \(0.6 + 0.4/5 = 0.6 + 0.08 = 0.68\).

With a high epsilon like 0.4, the agent explores a lot — each non-greedy action is selected with probability \(0.4/5 = 0.08\) (8%).

The 10-Armed Testbed Experiment

Sutton and Barto's classic experiment: 2000 randomly generated 10-armed bandit problems, each run for 1000 steps. Three strategies are compared:

StrategyBehaviorOutcome
Greedy (\(\varepsilon = 0\))Pure exploitationFinds an OK action fast, but gets stuck. Only ~33% optimal action rate.
\(\varepsilon = 0.01\)Mostly exploit, rarely exploreSlow but steady improvement. Eventually reaches ~91% optimal action rate but takes many steps.
\(\varepsilon = 0.1\)More explorationLearns faster initially. Reaches ~80% optimal action rate quickly but plateaus lower because it keeps exploring.

Two performance curves

Average reward over time: Greedy plateaus low. Epsilon=0.1 rises fastest early, but epsilon=0.01 eventually overtakes it in the long run.

% optimal action over time: Greedy flatlines around 33%. Epsilon=0.1 quickly reaches ~80%. Epsilon=0.01 slowly climbs toward ~91%.

Key takeaway for exams

Higher epsilon means faster learning but lower asymptotic performance (you never stop exploring). Lower epsilon means slower learning but higher asymptotic performance. Greedy gets stuck because early misleading estimates are never corrected.

Why greedy gets only ~33% optimal action

With 10 arms and initial estimates of 0, the first reward received from any arm (even if it was just noise) makes that arm look best. Greedy locks onto whichever arm happened to give a positive reward first. Since the probability that the first "lucky" arm is actually the best is roughly 1/10, and greedy sometimes finds it, the average hovers around 1/3 (a bit above 1/10 because greedy does improve slightly from repeated pulls of its chosen arm).

Key Insight: Exploration Corrects Early Mistakes

The fundamental lesson of this chapter:

The core insight

Early estimates are unreliable because they are based on very few samples. Exploration forces the agent to revisit actions it might have dismissed, allowing it to discover that its first impressions were wrong. Without exploration, the agent is a prisoner of its initial lucky (or unlucky) experiences.

This is the exploration-exploitation dilemma — the central tension of all bandit and RL problems. Pure exploitation settles too early. Pure exploration wastes time on bad actions. Epsilon-greedy is a simple compromise: mostly exploit, but occasionally explore.

Action-Value Methods: Core Formulas

True action value
\( q_*(a) = \mathbb{E}[R_t \mid A_t = a] \)
Sample-average estimate
\( Q_t(a) = \dfrac{\sum_{i=1}^{t-1} R_i \cdot \mathbf{1}_{A_i=a}}{\sum_{i=1}^{t-1} \mathbf{1}_{A_i=a}} \)
Greedy selection
\( A_t = \arg\max_a \; Q_t(a) \)
Epsilon-greedy probability of selecting greedy action
\( P(\text{greedy}) = (1 - \varepsilon) + \dfrac{\varepsilon}{k} \)
Epsilon-greedy probability of a specific non-greedy action
\( P(\text{non-greedy}_i) = \dfrac{\varepsilon}{k} \)

Sanity check: \((1-\varepsilon) + \varepsilon/k + (k-1)\cdot\varepsilon/k = 1-\varepsilon+\varepsilon = 1\).

Quick-Fire Quiz

Test yourself — click an option to check your answer.

Correct! (1 - 0.12) + 0.12/6 = 0.88 + 0.02 = 0.90
Not quite. Use: (1 - epsilon) + epsilon/k = (1-0.12) + 0.12/6 = 0.88 + 0.02 = 0.90
2. What is the main weakness of pure greedy action selection?
Correct! Greedy never corrects its early mistakes because it stops exploring alternatives.
No. Greedy's weakness is that it locks in on whatever looks best early, even if those estimates are noisy and wrong.
3. In the 10-armed testbed, which strategy reaches the highest % optimal action in the long run?
Correct! Lower epsilon means less ongoing exploration, so asymptotically more time is spent on the best action (~91%).
epsilon=0.01 wins asymptotically (~91%) because it explores enough to find the best action but spends most of its time exploiting it.
4. What does the sample-average method converge to as the number of samples goes to infinity?
Correct! By the law of large numbers, the sample average converges to the expected value.
By the law of large numbers, the average of IID samples converges to the expected value, which is q*(a).

S&B Ex 2.2 — Bandit Action Trace [5 marks]

In a 4-action bandit (\(k=4\)), ε-greedy is used with \(Q_1(a)=0\) for all actions. The following action–reward sequence is observed:

\(t\)\(A_t\)\(R_t\)
11−1
221
32−2
422
530

On which time steps was the ε random draw definitely taken (an exploratory action must have occurred)?

Solution — trace the Q estimates:

  • After t=1: \(Q(1)=-1\), all others \(=0\). At t=2, greedy ∈ {2,3,4}; action 2 is consistent with greedy.
  • After t=2: \(Q(2)=1\). At t=3, greedy = action 2; action 2 is consistent with greedy.
  • After t=3: \(Q(2)=\frac{1+(-2)}{2}=-0.5\). At t=4, greedy ∈ {3,4} (both \(Q=0 > -0.5\)); choosing action 2 must have been exploratory — ε definitely fired.
  • After t=4: \(Q(2)=\frac{1-2+2}{3}=\frac{1}{3}\). At t=5, greedy = action 2; action 3 may be exploratory but is not certain.

Answer: Step 4 is the only step where ε must have fired.

▶  Key formula & step-by-step
Sample-average update (incremental form): $$ Q_{n+1}(a) = Q_n(a) + \frac{1}{n}\bigl[R_n - Q_n(a)\bigr] $$ \(Q_1(a)=0\) for all arms. Greedy = action with highest \(Q\).
  1. After t=1 (A₁=1, R₁=−1): \(Q(1)=-1\), others \(=0\). Greedy ∈ {2,3,4} — action 2 could be chosen greedily ✓
  2. After t=2 (A₂=2, R₂=+1): \(Q(2)=1\). Greedy = action 2 (highest Q). A₃=2 is greedy ✓
  3. After t=3 (A₃=2, R₃=−2): \(Q(2)=\tfrac{1+(-2)}{2}=-0.5\). Now Q(3)=Q(4)=0 > −0.5. Greedy ∈ {3,4}. A₄=2 is not greedy → ε must have fired here ✗ (definite exploration)
  4. After t=4 (A₄=2, R₄=+2): \(Q(2)=\tfrac{1-2+2}{3}=\tfrac{1}{3}\). Greedy = action 2 again. A₅=3 could be greedy if Q(3)>Q(2)… but Q(3)=0 so A₅=3 is exploratory too (possible, not certain).
  5. Answer: Step t=4 is the only certain exploration — the only time the action taken is definitively not greedy given the Q-values at that moment.
▶  Key formula & step-by-step
Constant step-size as exponential recency-weighting: $$ Q_{n+1} = (1-\alpha)^n Q_1 + \sum_{k=1}^{n} \alpha(1-\alpha)^{n-k} R_k $$ Weight of reward \(R_k\) decays as \((1-\alpha)^{n-k}\) — older rewards matter less.
  1. Start from the update: \(Q_{n+1} = (1-\alpha)Q_n + \alpha R_n\). Substitute \(Q_n = (1-\alpha)Q_{n-1} + \alpha R_{n-1}\).
  2. Expand one step: \(Q_{n+1} = (1-\alpha)^2 Q_{n-1} + \alpha(1-\alpha)R_{n-1} + \alpha R_n\).
  3. Keep substituting back to \(Q_1\): The pattern telescopes to the closed-form above.
  4. Why this suits non-stationarity: The weight of \(R_k\) decays exponentially with age. Recent rewards dominate, old rewards fade — perfect when the true action values drift over time.
  5. Contrast with \(1/n\): The sample average weights all rewards equally. It converges in stationary environments but cannot track a moving target.
▶  Key formula & step-by-step
UCB action selection: $$ A_t = \arg\max_a \left[ Q_t(a) + c\sqrt{\frac{\ln t}{N_t(a)}} \right] $$ When \(N_t(a)=0\) the bonus is \(+\infty\) → arm is selected immediately.
  1. Steps 1–10 (k=10 arms): Every arm starts with \(N(a)=0\), so the UCB bonus is infinite for all. Each arm must be tried once. After step 10 every arm has \(N(a)=1\).
  2. Step 11: All bonuses equal \(c\sqrt{\ln 11}\). Selection is by Q-value — the arm with the single highest sampled reward. Likely a good arm → high reward → spike.
  3. Steps 12 onward: The chosen arm now has \(N=2\) → smaller bonus. Other arms with \(N=1\) have larger bonuses → some sub-optimal arm gets selected → average reward dips back down.
  4. Takeaway: The spike at step 11 is structural, not noise — it is the moment UCB first fully exploits after its mandatory exploration phase ends.
S&B Ex 2.4 — Constant Step-Size Weighting [5 marks]

Replace the sample-average step-size \(\alpha_n = \frac{1}{n}\) with a constant \(\alpha \in (0,1]\). The update is:

$$ Q_{n+1} = Q_n + \alpha\bigl[R_n - Q_n\bigr] $$

Expand this recurrence to show that \(Q_{n+1}\) is an exponentially weighted average of past rewards. What weight does reward \(R_k\) receive?

Solution — expand the recurrence:

Write \(Q_{n+1} = (1-\alpha)Q_n + \alpha R_n\). Substituting recursively:

$$ Q_{n+1} = (1-\alpha)^n Q_1 + \sum_{k=1}^{n} \alpha(1-\alpha)^{n-k} R_k $$

Reward \(R_k\) receives weight \(\alpha(1-\alpha)^{n-k}\). Because \(0 < 1-\alpha < 1\), older rewards are down-weighted exponentially — this is a recency-weighted (“exponential recency-weighted”) average, suitable for non-stationary problems where recent rewards matter more.

S&B Ex 2.8 — UCB Spike on Step 11 [4 marks]

In Figure 2.4 (UCB action selection with \(k=10\) arms), there is a sharp spike in average reward on step 11. Explain why reward jumps up on step 11 and then immediately drops back down.

Solution:

UCB selects \(A_t = \arg\max_a \left[Q_t(a) + c\sqrt{\frac{\ln t}{N_t(a)}}\right]\). Any arm with \(N_t(a)=0\) has an infinite bonus, so each arm must be tried at least once first.

With 10 arms, steps 1–10 force each arm to be tried exactly once.

Step 11: All arms have been tried once; the confidence bonus is equal for all, so UCB now picks based on actual estimated value — it selects the best-looking arm so far, which is likely genuinely good, causing the reward spike.

Immediately after: The selected arm’s count \(N_{11}(a)\) increases, shrinking its confidence bonus. Other arms now score higher and get selected — including suboptimal ones — pulling the average back down until estimates stabilise.
