Three Algorithms — Know Which Is Which

The exam mixes these up deliberately. Before any calculation, identify which one the question is asking for.

Policy Evaluation

Given a fixed policy π, compute Vπ(s) for all states. Uses the Bellman expectation equation.

Value Iteration

Find the optimal value function V*(s) directly, without fixing a policy. Uses the Bellman optimality equation. Takes a max over actions.

Policy Improvement

Given Vπ(s), compute Q(s,a) for each action, then greedify — pick whichever action has the highest Q.

Exam tell — which equation to use

If the question says "synchronous Value Iteration" → Bellman optimality with a max. If it says "policy π where π(action|state)=1" → Bellman expectation, no max needed. EC-2 Q1c says "Value Iteration" → use max. EC-3 Q2a says "follows policy π where π(Continue|M)=1" → use expectation.

Policy Evaluation — Bellman Expectation Equation

Used when the policy is already fixed and you want to know how good it is.

Bellman Expectation — for a fixed policy π
\[ V^\pi(s) = \sum_a \pi(a \mid s) \sum_{s'} p(s' \mid s, a) \bigl[r(s,a,s') + \gamma V^\pi(s')\bigr] \]
Plain English

For each action the policy might take (weighted by how often it takes it), look at every place you might land, take the reward plus discounted future value there, and average it all together.

Quick example — EC-3 Q2a (Warehouse Robot)

Policy π: always Continue at state M. So π(Continue|M) = 1, γ = 0.9, V(L)=V(M)=V(H)=1:

Step-by-step

Vπ(M) = π(Continue|M) × [p(M|M,Continue) × (r + γV(M)) + p(L|M,Continue) × (r + γV(L))]
           = 1.0 × [0.6 × (1 + 0.9×1) + 0.4 × (1 + 0.9×1)]
           = 0.6 × 1.9 + 0.4 × 1.9
           = 1.9

Value Iteration — Bellman Optimality Equation

Don't fix a policy. Instead, at each state, ask: what's the best I can do from here? Take the max over all actions.

Bellman Optimality — Value Iteration update rule
\[ V_{k+1}(s) = \max_a \sum_{s'} p(s' \mid s,a)\bigl[r(s,a,s') + \gamma \, V_k(s')\bigr] \]
What "synchronous" means

Synchronous Value Iteration: compute Vk+1(s) for all states simultaneously using the old values Vk. Don't update V(s) mid-sweep and use the new value for another state in the same sweep. The exam specifies "synchronous" — always use Vk, not Vk+1, on the right-hand side.

Most common exam mistake

Students use the updated V(s) for states computed earlier in the same iteration. Synchronous means: compute all Vk+1 using only Vk values. Write V0, V1, V2… as separate columns — never overwrite.

Policy Improvement

Once you have Vπ(s), compute the Q-value for each (state, action) pair, then choose the greedy action.

Q-value formula
\[ Q^\pi(s,a) = \sum_{s'} p(s' \mid s,a)\bigl[r(s,a,s') + \gamma V^\pi(s')\bigr] \]
Greedy policy improvement
\[ \pi'(s) = \arg\max_a \; Q^\pi(s,a) \]

If the new greedy action differs from the current policy's action → the policy has changed → continue improving. If no state changes → policy is already optimal.

Policy Improvement — EC-3 Q2b

Now the question changes the initial values: V0(L) = −10, V0(M) = 5, V0(H) = 8. Policy π₀ always Continues at M. Should it change?

Compute Qπ₀(M, Continue) and Qπ₀(M, Recharge)

Q(M, Continue) = 0.6×(1 + 0.9×5) + 0.4×(1 + 0.9×(−10))
                = 0.6×(1 + 4.5) + 0.4×(1 − 9)
                = 0.6×5.5 + 0.4×(−8)
                = 3.3 − 3.2 = 0.1

Q(M, Recharge) = 1.0×(−2 + 0.9×8)
                 = 1.0×(−2 + 7.2) = 5.2

Q(M, Recharge) = 5.2 > Q(M, Continue) = 0.1
→ Policy must change. New policy: π₁(Recharge | M) = 1.0
The original "always Continue at M" is suboptimal given these initial estimates.

How to Layout Your Exam Answer

The 4-mark calculation question penalises missing working. Use this exact structure every time.

Exam answer template

Step 1. State the formula: Vk+1(s) = maxa Σ p(s′|s,a)[r + γVk(s′)]

Step 2. State γ and V0 values explicitly.

Step 3. For each state, compute Q(s, a) for every action — show every multiplication.

Step 4. Take the max → write V1(s) and state which action achieves it.

Step 5. Summarise in a table: State | V₀ | V₁ | Best action.

Marks are in the working, not the final number

If your final V₁ is wrong but your Q calculations are correct and clearly shown, you still collect most marks. Never skip the intermediate steps. The examiner reads each line — not just the last one.

⚑ MEMORISE THIS — Closed Book

Bellman Expectation (Policy Evaluation) — use when policy π is given
V^π(s) = Σ_a π(a|s) Σ_s' p(s'|s,a) [ r + γ V^π(s') ]
Bellman Optimality (Value Iteration) — use when finding V*
V_{k+1}(s) = max_a Σ_s' p(s'|s,a) [ r + γ V_k(s') ]
Q-value (for Policy Improvement)
Q^π(s,a) = Σ_s' p(s'|s,a) [ r + γ V^π(s') ]
Policy Improvement (greedify)
π'(s) = argmax_a Q^π(s,a)
Three things to never forget

① Synchronous = use old V_k throughout the sweep — never mix new values in the same iteration.

② Value Iteration takes a max; Policy Evaluation takes a weighted sum (no max).

③ Show every Q(s,a) calculation before writing V(s) — marks live in the working.

Exam-Style Quiz

Closed-book conditions. No scrolling up during the quiz.

Q1  [1 mark]   A question says "use synchronous Value Iteration for 1 iteration." Which formula do you use and what makes it different from Policy Evaluation?
Correct. Key difference: max (VI) vs weighted average (PE). And synchronous means use V_k — not V_{k+1} — on the right side for every state in the same sweep.
Q2  [2 marks]   Using the warehouse robot MDP with γ=0.9 and V⁰=(L=1, M=1, H=1): what is V₁(L)?
Correct. Even though both options are negative, Recharge (−2.1) beats Continue (−4.1). The max of two bad options is still the less-bad one.
Q3  [1 mark]   For the Policy Improvement step at state M (with V⁰(L)=−10, V⁰(M)=5, V⁰(H)=8): what does Q(M, Recharge) equal?
Correct. 1.0 × (r + γV(H)) = 1.0 × (−2 + 0.9×8) = 5.2. The high V(H)=8 makes Recharging attractive despite the −2 penalty.
Q4  [1 mark]   What does "synchronous" mean in synchronous Value Iteration?
Correct. Synchronous = one snapshot of old values feeds the entire sweep. The alternative (asynchronous) updates states in-place one at a time, which can converge faster but is harder to compute by hand.
Q5  [1 mark]   After computing Q(M, Continue)=0.1 and Q(M, Recharge)=5.2, what is the updated policy π₁(a|M)?
Correct. Greedy policy improvement = pick the argmax action deterministically. Never split proportionally — that's a different (softmax) policy type.

+12 Analogy — Value Iteration

Imagine a GPS app recalculating your route. Each road junction is a state. At each junction you pick a direction (action). The GPS starts with a rough guess of travel times, then repeatedly refines: "From this junction, taking the fastest road, how long will I really take?" Each iteration is one recalculation sweep across all junctions. After enough sweeps, the estimated times stop changing — that's convergence, and the resulting route is optimal.

Try It Yourself

S&B Ex 4.10 — Action-Value Update for Value Iteration [5 marks]

Value iteration for state values uses:

$$ V_{k+1}(s) = \max_a \sum_{s',r} p(s',r|s,a)\bigl[r + \gamma V_k(s')\bigr] $$

Write the analogous value iteration update for action values, \(q_{k+1}(s,a)\). Why does this form allow policy extraction without needing the model dynamics again?

Solution:

$$ q_{k+1}(s,a) = \sum_{s',r} p(s',r|s,a)\!\left[r + \gamma \max_{a'} q_k(s',a')\right] $$

Replace \(V_k(s')\) with \(\max_{a'} q_k(s',a')\) — the best action-value at the next state.

Policy extraction without a model: Once \(q_*\) is known, the optimal action is simply \(\pi_*(s) = \arg\max_a q_*(s,a)\) — no transition model \(p\) needed. With state values \(V_*\), you would still need \(p\) to do a one-step lookahead.

S&B Ex 4.1 — Gridworld Action Values [5 marks]

In the 4×4 gridworld (Example 4.1): states 1–14 are non-terminal, corners 0 and 15 are terminal. All actions yield reward \(-1\). Under the equiprobable random policy, the full solution gives \(v_\pi(11) = -14\) and \(v_\pi(15) = 0\).

(a) Compute \(q_\pi(11,\,\text{down})\).
(b) Compute \(q_\pi(7,\,\text{down})\), given \(v_\pi(11) = -14\).

Solution: Use \(q_\pi(s,a) = \sum_{s',r} p(s',r|s,a)\bigl[r + v_\pi(s')\bigr]\). Actions are deterministic (prob = 1 for one next state).

(a) State 11 + down → state 15 (terminal), reward \(-1\), \(v(15)=0\):

$$ q_\pi(11,\text{down}) = 1 \times \bigl[-1 + 0\bigr] = \mathbf{-1} $$

(b) State 7 + down → state 11, reward \(-1\), \(v(11)=-14\):

$$ q_\pi(7,\text{down}) = 1 \times \bigl[-1 + (-14)\bigr] = \mathbf{-15} $$
S&B Ex 4.8 — Gambler’s Problem: Capital 50 vs 51 [5 marks]

In the Gambler’s Problem (coin with \(p_h = 0.4\), goal = reach £100), the optimal policy bets everything at capital £50 but only a small amount at capital £51. Explain why.

Solution:

At £50: a single bet of £50 reaches £100 in one step if heads — the shortest possible path. With a disadvantaged coin (\(p_h < 0.5\)), taking the shortest path to the goal is optimal.

At £51: betting £51 would overshoot (£102 > £100 is not allowed). The optimal bet is small — e.g. £1 — because losing brings you to £50, re-enabling the big leveraged bet. The policy exploits arithmetic structure: multiples of 50 (and 25, 12, …) allow direct shots at £100. Stepping from £50 to £51 removes that direct shot, so the bet collapses to a cautious “return to £50” strategy.
