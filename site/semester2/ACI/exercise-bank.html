CS3 Problem Formulation & Search

State space, actions, transition model, goal test, path cost. A* search traces.

Exercise 1 — Formulate Three Search Problems

For each scenario, define: State, Initial state, Actions, Goal test, Path cost.

  1. A delivery drone must plan a trip from warehouse A to warehouse B using GPS waypoints.
  2. A robot must navigate a maze from entrance to exit.
  3. A scheduling agent must assign patients to doctors in a clinic to minimise total wait time.
Reveal worked solution

(a) Delivery Drone

  • State: Current GPS coordinates (x, y) of the drone
  • Initial state: Warehouse A coordinates
  • Actions: Fly to adjacent waypoint (N, S, E, W, diagonal)
  • Goal test: Current location = Warehouse B coordinates
  • Path cost: Total distance flown (or energy consumed)

(b) Maze Navigation

  • State: (row, col) cell position in the maze grid
  • Initial state: Entrance cell
  • Actions: Move Up / Down / Left / Right (if not blocked by wall)
  • Goal test: Current cell = exit cell
  • Path cost: Number of steps taken (each move costs 1)

(c) Clinic Scheduling

  • State: Partial assignment of patients to doctors + time slots
  • Initial state: Unassigned list of patients, available doctors and slots
  • Actions: Assign one patient to one doctor at one time slot
  • Goal test: All patients assigned without conflicts
  • Path cost: Total expected wait time across all patients

Exercise 2 — Apply A* on a Graph

Graph edges (directed):

  • S → A (cost 1), S → C (cost 10)
  • A → B (cost 2), A → C (cost 1)
  • B → D (cost 5)
  • C → D (cost 3)
  • D → G (cost 2)

Heuristic values: \(h(S)=5,\ h(A)=3,\ h(B)=4,\ h(C)=2,\ h(D)=6,\ h(G)=0\)

Task: Run A* from S to G. Show the open list at each step. What is the optimal path and cost?

Reveal worked solution

A* uses \(f(n) = g(n) + h(n)\) where \(g\) = cost from start, \(h\) = heuristic estimate to goal.

How each step works

Step 1 — Expand S: \(g=0,\ h=5,\ f=5\). Generates A (\(g=1,f=4\)) and C (\(g=10,f=12\)). Open: {A:4, C:12}. Pick A.

Step 2 — Expand A: \(g=1,\ h=3,\ f=4\). Generates C via A (\(g=2,f=4\)) — cheaper than C via S (\(g=10\)), update C. Generates B (\(g=3,f=7\)). Open: {C:4, B:7}. Pick C.

Step 3 — Expand C (via A): \(g=2,\ h=2,\ f=4\). Generates D via C (\(g=5,f=11\)). Open: {B:7, D:11}. Pick B.

Step 4 — Expand B: \(g=3,\ h=4,\ f=7\). Generates D via B (\(g=8,f=14\)) — worse than D via C (\(g=5\)), discard. Open: {D via C:11}. Pick D.

Step 5 — Expand D (via C): \(g=5,\ h=6,\ f=11\). Generates G (\(g=7,f=7\)). Open: {G:7}. Pick G.

Step 6 — Expand G: Goal! Path = S → A → C → D → G, cost = 7.

3. Admissibility & Consistency

Admissibility check (h(n) ≤ true cost to goal):

  • h(B)=19: true cost B→A→D→H→G=19 ✓ (tight); B→C→D→H→G=18, so h(B)=19 > 18 — inadmissible!

h(B)=19 overestimates (actual optimal from B is 18). The heuristic is not admissible.

Consistency: Would require \(h(n) \le c(n,n') + h(n')\) for every edge. Since h(B)=19 and B→C has cost 5 with h(C)=13: 19 ≤ 5+13=18 is false — not consistent either.

Exercise 2 — Warehouse Robot: State Representation & Relaxations

A robot in a warehouse must collect items and deliver them. The warehouse is a grid; some cells contain items, some are blocked.

  1. Define a state representation for this problem.
  2. Define 4 relaxed problems and explain what admissible heuristic each relaxation gives. Which relaxations are more or less admissible?
Reveal worked solution

A. State Representation

State = (robot position, set of uncollected items, set of delivered items)

  • Robot position: (row, col) on the grid
  • Uncollected items: subset of {item locations}
  • Delivered items: subset of {item IDs}

Initial state: Robot at start cell, all items uncollected, no deliveries made.

Goal state: All items delivered (uncollected set = ∅, robot at depot).

B. Four Relaxations

#RelaxationHeuristic h(n)Admissible?
R1 Ignore blocked cells (robot can pass through walls) Manhattan distance to nearest item then to depot Yes — removing constraints only reduces cost
R2 Ignore item weight (robot can carry all items at once) Distance from robot → nearest item + item → depot Yes — fewer trips needed in relaxed problem
R3 Ignore all constraints (teleport to each item) Number of remaining items (each "costs" 1) Yes — trivial lower bound
R4 Robot must still avoid walls but can carry all items Shortest path to collect all items + return Yes — combines path cost with item count
Key rule: a relaxed problem heuristic is admissible iff the relaxed problem's optimal cost ≤ original problem's optimal cost. Removing constraints always yields a heuristic that is admissible.

Sample Exam Problem — Sliding-Tile Minimax

Two players (A and B) play on a board with 3A tiles, 3B tiles, and 2 empty spaces. Players alternate moving one of their tiles into an adjacent empty space. The utility function (from A's perspective) is:

$$U = 3 \times (\text{MAX\_wc} - \text{MIN\_wc}) + 2 \times (\text{MAX\_ap} - \text{MIN\_ap})$$

Where: wc = number of winning configurations, ap = number of advancing positions.

  1. Draw the game tree (depth 3: A moves, B moves, A moves).
  2. Compute utility at each leaf node.
  3. Apply Minimax and determine A's optimal first move.
Reveal worked solution (methodology)

This problem tests whether you understand the process of Minimax, not just the formula. The exact tree depends on the initial board state (which the exam would provide). Here is the methodology:

Step 1 — Build the game tree

  • Root = initial board state. A moves first (MAX node).
  • For each of A's legal moves, create a child node (MIN node, B's turn).
  • For each of B's moves, create a child (MAX node, A's turn again).
  • At depth 3, stop and evaluate utility \(U\) at each leaf.

Step 2 — Evaluate leaves

For each leaf board state, count:

  • MAX\_wc: number of 3-in-a-row or winning patterns for A
  • MIN\_wc: same for B
  • MAX\_ap: A's tiles advancing toward the goal position
  • MIN\_ap: B's advancing positions

Then apply: \(U = 3(MAX\_wc - MIN\_wc) + 2(MAX\_ap - MIN\_ap)\).

A positive \(U\) favours player A; negative favours B.

Step 3 — Back-propagate

  • Depth-3 nodes (A's turn, MAX): propagate upward the maximum leaf value.
  • Depth-2 nodes (B's turn, MIN): propagate upward the minimum child value.
  • Depth-1 nodes (A's turn, MAX): A picks the child with the maximum value.
Exam tip: Show every intermediate MIN/MAX value, not just the final answer. Examiners stop marking at the first calculation error — show working clearly from leaves upward.

Key formula to memorise

$$U = 3 \times (\text{MAX\_wc} - \text{MIN\_wc}) + 2 \times (\text{MAX\_ap} - \text{MIN\_ap})$$

The coefficients (3, 2) signal that winning configurations matter 1.5× more than advancing positions.
