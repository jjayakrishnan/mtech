1. Search Problem Formulation

A search problem is the formal way we describe "getting from here to there." Think of Google Maps: you need a starting point, possible roads, and a destination. The computer formulates this identically.

Every search problem has six components:

ComponentWhat it meansGoogle Maps analogy
State SpaceAll possible configurations the world can be inEvery intersection in the city
Initial StateWhere the agent startsYour current location
ActionsWhat the agent can do in each stateTurn left, go straight, turn right
Transition ModelResult(s, a) → s' — what state results from action a in state sTaking Road X from Intersection A leads to Intersection B
Goal TestIs this state the destination?Have we arrived at the restaurant?
Path CostNumeric cost of the path taken (sum of step costs)Total driving time or distance

Key Insight

The state space is implicit — we do not enumerate every state upfront. Instead, we generate states on demand using the transition model. This is what makes search tractable even in enormous spaces (chess has ~$10^{47}$ states).

2. Uninformed Search (Recap)

Uninformed (or "blind") search strategies have no knowledge about how far away the goal is. They differ only in the order they expand nodes.

Breadth-First Search (BFS)

Expands the shallowest unexpanded node first. Uses a FIFO queue.

  • Complete? Yes (if branching factor is finite)
  • Optimal? Yes, for uniform step costs (otherwise no)
  • Time: $O(b^d)$ — exponential in depth $d$
  • Space: $O(b^d)$ — keeps all nodes in memory

Depth-First Search (DFS)

Expands the deepest unexpanded node first. Uses a LIFO stack.

  • Complete? No (can loop in infinite/cyclic spaces)
  • Optimal? No (may find a deep solution before a shallow one)
  • Time: $O(b^m)$ where $m$ = max depth (can be infinite)
  • Space: $O(bm)$ — linear! Only stores current path + siblings

Uniform-Cost Search (UCS)

Expands the node with the lowest path cost $g(n)$. Uses a priority queue ordered by $g$.

  • Complete? Yes (if every step cost > some $\epsilon > 0$)
  • Optimal? Yes — always finds the cheapest path
  • Reduces to BFS when all step costs are equal

Iterative Deepening (IDS)

Runs DFS with increasing depth limits: 0, 1, 2, 3, ... Combines BFS's completeness/optimality with DFS's memory efficiency.

  • Complete? Yes
  • Optimal? Yes (for uniform step costs)
  • Space: $O(bd)$ — like DFS
  • Overhead: Repeated work is small — most nodes are at the deepest level anyway

When to use what?

IDS is the default uninformed strategy when you don't know the solution depth and memory is limited. UCS is used when step costs differ. BFS is rarely practical for large spaces due to memory. DFS is only useful for constraint satisfaction or when all solutions are at the same depth.

3. Informed Search — Heuristics

An informed search strategy uses a heuristic function $h(n)$ that estimates the cost from node $n$ to the nearest goal. This extra "hint" guides the search toward the goal, potentially expanding far fewer nodes than blind search.

Analogy: Imagine you're lost in a city. Uninformed search tries every street systematically. Informed search is like knowing "the train station is roughly north" — you head north, skipping most wrong turns.

What makes a good heuristic?

  • It should be admissible (never overestimates)
  • It should be consistent (obeys triangle inequality)
  • It should be informative (higher values = fewer nodes expanded)
  • It should be cheap to compute (no point saving expansions if computing $h$ is expensive)

4. Greedy Best-First Search (GBFS)

GBFS expands the node that appears closest to the goal: it sorts the open list by $h(n)$ only, completely ignoring path cost $g(n)$.

  • Evaluation function: $f(n) = h(n)$
  • Complete? No — can get stuck in loops (e.g., A → B → A → B ...)
  • Optimal? No — may find a path that looks promising but is actually expensive
  • Time & Space: $O(b^m)$ worst case, but a good heuristic can reduce this dramatically

Why GBFS fails

Imagine a heuristic says city C is close to the goal, but the road A→C costs 100 while A→B→Goal costs 10 total. GBFS heads toward C because $h(C)$ is small, ignoring that it already spent 100 getting there. It is greedy — it never looks back at how much it has already paid.

5. A* Search

A* combines the best of UCS and GBFS. It expands nodes in order of:

$f(n) = g(n) + h(n)$

actual cost so far + estimated cost to goal

  • $g(n)$ = actual cost from start to node $n$ (like UCS)
  • $h(n)$ = estimated cost from $n$ to goal (like GBFS)
  • $f(n)$ = estimated total cost of the cheapest path through $n$

A* is optimal if $h$ is admissible, and complete if $h$ is consistent (or if the state space is finite). It expands the minimum number of nodes for any given heuristic — no other optimal algorithm using the same $h$ can do better.

Intuition: Why A* works

UCS is cautious (only trusts actual costs) but wastes time exploring in all directions. GBFS is bold (only trusts the heuristic) but can be fooled. A* balances both: it won't ignore expensive detours (because $g$ grows), and it won't explore unpromising directions (because $h$ guides it). The result: optimal and efficient.

6. Admissibility & Consistency

Admissibility

A heuristic $h(n)$ is admissible if it never overestimates the true cost to reach the goal:

$h(n) \leq h^*(n) \quad \forall n$

where $h^*(n)$ is the true cheapest cost from $n$ to a goal. Admissibility guarantees A* finds an optimal solution.

Analogy: Straight-line distance is admissible for road navigation because you can never drive less than the straight-line distance — roads only make it longer.

Consistency (Monotonicity)

A heuristic is consistent if for every node $n$ and every successor $n'$ generated by action $a$:

$h(n) \leq c(n, a, n') + h(n')$

This is the triangle inequality: the estimated cost from $n$ should not exceed the cost of one step plus the estimate from the next node. Consistency implies admissibility (but not vice versa).

Why it matters: With a consistent heuristic, once A* expands a node, it has found the optimal path to that node. This means we never need to re-open closed nodes.

7. Heuristic Construction

The Relaxed Problem Method

The most common technique: remove constraints from the original problem. The optimal solution cost of the relaxed problem is an admissible heuristic for the original (because removing constraints can only make the problem easier, never harder).

8-Puzzle Example

Original problem: A tile can move to an adjacent empty square.

Relaxation 1 — Misplaced Tiles ($h_1$): Remove the constraint that tiles must move to adjacent squares. A tile can teleport to its goal position. Cost = number of misplaced tiles.

Relaxation 2 — Manhattan Distance ($h_2$): Remove the constraint that other tiles block movement. A tile can slide through other tiles. Cost = sum of each tile's horizontal + vertical distance to its goal position.

Both are admissible because the real problem is harder (tiles do block each other, and they cannot teleport).

Heuristic Dominance

If $h_2(n) \geq h_1(n)$ for all nodes $n$, and both are admissible, then $h_2$ dominates $h_1$. A dominant heuristic is always better or equal — A* with $h_2$ expands fewer or equal nodes than A* with $h_1$.

For the 8-puzzle: Manhattan distance ($h_2$) dominates misplaced tiles ($h_1$) because $h_2(n) \geq h_1(n)$ always. In practice, $h_2$ expands dramatically fewer nodes.

Combining Heuristics

If you have multiple admissible heuristics $h_1, h_2, \ldots, h_k$, you can take their pointwise maximum: $h(n) = \max(h_1(n), h_2(n), \ldots, h_k(n))$. The result is still admissible and dominates each individual heuristic.

Detailed Expansion Explanation

Click to see step-by-step reasoning

Step 1 — Expand S (f=7):

  • Generate A: g(A) = g(S) + c(S,A) = 0 + 3 = 3, h(A) = 4, f(A) = 7
  • Generate B: g(B) = g(S) + c(S,B) = 0 + 4 = 4, h(B) = 6, f(B) = 10
  • Open list: {A: 7, B: 10}. Pick minimum f.

Step 2 — Expand A (f=7):

  • Generate G: g(G) = g(A) + c(A,G) = 3 + 2 = 5, h(G) = 0, f(G) = 5
  • Open list: {G: 5, B: 10}. Pick minimum f.

Step 3 — Expand G (f=5):

  • G is the goal — STOP.
  • Optimal path: S → A → G, cost = 5

Why not path S→B→C→A→G?
That path costs 4+5+1+2 = 12. A* never expanded B because f(B) = 10 > f(G) = 5. The heuristic correctly guided A* to the cheaper path.

Exam Tip

Always show: (1) which node you expand, (2) its g, h, f values, (3) the updated open list after expansion, (4) the closed list. If a node is already in the open list with a higher g, update it with the new lower g value. If a node is in the closed list (with consistent h), do not re-expand.

Closed-Book: Memorise This

Core Formulas

  • $f(n) = g(n) + h(n)$ — A* evaluation function
  • Admissible: $h(n) \leq h^*(n)$ — never overestimates true cost
  • Consistent: $h(n) \leq c(n, a, n') + h(n')$ — triangle inequality
  • Dominance: $h_2$ dominates $h_1$ if $h_2(n) \geq h_1(n)\ \forall n$ (both admissible)

Algorithm Properties (Exam Checklist)

  • BFS: Complete, optimal (uniform cost only), $O(b^d)$ space
  • DFS: NOT complete, NOT optimal, $O(bm)$ space
  • UCS: Complete, optimal (any costs), expands by min $g(n)$
  • IDS: Complete, optimal, $O(bd)$ space — best uninformed default
  • GBFS: NOT complete, NOT optimal, uses only $h(n)$
  • A*: Complete + optimal (admissible $h$), uses $g(n) + h(n)$

Heuristic Design Rules

  • Relaxed problem → admissible heuristic (remove constraints, solve easier problem)
  • Manhattan distance dominates misplaced tiles for 8-puzzle
  • $\max(h_1, h_2, \ldots, h_k)$ preserves admissibility and dominates all components
  • Consistent ⇒ Admissible (but not the reverse)

A* Trace Template (For Exams)

  1. Initialize: Open = {Start: f(S)}, Closed = {}
  2. Pick node with minimum $f$ from Open
  3. If it is the goal → STOP, report path and cost
  4. Else: move it to Closed, expand successors
  5. For each successor: compute $g' = g(\text{parent}) + c(\text{edge})$
  6. If successor not in Open or Closed: add to Open
  7. If successor in Open with higher g: update with new lower g
  8. Repeat from step 2
