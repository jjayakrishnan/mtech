2. Closed-Book Cheat Sheet

A* Evaluation

$$f(n) = g(n) + h(n)$$

\(g(n)\) = actual cost from start • \(h(n)\) = heuristic estimate to goal • \(f(n)\) = total estimated cost

Admissibility

$$h(n) \leq h^*(n)$$

Never overestimates true cost. Guarantees A* optimality. Key: relaxed problem ⟹ admissible h.

Consistency (Monotonicity)

$$h(n) \leq c(n, a, n') + h(n')$$

Triangle inequality. Consistent ⟹ admissible. A* with consistent h never re-expands nodes.

Alpha-Beta Prune Condition

$$\text{Prune when } \beta \leq \alpha$$

\(\alpha\) = MAX's best so far (starts \(-\infty\)) • \(\beta\) = MIN's best so far (starts \(+\infty\))

Alpha-Beta Complexity

$$O(b^{m/2}) \text{ (best case)}$$

Minimax: \(O(b^m)\). Perfect ordering halves depth. Doubles search depth for same computation.

MCTS UCB Formula

$$\text{UCB}(i) = v_i + C\sqrt{\frac{\ln N}{n_i}}$$

\(v_i\) = win rate • \(C\) = exploration constant • \(N\) = parent visits • \(n_i\) = node visits

ACO Pheromone Update

$$\tau_{ij} \leftarrow (1-\rho)\,\tau_{ij} + \Delta\tau_{ij}$$

\(\rho\) = evaporation rate • \(\Delta\tau_{ij}\) = quality of solutions using edge \((i,j)\)

NEAT Fitness Sharing

$$f'_i = \frac{f_i}{\sum_j \text{sh}(d_{ij})}$$

Shared fitness prevents one species dominating. \(\text{sh}(d)=1\) if \(d < \delta_t\), else 0.

Simulated Annealing Accept

$$P(\text{accept}) = e^{\Delta E / T}$$

Accept worse move with probability \(e^{\Delta E/T}\). As \(T \to 0\), behaves like hill climbing.

Pattern Database Heuristic

$$h_{PDB}(n) = \text{lookup}(\text{pattern}(n))$$

Precompute BFS backward from goal on a relaxed sub-problem (e.g., only tiles 1–4). Store costs indexed by pattern. Admissible: sub-problem cost ≤ full cost. Dominates \(h_1\) and \(h_2\).

Static Evaluation (Chess)

$$\text{SEV} = \sum \text{MAX pieces} - \sum \text{MIN pieces}$$

Piece values: Queen = 9, Bishop = 3, Knight = 3, Pawn = 1. Positive = MAX advantage. Used when full tree search is too deep.

(b) Babloo says: "Greedy Best-First Search is always smart because it selects the node closest to the goal." Prove or disprove with an example/argument. [1 Mark]

Show Answer — Part (b)

Disprove. GBFS is NOT always optimal. It uses only \(f(n) = h(n)\) — ignoring the actual cost \(g(n)\) to reach a node.

Using the same graph: GBFS from S expands A (h=8) over B (h=6) — wait, it picks B (lower h). From B it picks D (h=4). From D it picks G (h=0). Path: S→B→D→G, cost = 5+4+3 = 12.

A* found S→A→D→G with cost 8. GBFS found a suboptimal path. It was "fooled" by a high-h node (A) that actually led to the optimal route. GBFS is fast but not optimal.

Common trap: "GBFS finds the path with smallest heuristic values, so it must be shortest." WRONG — heuristic estimates the remaining distance, not total path cost. GBFS ignores how far it has already travelled.

Question 4 — Heuristics: Pattern Database [4.5 Marks]

A puzzle enthusiast wants to build an AI-based Rubik's Cube solver. Instead of searching through all possible configurations, they use a pattern database to store precomputed solution costs for selected cube patterns and use these values as heuristics during search.

(a) Explain what "patterns" may be stored in a pattern database for the 8-puzzle (not the Rubik's Cube). What is stored and why? [1.5 Marks]

Show Answer — Part (a)

Patterns in an 8-Puzzle PDB

A pattern is a partial board state: the positions of a chosen subset of tiles, with all other tiles treated as blank ("don't care").

Example: A "1–4 pattern database" tracks only where tiles 1, 2, 3, 4 are located. The positions of tiles 5, 6, 7, 8 are ignored. Each unique arrangement of tiles 1–4 on the 9 cells is one entry in the database.

What is stored: For every possible arrangement of the pattern tiles, the database stores the minimum number of moves required to move just those tiles to their goal positions (solving the sub-problem). This is computed offline by running BFS backwards from the goal state on the relaxed sub-problem.

Why useful: During A* search, h(n) = look up the current pattern in the database. This is faster than computing Manhattan distance and often more accurate — it captures interactions between tiles.

Exam tip: A PDB is always admissible. The sub-problem cost ≤ full problem cost because moving only a subset of tiles is strictly easier than moving all tiles. Admissibility is guaranteed by construction.

(b) Describe how the pattern database is generated. [1.5 Marks]

Show Answer — Part (b)

Generation: Backward BFS from Goal

  1. Define the sub-problem: Choose a subset of tiles (e.g., tiles 1–4). Project the full goal state onto just these tiles — this is the sub-goal state.
  2. Run BFS backwards: Start from the sub-goal state. Apply all legal moves in reverse. Each move gives a new partial arrangement. Record the cost (number of moves) to reach this arrangement from the goal.
  3. Store all reachable arrangements: Index by the pattern (arrangement of the subset tiles). Cost = BFS depth = number of moves to solve just this sub-problem.
  4. Result: A table mapping every partial arrangement → minimum moves to solve it.

At search time: given state n, extract the positions of the pattern tiles, look up in the table → h(n) is returned in O(1).

(c) How does using a PDB as the heuristic help A* find the solution faster compared to \(h_1\) (misplaced tiles) or \(h_2\) (Manhattan distance)? [1.5 Marks]

Show Answer — Part (c)

Why PDB Dominates h₁ and h₂

HeuristicWhat it capturesWeakness
\(h_1\) misplaced tilesBinary: is tile in place or notIgnores HOW far out of place; very loose estimate
\(h_2\) Manhattan distanceSum of individual tile distancesAssumes tiles move independently; ignores tile-tile interactions
\(h_{PDB}\)Actual cost to solve a sub-problem including interactions among chosen tilesMore accurate; fewer nodes expanded

Dominance: \(h_{PDB}(n) \geq h_2(n) \geq h_1(n)\) for all n. A more informed (higher) admissible heuristic means A* expands fewer nodes — it prunes more of the search tree. The PDB captures cases where two tiles must "pass through each other," which Manhattan distance misses entirely.

For the Rubik's Cube: storing corner-cubelet positions gives a PDB that massively reduces search — without it, the search space (~4.3×10¹⁹ states) is intractable.

Question 6 — NEAT & Neural Architecture Search [3 Marks]

(a) A CoDeepNEAT-evolved image classification network achieves only 60% accuracy after 50 generations. As an AI engineer, suggest modifications to improve performance and justify your choices. Answer in fewer than 70 words. Vague and generic answers will earn no credit. [3 Marks]

Model Answer (≤70 words)

2. Extend training epochs per evaluation: If each candidate is trained for only 5 epochs, fitness estimates are noisy. More epochs → more accurate selection.

A game tree is given. Player 1 (MAX) makes decisions at the top level; Player 2 (MIN) makes decisions at the second level, and so on. Left-to-right traversal. Root is MAX.

(a) Apply Minimax to the tree. Show backed-up values at every node. What is MAX's optimal move? [3 Marks]

(b) A game tree has the structure: ROOT(MAX) → 3 MIN children → each MIN has 3 MAX leaves. That is 9 leaves total (x1–x9).
(i) Fill the nine leaves with values such that alpha-beta performs NO pruning at all. Show the α/β working.
(ii) Fill nine different values for maximum pruning. Show the α/β working. [4 Marks]

Show Answer — Part (b)

(i) No Pruning — arrange leaves in ascending order within each MIN group

Key insight: Alpha-beta cannot prune if no MIN node's β ever drops below or equals the ROOT's α. This happens when each MIN group's best (leftmost) value is the worst — i.e., leaves ascend within each group.

Choose: x1=1, x2=5, x3=9 | x4=2, x5=6, x6=10 | x7=3, x8=7, x9=11

  ROOT(MAX, α=−∞)
  ├── MIN-A(β=+∞): sees x1=1→β=1, x2=5 (5>1, no update), x3=9 → β stays 1. ROOT α=1.
  ├── MIN-B(β=+∞): sees x4=2→β=2. Check: β=2 > α=1 → no prune.
  |                sees x5=6→no update. sees x6=10→no update. β=2. ROOT α=2.
  └── MIN-C(β=+∞): sees x7=3→β=3. Check: β=3 > α=2 → no prune.
                   sees x8=7→no update. sees x9=11→no update. β=3.
  ROOT = max(1,2,3) = 3. No pruning occurred.

(ii) Maximum Pruning — arrange leaves in descending order within each MIN group

Key insight: Maximum pruning happens when each MIN group's first leaf is the highest (so the group's β drops very low immediately), AND the left group sets a high α for ROOT (so subsequent groups get pruned early).

Choose: x1=9, x2=5, x3=1 | x4=8, x5=? | x6=7, x7=?

  ROOT(MAX, α=−∞)
  ├── MIN-A(β=+∞): x1=9→β=9, x2=5→β=5, x3=1→β=1. MIN-A=1. ROOT α=1.
  ├── MIN-B(β=+∞): x4=8→β=8. Check: β=8 > α=1 → no prune yet.
  |                x5=0→β=0. Check: β=0 ≤ α=1 → PRUNE x6 (third leaf never evaluated).
  └── MIN-C(β=+∞): x7=8→β=8. Check: β=8 > α=1 → no prune.
                   x8=0→β=0. Check: β=0 ≤ α=1 → PRUNE x9.

Pruned: x6 and x9. ROOT = max(1, 0, 0) ... actually, MIN-B and MIN-C both return values ≤ α=1, so ROOT stays 1.

Exam tip — Rule of thumb: For NO pruning: within each MIN group, leftmost leaf = smallest (MIN takes it immediately, β is set low but never triggers α ≥ β). For MAX pruning: within each MIN group, leftmost leaf = largest (β drops quickly) AND early groups set a high ROOT α so later groups get cut off.

(c) Under what condition(s) does alpha-beta pruning have better time complexity than minimax? State the best-case complexity. [1 Mark]

Show Answer — Part (c)

Condition: Alpha-beta beats minimax when moves are ordered well — specifically, when the best move at each MAX node is examined first and the worst move at each MIN node is examined first.

Best-case complexity: \(O(b^{m/2})\) — effectively doubles the search depth for the same computational budget as minimax's \(O(b^m)\).

Worst case: If moves are ordered adversarially (worst move first at every level), alpha-beta degrades to \(O(b^m)\) — same as minimax with no pruning.

Average case (random ordering): approximately \(O(b^{3m/4})\).

4. Top Exam Traps

Top 8 Exam Traps (based on actual 2026 regular paper)
  1. Acting humanly ≠ acting intelligently — Sana-type scenario: mimicking human interview style is not the same as optimising the performance measure. A rational agent maximises utility, not human-likeness.
  2. Stop A* at EXPANSION, not generation — generating the goal node does not complete the search; you must expand it (dequeue from frontier). Students often mark the solution too early.
  3. Pattern database is admissible — the PDB solves a relaxed sub-problem (fewer tiles), so its cost ≤ full problem cost. State this explicitly in any proof of admissibility.
  4. GA crossover on complex chromosomes — crossover point is between records (time slots), not inside a field. Swapping half a time-slot record is wrong.
  5. Inverse alpha-beta: no-prune ordering — for no pruning, each MIN group's leftmost leaf must be its minimum (sets β low immediately, but β stays > α). Ascending order within MIN groups achieves this.
  6. Inverse alpha-beta: max-prune ordering — leftmost leaf in each MIN group must be high (β drops toward the floor set by ROOT α). Descending order within MIN groups + increasing left-to-right across groups achieves maximum cuts.
  7. Pruning condition is β ≤ α (not β < α) — equal values also trigger a prune. The condition is "no benefit possible," which includes the equal case.
  8. MCTS phases order — always: Selection → Expansion → Simulation → Backpropagation. Never simulate before expanding a new node.
