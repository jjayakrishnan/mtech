Lesson 5: NEAT, NAS & Neural Architecture Search

Lesson 5 of 6 • NeuroEvolution of Augmenting Topologies • Architecture Search

1. The Competing Conventions Problem

Imagine you have two neural networks that compute exactly the same function, but their hidden neurons are numbered differently. If you try standard genetic crossover on their weight vectors, you get garbage — you're mixing weights that belong to completely different connections.

The Core Problem

Standard GA represents a neural network as a fixed-length vector of weights. But two networks with identical behavior can have their neurons in any order — this is the permutation problem (also called "competing conventions").

Result: crossover between two equally-fit parents produces a broken offspring most of the time.

Why This Kills Standard Neuroevolution

  • Crossover is destructive — swapping genes at position $i$ mixes unrelated weights
  • No meaningful alignment — position in the genome does not correspond to function
  • Topology is fixed — you must pre-decide the network structure; GA only tunes weights
  • No structural innovation — cannot discover that adding a hidden node helps

2. NEAT: NeuroEvolution of Augmenting Topologies

NEAT (Stanley & Miikkulainen, 2002) solves the competing conventions problem with one elegant idea: give every connection a historical birth certificate.

Key Insight

If we know when each connection was invented (its innovation number), we can align genomes by innovation number instead of position. Two genes with the same innovation number represent the same structural connection — regardless of where they sit in the genome.

NEAT Genome Encoding

A NEAT genome is a list of connection genes, each containing:

FieldMeaning
In-NodeSource neuron ID
Out-NodeDestination neuron ID
WeightConnection weight (real number)

Crossover by Innovation Number

When two parents mate, we line up genes by innovation number (not position):

  • Matching genes (same innovation # in both parents): randomly pick from either parent
  • Disjoint genes (in range of other parent but missing): inherited from fitter parent
  • Excess genes (beyond range of other parent): inherited from fitter parent

3. NEAT Mutation Operators

Start Minimal

NEAT begins with the simplest possible topology: inputs directly connected to outputs, no hidden nodes. Complexity grows only when evolution discovers it helps.

Structural Mutations

NEAT adds complexity through two structural mutations:

Add-Connection Mutation

A new connection is created between two previously unconnected nodes. It receives a new innovation number from the global counter.

Before: A → C After: A → C B → C B → C A → B (NEW! Innovation #9)

Add-Node Mutation

An existing connection is split: a new node is inserted in the middle. The old connection is disabled, and two new connections are created:

  1. Old-source → New-node (weight = 1.0)
  2. New-node → Old-destination (weight = old weight)

Why Start Minimal?

  • Avoids bloat — only adds nodes when needed
  • Search begins in a low-dimensional space (fast to explore)
  • Complexification: structure grows incrementally alongside fitness

4. Speciation & Fitness Sharing

New structural mutations initially perform worse than optimized simpler networks. Without protection, evolution kills innovations before they can be refined. NEAT solves this through speciation.

Compatibility Distance

NEAT measures how different two genomes are:

Compatibility Formula

$$\delta = \frac{c_1 \cdot E}{N} + \frac{c_2 \cdot D}{N} + c_3 \cdot \bar{W}$$

  • $E$ = number of excess genes
  • $D$ = number of disjoint genes
  • $N$ = number of genes in larger genome (1 if $N < 20$)
  • $\bar{W}$ = average weight difference of matching genes
  • $c_1, c_2, c_3$ = tunable coefficients

If $\delta < \delta_t$ (threshold), two genomes belong to the same species.

Fitness Sharing

Within each species, fitness is shared (divided) among members. This prevents any one species from dominating and gives new topologies room to improve.

NEAT Fitness Sharing Formula

$$f'_i = \frac{f_i}{|S_i|}$$

Where $f_i$ is raw fitness and $|S_i|$ is the number of individuals in species $S_i$.

Effect: large species are penalised, small species (new innovations) get a fitness boost relative to their population share.

Speciation Mechanics

  1. Each species has a representative (random member from previous generation)
  2. Each new organism is placed in the first species whose representative it is compatible with
  3. Species that don't improve for $n$ generations are eliminated
  4. Number of offspring per species is proportional to total adjusted fitness of that species

5. DeepNEAT & CoDeepNEAT

Original NEAT works for small networks. For deep learning scale, we need extensions:

DeepNEAT

DeepNEAT Approach

  • Each node in the genome represents a layer (not a single neuron)
  • Layer properties (type, size, activation, dropout) are evolved
  • Connections between layers are evolved using the same NEAT mechanics
  • Still uses innovation numbers, speciation, and complexification

CoDeepNEAT (Coevolutionary DeepNEAT)

CoDeepNEAT evolves at two levels simultaneously:

  1. Module population: evolves small network modules (e.g., a conv block)
  2. Blueprint population: evolves how modules are assembled together

A blueprint is a graph where each node is a pointer to a module species. To evaluate a blueprint, modules are sampled from their respective species and assembled. This enables combinatorial exploration of both local structure (modules) and global architecture (assembly).

Blueprint: [Module-A-species] → [Module-B-species] → [Output] | | Samples from: Samples from: Module pop. A Module pop. B Each evaluation assembles different module instances into the same blueprint skeleton.

6. Neural Architecture Search (NAS)

NAS generalises the idea: instead of hand-designing network architectures, let an algorithm search for the best architecture automatically.

Three Components of NAS

ComponentWhat It DefinesExamples
Search SpaceWhat architectures are possibleLayer types, skip connections, kernel sizes
Search StrategyHow to explore the spaceEvolutionary, RL, gradient-based
Performance EstimationHow to evaluate a candidateFull training, proxy tasks, weight sharing

7. NAS Methods Compared

Evolutionary NAS

  • Directly applies GA/NEAT principles to architecture search
  • Population of architectures; mutation adds/removes layers/connections
  • Selection based on validation accuracy
  • Example: AmoebaNet (Real et al., 2019) — competitive with hand-designed architectures

RL-based NAS

  • A controller (RNN) generates architecture descriptions as sequences of tokens
  • Architecture is trained; validation accuracy is the reward signal
  • Controller is updated via policy gradient (REINFORCE)
  • Example: NASNet (Zoph & Le, 2017) — pioneering but very expensive (800 GPUs)

Gradient-based NAS (DARTS)

  • DARTS: Differentiable Architecture Search (Liu et al., 2019)
  • Makes the search space continuous — assign a weight to every possible operation
  • Joint optimization: architecture weights + network weights via gradient descent
  • Orders of magnitude faster (single GPU, few days)

Performance Estimation Strategies

  • Full training: accurate but extremely expensive
  • Proxy tasks: train for fewer epochs, smaller dataset, smaller model
  • Weight sharing / Supernet: all candidate architectures share weights in one big network; avoids retraining from scratch
  • Predictor-based: learn to predict accuracy from architecture features

8. The GA → NAS Connection

Unifying View

NAS is fundamentally GA applied to neural network design:

GA ConceptNAS Equivalent
IndividualNetwork architecture
GeneLayer type, connection, hyperparameter
Fitness functionValidation accuracy (or proxy)
MutationAdd/remove layers, change operations
CrossoverCombine sub-graphs from parents
SelectionTournament on accuracy
Speciation (NEAT)Diversity preservation in search

The evolution from fixed-topology neuroevolution → NEAT → DeepNEAT → NAS shows a clear trajectory: automating more and more of the neural network design process.

Parent B (fitness = 7.2) has genes with innovation numbers: [1, 2, 3, 5, 6, 7]

Question: Which genes are disjoint in Parent A, disjoint in Parent B, and excess in Parent A?

  • Matching genes: 1, 2, 3, 5 (present in both — pick randomly from either parent)
  • Disjoint in A: 4 (within B's range [1-7] but missing from B)
  • Disjoint in B: 6, 7 (within A's range [1-8] but missing from A)
  • Excess in A: 8 (beyond B's maximum innovation # of 7)

Since A is fitter: child inherits matching genes (random choice), disjoint from A (gene 4), and excess from A (gene 8). Disjoint from B (6, 7) are not inherited.
