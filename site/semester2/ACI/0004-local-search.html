Lesson 4: Local Search & Hill Climbing

Lesson 4 of 8 | Core algorithms — pseudocode expected in exam

1. What is Local Search?

In standard search (BFS, DFS, A*), we care about the path from start to goal. Local search is fundamentally different: we only care about the final state, not how we got there.

Think of it like this: if you need to find the highest point in a mountain range, you don't care about the trail you took — you only care that you're standing on the tallest peak.

Key Characteristics

  • Operate on a single current state (not a search tree of many paths)
  • Move to neighboring states by making small modifications
  • No path is stored — very memory efficient (constant space)
  • Useful for optimization problems where the goal is to find the best state
  • Can find solutions in infinite or very large state spaces where systematic search fails

When to use Local Search?

  • N-Queens placement (find a valid arrangement, don't care how you got there)
  • VLSI circuit layout optimization
  • Job scheduling — find the best schedule
  • Travelling Salesman Problem — find a short tour

Exam Trap: What "1NN" Actually Means

  • 1NN means one queen moves — change exactly one queen's position (to any other row in her column). It does NOT mean "nearest neighbor" from ML.
  • One movement = any row skip — moving a queen from row 4 to row 1 is still ONE movement. Don't count steps.
  • Enumerate all 12 neighbors — for 4-Queens, 1NN from any state always gives exactly 12 neighbors (3 alternatives × 4 queens).
  • Fitness value = heuristic value — these terms are used interchangeably in exam questions.

2. Hill Climbing (Steepest Ascent)

The simplest local search algorithm. Imagine you're blindfolded on a hillside and want to reach the top. At each step you feel around in all directions and take one step in whichever direction goes up the most. That's hill climbing.

Pseudocode (Exam-Ready)

function HILL_CLIMBING(problem): current = initial_state while true: neighbor = best successor of current if value(neighbor) <= value(current): return current current = neighbor

Step-by-Step Breakdown

  1. Start at some state (could be random or given)
  2. Generate all neighbors of the current state
  3. Evaluate each neighbor using an objective/value function
  4. Pick the neighbor with the highest value (steepest ascent)
  5. If that best neighbor is no better than current → stop and return current
  6. Otherwise, move to that neighbor and repeat

Important: Hill climbing is a greedy algorithm. It never looks back, never keeps alternatives in memory, and never backtracks. This makes it fast but incomplete.

3. Problems with Hill Climbing

Hill climbing can get stuck. There are exactly three failure modes you must memorize:

a) Local Maxima

A state that is better than all its neighbors, but not the global maximum. The algorithm stops here because no neighbor improves the value — but a better solution exists elsewhere.

Value | * <-- Global Maximum | / \ | * / \ <-- Local Maximum (stuck here!) | / \/ \ |/ \ +----------------> State space

b) Plateaus (Flat Areas) and Shoulders

A plateau is a flat region where all neighbors have the same value. The algorithm cannot tell which direction to go. A shoulder is a plateau with an uphill edge — it looks flat locally but eventually leads to improvement.

Value | ___* | ______/ <-- Shoulder (flat then up) | / |___/ <-- Plateau (flat, might get stuck) | +----------------> State space

c) Ridges

A narrow region of high value that slopes away steeply on both sides. The ridge itself goes uphill, but no single step in the neighbor directions actually moves along the ridge. Successive steps oscillate back and forth without making progress upward.

Exam Tip

When asked "what are the problems with hill climbing?", always list all three: local maxima, plateaus/shoulders, ridges. Then briefly explain each. This is a common 3-5 mark question.

4. Hill Climbing Variants

a) First-Choice Hill Climbing

Instead of evaluating all neighbors to find the best one, generate neighbors randomly and accept the first one that improves the current state.

  • Advantage: Much faster when there are many neighbors (e.g., thousands of successors)
  • Disadvantage: Still gets stuck in local maxima — just arrives there faster

b) Random-Restart Hill Climbing

Run basic hill climbing multiple times from different random starting states. Keep the best result across all runs.

  • Key Insight: If each run has probability $p$ of finding the global max, then after $k$ restarts the probability of finding it is $1 - (1-p)^k$
  • Completeness: With enough restarts, it will find the global maximum with probability approaching 1
  • Trivially parallelizable — run each restart on a different processor

Random-Restart is Surprisingly Effective

For the 8-Queens problem, basic hill climbing succeeds only about 14% of the time. But random-restart solves it quickly because each failure only takes a few steps before getting stuck, and 7 restarts on average are enough to find a solution.

Worked Example: 4-Queens Stochastic Hill Climbing

State Encoding

A state is a vector \([V_1, V_2, V_3, V_4]\) where \(V_i\) = row of the queen in column \(i\).
Example: \([2, 4, 2, 2]\) means queen in column 1 is in row 2, column 2 in row 4, etc.

Fitness Function

Fitness = number of non-conflicting queen pairs (maximize).
Total possible pairs = \(\binom{4}{2} = 6\). Goal state has fitness = 6 (no conflicts at all).

1NN (1-Neighbor-Neighborhood) Expansion

From any state, move exactly one queen to a different row within her column.
Each queen has 3 alternative rows × 4 queens = 12 neighbors total.

Stochastic Hill Climbing: Step-by-Step

Starting state: \([2, 4, 2, 2]\) — fitness = 4

  1. Generate all 12 neighbors by moving each queen once.
  2. Compute fitness for each of the 12 neighbors.
  3. Count frequency of each unique fitness value.
  4. Compute probability = frequency / 12.
  5. Pick the group with highest fitness (not just most frequent — this is the key difference from frequency-based selection).

Neighbor Fitness Distribution (Example Trace)

Fitness Value (non-conflicting pairs) Frequency Probability
1 2 \(2/12 \approx 0.17\)
2 5 \(5/12 \approx 0.42\)
3 4 \(4/12 \approx 0.33\)
4 (current state) 1 \(1/12 \approx 0.08\)

Observation: All 12 neighbors have fitness ≤ 4 (the current state's fitness). No neighbor is strictly better. This means we are at a local maximum.

Local Maximum — Hill Climbing Stops Here

When no neighbor has higher fitness than the current state, that is a local maximum. Hill climbing stops here. The algorithm returns the current state even though it may not be the global optimum (fitness = 6).

Solutions: Use random restart (try again from a different starting state) or beam search (run multiple hill climbers in parallel and share information).

Basic vs Stochastic Selection

Method Selection Rule
Basic Hill Climbing Always pick the single best neighbor
Stochastic Hill Climbing Pick randomly from neighbors with fitness ≥ current, weighted by fitness

Beam Search = Parallelized Hill Climbing

Prof's Framing

"Beam search is nothing but parallelizing hill climbing. Initially K random states at the start. Calculate the fitness value for all their successors. If any one finds the global optimum, they immediately inform the others to stop."

5. Simulated Annealing (Reference Only — Out of Scope)

Inspired by the metallurgical process of annealing (heating metal and slowly cooling it to remove defects). The key idea: allow "bad" moves early on to escape local maxima, then gradually become more greedy as the "temperature" drops.

How It Works

  1. Pick a random neighbor (not necessarily the best)
  2. If the neighbor is better → always accept it
  3. If the neighbor is worse → accept it with probability $P = e^{\Delta E / T}$
  4. Decrease temperature $T$ over time according to a cooling schedule

The Acceptance Formula

Formula (Memorize)

$P(\text{accept worse move}) = e^{(\text{value}(\text{next}) - \text{value}(\text{current})) / T}$

Where:

  • $\Delta E = \text{value(next)} - \text{value(current)}$ — this is negative for a worse move
  • $T$ = current temperature (starts high, decreases over time)
  • When $T$ is high: $e^{\text{small negative}/\text{large}} \approx e^0 \approx 1$ → almost always accept
  • When $T$ is low: $e^{\text{negative}/\text{small}} \approx e^{-\infty} \approx 0$ → almost never accept
  • When $T = 0$: behaves exactly like hill climbing (no bad moves accepted)

Pseudocode

function SIMULATED_ANNEALING(problem, schedule): current = initial_state for t = 1, 2, 3, ...: T = schedule(t) // temperature at time t if T = 0: return current next = random neighbor of current delta_E = value(next) - value(current) if delta_E > 0: current = next // always accept improvement else: accept with probability e^(delta_E / T) if accepted: current = next

Intuition

Early on (high T): the algorithm behaves like a random walk — explores widely, escapes traps.
Late (low T): the algorithm behaves like hill climbing — settles into the nearest peak.
If cooled slowly enough, simulated annealing is guaranteed to find the global optimum (in theory).

6. Local Beam Search

Instead of keeping just one current state, keep $k$ states in parallel.

Algorithm

  1. Start with $k$ randomly generated states
  2. At each step, generate all successors of all $k$ states
  3. From the entire pool of successors, keep the best $k$
  4. Repeat until a goal is found or no improvement is possible

Common Misconception

Local Beam Search is NOT the same as running $k$ random restarts in parallel! The key difference: in beam search, the $k$ states share information. If one state generates a great successor, all $k$ slots can be filled from that state's children. In random restarts, each run is completely independent.

Problem with Basic Beam Search

The $k$ states can quickly become concentrated in a small region (loss of diversity). All $k$ states converge to the same neighborhood and you lose the benefit of parallel exploration.

Stochastic Beam Search fixes this by choosing successors probabilistically (weighted by value) rather than always picking the top $k$.

7. Online Search Agents

All algorithms so far assume the agent knows the full state space in advance (offline search). An online search agent does not know:

  • What states exist beyond the current one
  • What the result of an action will be until it actually tries it
  • The full transition model — it learns as it goes

The agent must interleave computation and action: take an action, observe the result, plan the next action based on what it now knows. This is useful for:

  • Robot navigation in unknown terrain
  • Exploration problems where the map is unknown
  • Real-time systems where you can't afford to compute a full plan

Online vs Offline

PropertyOffline SearchOnline Search
State space known?Yes, fullyNo, discovered as agent acts
Plans before acting?Yes, complete plan firstNo, interleaves planning and acting
ExampleA* on a known mapRobot in unknown building
Can backtrack?In memory (search tree)Must physically move back

Competitive ratio: We measure online algorithms by comparing the path cost they actually incur vs the optimal cost an omniscient agent would incur. An online agent cannot do worse than the competitive ratio bound for its algorithm class.

Consider a 1D state space with 5 states and their objective function values:
A(2), B(5), C(8), D(6), E(3).
Each state's neighbors are the states adjacent to it (A's neighbor is B; B's neighbors are A and C; etc.).

Solution (Written Trace)

Show full written trace

Hill Climbing from A:

  1. current = A (value 2). Neighbor: B (value 5). Since 5 > 2, move to B.
  2. current = B (value 5). Neighbors: A (value 2), C (value 8). Best neighbor: C (value 8). Since 8 > 5, move to C.
  3. current = C (value 8). Neighbors: B (value 5), D (value 6). Best neighbor: D (value 6). Since 6 < 8, do not move.
  4. Return C (value 8). This is the global maximum in this space, so we got lucky!

Hill Climbing from E:

  1. current = E (value 3). Neighbor: D (value 6). Since 6 > 3, move to D.
  2. current = D (value 6). Neighbors: C (value 8), E (value 3). Best neighbor: C (value 8). Since 8 > 6, move to C.
  3. current = C (value 8). Neighbors: B (value 5), D (value 6). Best neighbor: D (value 6). Since 6 < 8, do not move.
  4. Return C (value 8). Both starting points converge to C here.

Why Random Restart Helps in General:

In this simple example both paths converge to C. But imagine if the state space were: A(2), B(5), C(4), D(3), E(9). Starting from A, hill climbing reaches B (local max, value 5) and gets stuck. Random-restart from E would find E (value 9) immediately — the global max. Different starting points explore different regions of the landscape.

Try It Yourself

Trace Simulated Annealing on a small state space

Scenario: State space: P(3), Q(7), R(5), S(9), T(4). Neighbors are adjacent states. You start at state R (value 5). Temperature T=10, cooling: T = T * 0.5 each step.

Questions:

  1. At step 1 (T=10): A random neighbor Q (value 7) is chosen. Do you accept? Why?
  2. At step 2 (T=5): You're at Q. Random neighbor P (value 3) is chosen. Calculate acceptance probability.
  3. At step 3 (T=2.5): You're still at Q. Random neighbor R (value 5) is chosen. Calculate acceptance probability.

Sample Answers

  1. Yes, accept. value(Q)=7 > value(R)=5. Since the move improves, we always accept regardless of temperature.
  2. $\Delta E = 3 - 7 = -4$. $P = e^{-4/5} = e^{-0.8} \approx 0.449$. About 45% chance of accepting this bad move. With T still relatively high, bad moves are often accepted.
  3. $\Delta E = 5 - 7 = -2$. $P = e^{-2/2.5} = e^{-0.8} \approx 0.449$. Same probability! (Different numbers give same ratio.) As T drops further, this probability would shrink.

Hill Climbing Pseudocode

current = initial_state
loop: neighbor = best successor
  if value(neighbor) <= value(current): return current
  else: current = neighbor

Three Failures of Hill Climbing

  1. Local Maxima — peak that is not the global best
  2. Plateaus/Shoulders — flat regions, no gradient to follow
  3. Ridges — narrow high region; no single neighbor step follows the ridge

Simulated Annealing Acceptance

$P(\text{accept}) = e^{(\text{value(next)} - \text{value(current)}) / T}$

  • High T → accept almost anything (exploration)
  • Low T → accept only improvements (exploitation)
  • T = 0 → pure hill climbing

Local Beam Search

  • Keep $k$ states, generate all successors of all $k$, keep best $k$
  • NOT the same as $k$ parallel random restarts (states share information)
