1. Recap & Why Heuristics Matter

CS2 covered uninformed search (BFS, DFS, UCS) and CS3 begins by reviewing uninformed algorithms before moving to informed search. The key insight: a good heuristic tells the agent where to look, dramatically cutting the work.

Uninformed vs Informed Search

PropertyBFSDFSUCSGBFSA*
Uses \(h(n)\)?NoNoNoYesYes
Uses \(g(n)\)?NoNoYesNoYes
Optimal?No*NoYesNoYes (if adm. \(h\))

*BFS optimal when all step costs are equal.

A heuristic function \(h(n)\) estimates the cost from node \(n\) to the goal. It encodes problem-specific knowledge that a blind search cannot use. The quality of \(h\) determines how efficiently A* finds the optimal path.

2. Admissibility — Never Overestimate

Admissibility Condition
$$h(n) \leq h^*(n) \quad \text{for all nodes } n$$

\(h^*(n)\) = true optimal cost from \(n\) to goal

A heuristic is admissible if it never overestimates the cost to reach the goal. An admissible heuristic is always optimistic вҖ” it thinks the goal is at least as close as it really is.

Why admissibility guarantees A* optimality

A* expands node \(n\) when \(f(n) = g(n) + h(n)\) is minimum on the frontier. If \(h\) never overestimates, then \(f(n)\) never overestimates the true cost through \(n\). So when A* expands the goal node, the path found must be optimal вҖ” any cheaper path would have been expanded first.

Worked Example: Checking Admissibility Node-by-Node

Given a 5-node graph (goal = Node 3) with these heuristic values: \(h(1)=80\), \(h(2)=60\), \(h(3)=0\), \(h(4)=200\), \(h(5)=190\). Edge costs: 1вҶ’2: 70, 2вҶ’3: 50, 1вҶ’3: 125, 4вҶ’5: 50, 5вҶ’1: 75.

For each node, find the minimum actual cost to the goal, then compare to \(h(n)\):

Node\(h(n)\) givenPaths to goal (Node 3)Min actual cost \(h^*(n)\)Admissible?
Node 1 80 1вҶ’3: 125; 1вҶ’2вҶ’3: 70+50=120 120 Yes (80 вүӨ 120)
Node 2 60 2вҶ’3: 50 50 No (60 > 50) вң—
Node 4 200 4вҶ’5вҶ’1вҶ’2вҶ’3: 50+75+70+50=245; 4вҶ’5вҶ’1вҶ’3: 50+75+125=250 245 Yes (200 вүӨ 245)
Node 5 190 5вҶ’1вҶ’2вҶ’3: 75+70+50=195; 5вҶ’1вҶ’3: 75+125=200 195 Yes (190 вүӨ 195)

Result: Heuristic NOT admissible

Node 2 fails: \(h(2) = 60 > h^*(2) = 50\). The heuristic overestimates for Node 2. Go back to the domain expert (SME) and ask for a corrected heuristic value for Node 2.

Admissibility must hold for every node вҖ” one failure makes the entire heuristic inadmissible.

Examples of Admissible Heuristics

  • 8-puzzle, \(h_1\): Number of misplaced tiles. Each misplaced tile needs at least 1 move \(\Rightarrow\) admissible.
  • 8-puzzle, \(h_2\): Manhattan distance. Each tile's actual path is \(\geq\) its Manhattan distance \(\Rightarrow\) admissible.
  • Romania route-finding: Straight-line distance to Bucharest. Road distance \(\geq\) straight-line distance \(\Rightarrow\) admissible.

Non-admissible example (violates admissibility)

Suppose a heuristic says "the goal is 5 steps away" but the true distance is 3. This overestimates: \(h(n) = 5 > h^*(n) = 3\). A* might skip the optimal path because \(f = g + 5\) looks worse than an inferior path with \(f = g' + h'\). Result: A* is no longer guaranteed to find the optimal path.

3. Consistency (Monotonicity)

Consistency Condition
$$h(n) \leq c(n, a, n') + h(n')$$

\(c(n, a, n')\) = cost of action \(a\) from node \(n\) to successor \(n'\)

A heuristic is consistent (also called monotone) if the estimated cost from any node \(n\) is no greater than the step cost to a successor \(n'\) plus the estimated cost from \(n'\). This is a triangle inequality.

Key relationship: Consistency \(\Rightarrow\) Admissibility (but not vice versa). A consistent heuristic is always admissible. An admissible heuristic is not always consistent.

Why consistency matters for A*

  • With a consistent heuristic, \(f(n)\) is non-decreasing along any path.
  • Once A* expands a node, it has found the optimal path to that node вҖ” it never needs to reopen it.
  • This makes A* more efficient in practice (no duplicate expansions).

Worked Example: Checking Consistency Edge-by-Edge

Using the same 5-node graph (heuristics: \(h(1)=80\), \(h(2)=60\), \(h(3)=0\), \(h(4)=200\), \(h(5)=190\)):

EdgeFormula check: \(h(i) \leq c(i,j) + h(j)\)Result
1 вҶ’ 2 (cost 70) \(80 \leq 70 + 60 = 130\) Consistent вң“
1 вҶ’ 3 (cost 125) \(80 \leq 125 + 0 = 125\) Consistent вң“
2 вҶ’ 3 (cost 50) \(60 \leq 50 + 0 = 50\) Not consistent вң— (60 > 50)
5 вҶ’ 1 (cost 75) \(190 \leq 75 + 80 = 155\) Not consistent вң— (190 > 155)

The heuristic fails consistency on edges 2вҶ’3 and 5вҶ’1. Note: the heuristic for Node 2 was already inadmissible вҖ” an inadmissible heuristic is never consistent on the edges touching that node.

Try It: Verify Consistency

For the Romania map, straight-line distance to Bucharest: \(h(\text{Arad}) = 366\), \(h(\text{Sibiu}) = 253\), road cost AradвҶ’Sibiu = 140.

Check: is \(h(\text{Arad}) \leq c(\text{Arad}, \text{Sibiu}) + h(\text{Sibiu})\)?

\(366 \leq 140 + 253 = 393\). Yes, \(366 \leq 393\) вң“ вҖ” the straight-line heuristic is consistent here.

4. Designing Admissible Heuristics

4a. The Relaxed Problem Method

The most principled technique: remove one or more constraints from the original problem. The optimal cost of the relaxed problem \(\leq\) the optimal cost of the original (relaxing constraints can only make things easier). So the relaxed cost is an admissible heuristic for the original.

How to derive a heuristic via relaxation

  1. Write down the problem's action constraints precisely.
  2. Remove one constraint (e.g. "tiles can only move to adjacent empty squares" вҶ’ remove "empty" or remove "adjacent").
  3. Compute the optimal cost in the relaxed problem вҖ” this is your \(h(n)\).
  4. Verify: is it still \(\leq h^*(n)\) for the original? (It will be, by construction.)

8-Puzzle: Two Relaxations

Original: A tile may move to an adjacent empty square.

HeuristicRemoved constraint\(h(n)\) formulaQuality
\(h_1\) вҖ” Misplaced Tiles Remove "adjacent" AND "empty" вҖ” tile teleports to goal Count of tiles not in goal position Easy to compute; weakest estimate
\(h_2\) вҖ” Empty Tile Manhattan Remove "other tiles must not be in the way" вҖ” only track the blank Manhattan distance of the blank tile to its goal position Ignores all numbered tiles; not very informative
\(h_3\) вҖ” Full Manhattan Distance Remove "empty" constraint вҖ” tiles can slide through each other \(\sum_{\text{tiles}} |r_i - r_g| + |c_i - c_g|\) Best of the three; dominates both \(h_1\) and \(h_2\)

Prof's Recommendation

"HвӮҒ (misplaced tiles) is not very good. You should always go for HвӮғ (Manhattan distance of all tiles). HвӮғ is the correct heuristic to use. In exams, if the initial state is not given, you can assume any starting configuration."

\(h_2\) dominates \(h_1\): Manhattan distance is always \(\geq\) misplaced count (each misplaced tile contributes \(\geq 1\) to Manhattan distance). A* with \(h_2\) expands far fewer nodes.

Try It: 8-Puzzle State

Goal state: \(\begin{bmatrix}1&2&3\\4&5&6\\7&8&\_\end{bmatrix}\)

Current state: \(\begin{bmatrix}7&2&4\\5&\_&6\\8&3&1\end{bmatrix}\)

Compute \(h_1\) (misplaced tiles) and \(h_2\) (Manhattan distance). Which is larger?

\(h_1\) (misplaced tiles): Tiles 7, 4, 5, 8, 3, 1 are not in their goal positions. That's 6 misplaced tiles (blank excluded).

\(h_2\) (Manhattan distance): For each tile, compute \(|\text{goal row} - \text{current row}| + |\text{goal col} - \text{current col}|\). Always use absolute values вҖ” distances are never negative.

  • Tile 7: goal (2,0), current (0,0) вҶ’ \(|2-0|+|0-0|=2\)
  • Tile 2: goal (0,1), current (0,1) вҶ’ 0
  • Tile 4: goal (1,0), current (0,2) вҶ’ \(|1-0|+|0-2|=1+2=3\)  (absolute value: column distance is |0вҲ’2|=2, not вҲ’2)
  • Tile 5: goal (1,1), current (1,0) вҶ’ 1
  • Tile 6: goal (1,2), current (1,2) вҶ’ 0
  • Tile 8: goal (2,1), current (2,0) вҶ’ 1
  • Tile 3: goal (0,2), current (2,1) вҶ’ \(2+1=3\)
  • Tile 1: goal (0,0), current (2,2) вҶ’ \(2+2=4\)

Total \(h_2 = 2+0+3+1+0+1+3+4 = \mathbf{14}\). Since \(14 > 6\), \(h_2\) dominates \(h_1\) here.

4b. Dominance

Heuristic Dominance
$$h_2 \text{ dominates } h_1 \iff h_2(n) \geq h_1(n) \text{ for all } n, \text{ with both admissible}$$

If \(h_2\) dominates \(h_1\), then A* with \(h_2\) will expand no more nodes than A* with \(h_1\). A dominant heuristic provides tighter lower bounds, leaving fewer nodes to explore.

Combining Heuristics (Free Lunch)

If you have multiple admissible heuristics \(h_1, h_2, \ldots, h_k\), take their pointwise maximum:

$$h(n) = \max(h_1(n), h_2(n), \ldots, h_k(n))$$

The result is still admissible (max of admissible values \(\leq h^*\)) and dominates each individual one. Cost: you compute all \(k\) heuristics.

4c. Heuristic Design for N-Queens

The N-Queens problem: place \(N\) queens on an \(N \times N\) board so that no two queens attack each other (same row, column, or diagonal). The 4-Queens problem has \(4! = 24\) possible column assignments to explore.

Three Heuristics the Class Derived

For a board with 4 queens, there are \(\binom{4}{2} = 6\) possible pairs:

HeuristicDefinitionGoal valueOptimize
\(h_1\) вҖ” Non-conflicting pairs Count pairs of queens that do NOT attack each other 6 (all pairs safe) Maximize
\(h_2\) вҖ” Conflicting pairs Count pairs of queens that DO attack each other 0 (no conflicts) Minimize
\(h_3\) вҖ” Safe queens Count queens that are not attacked by any other queen \(N\) (all queens safe) Maximize

Note: These heuristics guide local search for N-Queens (hill climbing, simulated annealing). The goal is to move from a state with conflicts to one with none. \(h_2\) (conflicting pairs) is most commonly used вҖ” you want to minimize it to 0.

Exam tip (Prof's words): "Today's Queens problem is very important. If you have not understood that clearly, please watch again вҖ” it will be helpful in local search."

Exam Warning вҖ” From the Professor

"If I show you a graph, there are some things you'll have to apply: GBFS, A*, and check for admissibility and consistency. There are also questions where things are given to you and you have to come up with relaxed versions of the problem, identify which constraint got relaxed, and design heuristics. If I were you, I would start taking this seriously."

Exam question types to expect:

  • Given a graph with heuristic values: run A* step by step, then verify admissibility and consistency for each node/edge.
  • Given a problem description: derive an admissible heuristic by identifying which constraint to relax.
  • Given two heuristics: determine which dominates and explain why.

5. Pattern Databases

Pattern databases are a systematic way to generate admissible heuristics that are much more accurate than simple relaxations like Manhattan distance.

Core Idea

Solve a subproblem: consider only a subset of tiles (the "pattern"). Precompute the exact minimum cost to solve just those tiles, for every possible configuration of those tiles. Store this in a lookup table (the database).

At search time, look up the current configuration of the pattern tiles вҖ” this exact subproblem cost is a lower bound for the full problem (since we must solve at least those tiles), making it admissible.

Building a Pattern Database (15-puzzle example)

  1. Choose a pattern, e.g. tiles {1, 2, 3, 4} in the 15-puzzle.
  2. Run a backward BFS from the goal state, considering only moves involving the pattern tiles.
  3. Record: for each reachable arrangement of these tiles, what is the minimum moves to place them correctly?
  4. Store this as a lookup table: \(\text{PDB}[\text{config}] = \text{cost}\).
  5. At search time: \(h(n) = \text{PDB}[\text{config of tiles 1вҖ“4 in state } n]\).

Additive Pattern Databases

Split all tiles into disjoint groups: group A = {1,2,3,4,5,6,7} and group B = {8,9,10,11,12,13,14,15} (for 15-puzzle). Build one PDB per group. Because moves that affect group A don't affect group B's cost count, the costs are additive:

$$h(n) = \text{PDB}_A[\text{config}_A] + \text{PDB}_B[\text{config}_B]$$

This sum is still admissible and dominates each individual PDB alone.

Why pattern databases dominate Manhattan distance: Manhattan distance treats each tile independently. Pattern databases capture interactions between tiles (one tile blocking another), giving tighter lower bounds.

6. Heuristic Accuracy & Algorithm Performance

Effective Branching Factor \(b^*\)

The quality of a heuristic is measured empirically by the effective branching factor \(b^*\). If A* expands \(N\) nodes to find a solution at depth \(d\), then \(b^*\) satisfies:

Effective Branching Factor
$$N = 1 + b^* + (b^*)^2 + \cdots + (b^*)^d = \frac{(b^*)^{d+1} - 1}{b^* - 1}$$

A perfect heuristic (\(h = h^*\)) gives \(b^* = 1\). A poor heuristic gives \(b^* \approx b\) (uninformed).

What \(b^*\) tells you

\(b^*\)InterpretationExample
\(b^* \approx 1\)Near-perfect heuristic, almost no wasted nodesPerfect knowledge of \(h^*\)
\(b^* \approx 1.2\)вҖ“\(1.5\)Good heuristic, manageable growthManhattan distance for 8-puzzle
\(b^* \approx b\)Heuristic provides no useful guidance\(h(n) = 0\) (reduces A* to UCS)

Try It: Effective Branching Factor

A* with heuristic \(h_1\) finds a depth-6 solution after expanding 20 nodes. Estimate the effective branching factor \(b^*\).

We need \(N = 20\), \(d = 6\): \(20 = 1 + b^* + (b^*)^2 + \cdots + (b^*)^6\).

This is a good heuristic вҖ” far below the actual branching factor of ~3 for the 8-puzzle.

7. Learning Heuristics from Experience

When no relaxed problem is obvious, we can learn a heuristic from data. This bridges classical AI and machine learning вҖ” a key use case cited in the ACI handout (T1: Ch 3.6, NeurIPS 2023 research).

The Basic Idea

Generate many problem instances, solve them optimally to get true costs \(h^*(s)\), then train a supervised model to predict \(h^*(s)\) from features of state \(s\).

Learning Heuristics: Supervised Approach

  1. Feature extraction: Represent each state \(s\) as a feature vector \(\mathbf{x}(s)\) (e.g. tile positions, goal distances, blocked counts).
  2. Data generation: Solve \(M\) training instances optimally (e.g. using BFS or brute force for small instances) to get labels \(y_i = h^*(s_i)\).
  3. Training: Fit a model \(\hat{h}(s) = f_\theta(\mathbf{x}(s))\) to minimise prediction error \(\sum_i (f_\theta(\mathbf{x}(s_i)) - y_i)^2\).
  4. Use: At test time, use \(\hat{h}(s)\) as the heuristic in A*.

Models used in practice

  • Linear regression: Fast, interpretable. \(\hat{h}(s) = \mathbf{w}^T \mathbf{x}(s)\). Risk: may predict negative values or overestimate.
  • Neural networks: Can capture non-linear interactions. Used to learn heuristics for combinatorial search (NeurIPS 2023 work).
  • Decision trees / regression forests: Good for tabular features.

Admissibility risk with learned heuristics

A learned model may overestimate \(h^*(s)\) on some states, breaking admissibility. Two remedies:

  • Use \(\hat{h}(s) = \min(\hat{h}(s), h_{\text{safe}}(s))\) where \(h_{\text{safe}}\) is a known admissible heuristic.
  • Train with a loss that penalises overestimates more than underestimates (asymmetric loss).

Why this matters (NeurIPS 2023 context)

Recent work uses neural networks to learn \(h^*\) directly from millions of solved instances. This is especially powerful when no good relaxation exists вҖ” the network discovers structure that hand-crafted heuristics miss. The ACI course uses this as a use case bridging supervised learning and search.

Closed-Book: Memorise This

Key Definitions

  • Admissible: \(h(n) \leq h^*(n)\) — never overestimates. Guarantees A* optimality.
  • Consistent: \(h(n) \leq c(n,a,n') + h(n')\) — triangle inequality. Implies admissible.
  • Dominant: \(h_2 \geq h_1\) everywhere (both admissible) → A* with \(h_2\) expands \(\leq\) nodes.

Designing Heuristics

  • Relaxed problem: Remove constraints → optimal relaxed cost is admissible for original.
  • 8-puzzle relaxations: \(h_1\) = misplaced tiles (remove adj+empty), \(h_2\) = Manhattan (remove empty). \(h_2\) dominates \(h_1\).
  • Pattern databases: Precompute exact subproblem costs → tighter lower bounds. Additive PDBs: split tiles into disjoint groups, sum their PDB costs.
  • Combined: \(h(n) = \max(h_1, h_2, \ldots)\) — still admissible, dominates all.
