EXAM INTEL

Mid-Semester Exam — Key Pointers

30 marks 2 hours Subjective — no MCQ No Python/code — pseudocode fine Closed book Online in exam centre Scope: CS1–CS8
Topic What to master Prof's pointer (from class)
Agents, PEAS & Environments
 PEAS spec (Performance, Environment, Actuators, Sensors); 6 environment properties (fully/partially observable, deterministic/stochastic, episodic/sequential, static/dynamic, discrete/continuous, single/multi-agent); 5 agent types in complexity order; Problem-Solving Agent formulation
"Don't hard-code examples — apply PEAS to any new domain given." Likely 1 scenario-based PEAS question.
Informed Search — A* & GBFS
 f(n) = g(n) + h(n); admissibility h(n) ≤ h*(n); consistency h(n) ≤ c(n,n') + h(n'); relaxed constraints method for heuristic design; dominance; GBFS vs A* trade-offs (optimal/complete)
"Leaf/utility values will either be given directly in the tree OR a formula will be given — compute them from that."
Local Search
 Hill climbing variants (steepest ascent, first-choice, random restart); why it fails (local maxima, plateaus, ridges); simulated annealing escape mechanism; beam search (keep k states)
Weekly exercises had local search traces — practice tracing on a state-space diagram. Speed is the risk here.
Evolutionary Algorithms — GA, ACO, NEAT
 Generational vs Steady-State GA (steady-state replaces only a few individuals — not the whole population); crowding/niching for diversity; ACO pheromone rule: τij ← (1−ρ)·τij + Δτij; NEAT: historical markings, speciation, why it solves the permutation problem; connection to NAS/DARTS
"NEAT for NAS — these two are very important." Steady-state vs generational is a common exam distinction.
Adversarial Search — Minimax & Alpha-Beta
 Game tree expansion (3+ levels); MAX maximises, MIN minimises; back-propagation; static evaluation function (formula given in question); alpha-beta: prune when α ≥ β; show pruned branches with a cross in exam; O(bm/2) vs O(bm) complexity
"Alpha-beta is the same as Minimax — it just prunes. Show the crossed-out branches clearly in exam." Likely the highest-mark question.
MCTS
 4 phases: Selection (UCB1), Expansion, Simulation (random playout), Backpropagation; UCB1 formula: Q(n)/N(n) + c√(ln N(parent)/N(n)); when MCTS preferred over Minimax (very large branching factor, no good static eval)
"MCTS is a heuristic search algorithm — use when branching factor is too large for full Minimax." Compared directly to Alpha-Beta in class.
⚠ Marking rule (from class): "If you have done a wrong thing, why would we keep on evaluating? The first mistake, we stop." — Prof. Parthasarathy P D.  Write only what you are certain about. For game trees, if leaf calculation is wrong, back-propagation marks are also lost. Accuracy beats breadth.
