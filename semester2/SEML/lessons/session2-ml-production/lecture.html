Session 2 · Overview

From Research to Reality

A model that achieves 94% accuracy in a Jupyter notebook is not a product. This session is about everything that happens between "model works in the lab" and "system reliably serves users in production."

The chef analogy: A chef perfects a recipe for six dinner guests. Then a restaurant chain wants to serve it to 10,000 people per day across 50 cities. The recipe is only a tiny part of the problem. The same is true of ML models.

Production Challenges

What Actually Goes Wrong in Production

Six categories of production failure that every ML engineer will encounter.

Data Quality Drift

Training data and production data diverge over time. The model sees patterns it was never trained on. Performance silently degrades.

Latency & Cost

Research model takes 3 seconds; product requires 200ms. GPU cost at scale is 40× the lab estimate. Real-time serving is engineering-hard.

Pipeline Fragility

Preprocessing scripts break when upstream data format changes. One schema change cascades through the entire pipeline.

Team Silos

ML researchers and software engineers don't share vocabulary, tools, or processes. Handoff is painful and error-prone.

Monitoring Gaps

Performance degrades for weeks before anyone notices. No alerting on prediction quality. Only "is the server up?" is monitored.

Fairness Failures

Model performs 15% worse on non-native English speakers. No fairness metrics were tracked. Only discovered after launch.

AI Paradigm 1

Predictive AI

Predictive AI uses past data to forecast future outcomes. The model learns historical patterns and applies them to new inputs.

The detective analogy: A detective studies evidence from past cases and says "based on these patterns, the suspect is probably the butler." Predictive AI answers specific questions with a number or a label.

What it outputs

A number (predicted churn probability: 0.73), a label (spam / not spam), or a ranking (top 10 recommendations).

What it needs

Historical labelled data. Clear target variable. Features that correlate with the outcome.

Examples

Credit scoring, demand forecasting, fraud detection, churn prediction, disease risk.

Limitation

Can only interpolate within the training distribution. Cannot handle genuinely novel situations.
AI Paradigm 2

Generative AI

Generative AI creates new content by learning patterns from vast training data. A foundation model takes a natural language instruction and produces a digital artefact.

The novelist analogy: A detective (predictive AI) analyses facts to reach a conclusion. A novelist (generative AI) reads a thousand mystery novels and then writes a new one — with entirely new characters and plot.
Foundation Model architecture: Natural Language Instruction → Foundation Model → Digital Artefact (text, code, image, audio, video)

Text Generation

GPT-4, Claude 4, Gemini — write, summarise, translate, answer questions, generate code

Image Generation

DALL-E, Stable Diffusion, Midjourney — create images from text descriptions

Code Generation

GitHub Copilot, Cursor, Claude Code — auto-complete, explain, refactor code

Multimodal

GPT-4V, Gemini 1.5 — see images + text; generate across modalities
AI Paradigm 3

Agentic AI: Act, Reason, Repeat

Agentic AI goes beyond generating a response — it takes sequences of real-world actions to achieve a goal, using tools and feedback loops.

$$\text{Agentic AI} = \text{Generative AI} + \text{Reasoning} + \text{Tools} + \text{Feedback Loops}$$

Interactive: The Agentic Loop

Personal assistant analogy: Predictive AI = weather app (ask, get answer). Generative AI = ghostwriter (drafts your email). Agentic AI = a human assistant who reads your brief, books the flight, writes the agenda, sends the invite, and revises when the venue changes.

SE4ML Challenge: Complexity

28 models × multiple versions × multiple sensors = combinatorial test surface. A bug in model #3 corrupts input to model #17 silently.

SE4ML Challenge: Test Coverage

How do you write a unit test for "correctly identify a pedestrian"? Model outputs are probabilistic, not deterministic.

SE4ML Challenge: Safety

A wrong prediction doesn't cause a UI glitch — it may cause a collision. Safety-critical ML requires formal verification and fail-safe fallbacks.

SE4ML Challenge: Maintenance

Each of 28 models must be independently versioned, evaluated, and retrained. A change to model #1 may cascade to all downstream models.

Wrap-up

Key Takeaways from Session 2

The Production Gap

Research accuracy ≠ production value. The gap between them is mostly software engineering, not ML.

Systems Thinking

The model is a tiny island. Most of the ML system is data pipelines, serving infra, monitoring, and configuration.

Three Paradigms

Predictive (forecast from history) → Generative (create new content) → Agentic (act autonomously with tools).

Real-World Complexity

Apollo's 28 models and Microsoft's 9 stages show that industrial ML is a software engineering challenge first, and an ML challenge second.

Professor's Notes from Webinar

These highlights are drawn directly from Prof. Shreyas Suresh Rao's spoken transcript of the Session 2 webinar. Items marked Exam Signal are things he explicitly flagged; Key Concept items are stressed definitions; Professor's Comparison items are A vs B contrasts he drew.

⚑ Exam Signal

Three changing variables in ML (flagged as "a very important observation"): In traditional SE, code is everything. In data science, code + data. In machine learning, you have three changing variables: code, data, and model. "Machine learning is slightly more complex in nature because there are three changing variables."

⚑ Exam Signal

SE vs ML — Deterministic vs Probabilistic (flagged as "a very important one"): SE applications are deterministic — "did the payment go through?" — functional correctness focus. ML is "more experimental in nature" and probabilistic — "accuracy of the model becomes very important." SE focus = functional correctness; ML focus = model accuracy.

★ Key Concept

Definition of a Model: "A model is a trained algorithm which is used to make predictions or decisions." The ML pipeline has two halves: (1) managed data → train model (model training); (2) deploy model → give real data → get prediction (model inference).

⇄ Professor's Comparison

Predictive AI vs Generative AI vs Agentic AI: Predictive AI (1990s–2018) = supervised learning, moderate training, predicts future outcomes. Generative AI (2018+) = trained on vast web data, generates new content; "under the hood, generative AI uses prediction" (next-word prediction). Agentic AI = orchestrates multiple agents autonomously. "ChatGPT is an example of generative AI. Claude Opus in your IDE is an example of agentic AI."

⇄ Professor's Comparison

Fine-Tuning vs RAG: "If you have FAQs (structured Q&A), fine-tuning is best — but human verification is important. If you have unstructured data like PDF or PPT, RAG is definitely better." Fine-tuning requires JSONL format; RAG and fine-tuning are separate concepts — "we are using RAG to fine-tune our models" is a common misconception the professor explicitly corrected.

⇄ Professor's Comparison

Foundation Model vs LLM vs Large Vision Model: Foundation models are trained on almost all web data. LLM is a foundation model specifically for language (e.g., early GPT-3.5 — text only). Large vision model covers audio/video/images. "Foundation model and LLM are interchangeably used in scientific literature, but foundation model is the broader term."

★ Key Concept

Moving from research prototype to production: "Moving from research prototype to production system requires significant engineering beyond machine learning. This is where software engineering principles and practices become very important." CI/CD, Git, Docker, Kubernetes all originated in SE and are now applied to ML deployment.

★ Key Concept

8 components alongside the ML model in production: A deployed ML system needs: UI, database (relational/NoSQL/vector), cloud processing, logging, monitoring, plus the ML model. "Logging — very important. Monitoring." Not just the model in isolation — 8 components work together.
