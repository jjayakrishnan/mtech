The RL Loop

You already know the core idea: an agent takes actions and gets rewards. Let's make that precise enough to handle any exam question.

Everyday picture

Think of training a dog. You don't give step-by-step instructions. You just give a treat when it does something good. The dog figures out on its own which behaviours lead to treats. That's RL: learn from feedback, not from instructions.

The RL loop cycles through four things at every time step:

The Agent–Environment Loop — Step Through It

Press "Next Step" to begin
The loop in one sentence

At each time step t, the agent sees state St, picks action At, and receives reward Rt+1 and next state St+1 from the environment — then repeats.

The 5 Elements You Must Know

Every exam question in this course is about one or more of these five things. The wristband scenario from the EC-2 paper is used throughout so the analogy is exam-ready.

S
State

A snapshot of the world at this moment.
Wristband: "User activity = HIGH"

A
Action

What the agent can choose to do.
Wristband: Pick Mode A, B, or C

R
Reward

Immediate numeric feedback from the environment after one action.
Wristband: +2 for accurate reading at low battery cost

π
Policy

The agent's decision rule: given this state, which action to pick. It is the agent's "brain".

V
Value Function  (the most important one)

The expected total future reward from a state, following some policy. This is what the agent is truly optimising — not just right now, but long-run. The exam tests this distinction constantly.

Exam tip — Q(s, a)

The exam also uses Q(s, a) — value of taking action a in state s, then following policy π. Q is just V with the action made explicit. You'll need this for Monte Carlo and policy-improvement questions.

RL vs Supervised Learning

Question 1a of your EC-2 paper asks this directly — worth 1.5 marks for one clean answer.

Supervised LearningReinforcement Learning
What you have (input, correct label) pairs (state, action, reward) tuples
Who says what's right A human provides the correct answer Nobody — only a reward signal
What the model learns Correct output for each input Which actions lead to high future reward
Wristband example A doctor labels every hour: "for HIGH activity, Mode B was correct" Wristband tries modes, observes rewards — figures out Mode B is best for HIGH activity on its own
Exam-ready answer for Q1a

RL fits because there is no labelled dataset telling the wristband the correct mode — the environment only says how good the outcome was, not what the correct action was. RL learns from evaluative feedback (reward); supervised learning learns from instructive feedback (correct labels).

Immediate Reward vs Long-Term Value

Also tested in Q1a — 1 mark for a single clear distinction.

Everyday picture

Staying up all night to finish a project (immediate reward: task done). But you're exhausted tomorrow, make mistakes, lose focus for a week. The value of "stay up all night" is much lower than the immediate reward suggests.

One-line distinction — memorise this

Immediate reward Rt+1 is the single number you receive after one action. Long-term value V(s) is the expected total discounted reward you can accumulate from state s onwards, following policy π.

Wristband example worked through

Mode C gives high immediate reward (+3, very high accuracy). But Mode C drains the battery — future rewards drop because the battery temperature rises and power-hungry modes become less effective. So Mode C has low long-term value. The agent needs to learn this distinction.

The Discount Factor γ

Future rewards are discounted so nearby rewards matter more than distant ones:

\( G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \)

γ is between 0 and 1. Slide it below to see what happens to future rewards.

🎚 Adjust γ — watch how future rewards shrink

γ = 0.90

Exam tip — γ = 0.01 (Q2d in your EC-2 paper)

γ ≈ 0 means the agent is myopic — it ignores everything after the next reward. In the ICU ventilator scenario, this means the AI only cares about immediate SpO₂ change, ignoring whether the patient is actually stable long-term. γ close to 1 means the agent is patient and cares about long-run outcomes.

γ valueAgent behaviourRisk
≈ 0Myopic — only cares about next rewardIgnores long-term consequences
≈ 1Far-sighted — distant rewards matter nearly as muchMay not converge in continuing tasks
0.9 (typical)Balanced — weights near rewards more, but still plans ahead—

Quick-Fire Quiz

These mirror the exact style of your EC-2 paper. Answer before revealing.

Q1  [1.5 marks]   The wristband must choose Mode A/B/C to maximise patient benefit. No labelled dataset of correct modes exists. Give one reason why RL fits better than supervised learning.
Correct. The key phrase that earns marks: "no correct labels — only evaluative reward." That one distinction separates RL from SL.
Q2  [1 mark]   In one line: what is the difference between immediate reward Rt+1 and long-term value V(s)?
Correct. "Expected total discounted reward from this state onwards" is the phrase that wins the mark.
Q3  [1.5 marks]   γ = 0.01 is used for an ICU ventilator RL agent. What is the main risk to patient safety?
Correct. γ ≈ 0 = myopic. The agent chases the next reward and ignores downstream consequences — dangerous in a medical setting.
Q4  [1 mark]   Which element of RL captures the agent's decision-making rule — i.e., given a state, which action to pick?
Correct. π is the brain. It maps states → actions. Everything else (reward, value) helps shape π, but π is what the agent actually uses to decide.
Q5  [1 mark]   Mode C gives the wristband a high immediate reward (+3) but drains the battery, reducing future rewards. What does this illustrate?
Correct. This is the core tension in RL: greedy short-term choices can be bad long-term. The value function exists precisely to capture this.

+12 Analogy — RL Framework

Imagine you're playing a video game. You are the agent. The screen (health, position, enemies) is the state. Pressing buttons are actions. The score that flashes is your reward. Your strategy — which button to press in each situation — is the policy.

The discount factor \(\gamma\) is like caring more about coins you collect right now than coins promised 10 levels later — because you might die before reaching them. Setting \(\gamma=0.9\) means a reward in 10 steps is worth only \(0.9^{10} \approx 35\%\) of an immediate reward.

Try It Yourself

S&B Ex 1.1 — Self-Play [4 marks]

Suppose the tic-tac-toe reinforcement learning agent played against itself, with both sides learning simultaneously. What do you think would happen? Would it converge to a good policy for playing against a random (or fixed) opponent?

Solution:

Both players update toward beating each other, so the learned values reflect win probabilities against the current self, not against an external random player. The two sides create a non-stationary moving target for each other.

The system may converge to a symmetric equilibrium (neither player can improve against the other), but this policy could be suboptimal against a fixed random opponent — it may over-specialise in exploiting the other learner’s weaknesses rather than learning universally good play.

▶  Key formula & step-by-step
Temporal-difference value update: $$ V(S_t) \leftarrow V(S_t) + \alpha\bigl[V(S_{t+1}) - V(S_t)\bigr] $$
  1. What values represent: Each board position stores an estimated win probability. Values start at 0.5 (draws), 1 (winning position), 0 (losing position).
  2. How a backup happens: After each move the agent nudges \(V(S_t)\) a fraction \(\alpha\) of the way toward \(V(S_{t+1})\). Over many games estimates improve.
  3. Why self-play is non-stationary: Both X and O are updated simultaneously. When O improves, X's environment changes — so the values X is converging toward keep shifting.
  4. Answer: The values may converge (to a Nash equilibrium against itself) but are not guaranteed to be optimal against a random player. The agent over-specialises against its own weaknesses.
▶  Key formula & step-by-step
Greedy action selection: $$ A_t = \arg\max_a Q_t(a) $$ Greedy always picks the action with the highest current estimate — no exploration.
  1. Initial estimates are wrong: After very few games all \(Q_t(a)\) values are inaccurate. Greedy locks onto whichever action happens to look best first.
  2. Lock-in effect: Once an action is not chosen its estimate never updates. Good actions that started with a low estimate stay forever undervalued.
  3. Greedy vs ε-greedy: ε-greedy forces occasional random moves, ensuring every action gets tried and its value estimated correctly over time.
  4. Answer: Playing greedily from move 1 performs worse in the long run — it gets permanently stuck on a sub-optimal strategy because it never discovers better moves.
▶  Key formula & step-by-step
Two learning targets: \( \text{Case A: learn from all moves} \Rightarrow \text{target} = V(S_{t+1}^{\text{both}}) \) \( \text{Case B: learn only from greedy opponent moves} \Rightarrow \text{target} = V(S_{t+1}^{\text{greedy}}) \)
  1. Case A — learn from all moves: When the opponent makes a random exploratory move, we back up that state's value. But that value reflects a random action, not optimal play — it contaminates our estimates.
  2. Case B — learn from greedy moves only: We only update when the opponent plays greedy. The target \(V(S_{t+1})\) is what we expect under optimal opponent play — a cleaner signal.
  3. Which wins more? Case B, because the learned values accurately reflect what happens when both sides play optimally — which is exactly what the greedy policy will do at test time.
  4. Answer: Option B (learn only from greedy moves). Using exploratory moves as training targets adds noise because those moves are deliberately sub-optimal.
S&B Ex 1.3 — Greedy Play [4 marks]

Suppose the reinforcement learning player was greedy — it always selected the move leading to the highest-valued state and never made exploratory moves. Might it learn to play better, or worse, than a non-greedy player? What problems might arise?

Solution: Worse. Without exploratory moves, the agent never revisits states it currently ranks as low-value. If those states happen to be good (but were mis-estimated early on), the agent never corrects the error.

Value estimates for un-chosen states never get updated, so the agent can become locked in a suboptimal policy. Exploration is essential for building accurate value estimates across the full state space.

S&B Ex 1.4 — Learning from Exploration [5 marks]

Suppose learning updates occurred after all moves, including exploratory moves (not just the greedy ones). Assuming the step-size \(\alpha\) is reduced appropriately over time:

(a) What two sets of probabilities are computed when we do and when we do not learn from exploratory moves?
(b) Which would lead to more wins against an imperfect fixed opponent?

Solution (a):

  • Learning from exploratory moves: Values converge toward win probabilities under the mixed policy (greedy + exploratory moves). These reflect the true probabilities under the actual behaviour policy.
  • Not learning from exploratory moves: Values converge toward win probabilities under the pure greedy policy only.

Solution (b): Not learning from exploratory moves is better — those estimates correctly reflect how the greedy policy performs. The agent eventually plays greedily using accurate values that weren’t polluted by sub-optimal exploratory actions.
