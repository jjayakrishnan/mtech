2. Topic Reference

Topic 1

Multi-Armed Bandit

  • No state — just actions + rewards
  • UCB: Q(a) + c·√(ln t / N(a))
  • EWMA: Q(a) ← Q(a) + α·[r − Q(a)]
  • ε-greedy: explore with prob ε
  • ε-case: random action chosen
  • α=1 → only last reward matters
Very High Frequency
L2: MAB vs MDP L4: Action-Value Methods L5: UCB
Topic 2

MDP Formulation

  • Tuple: (S, A, P, R, γ)
  • Bellman: Vπ(s) = Σₐ π(a|s) · Σₛ' P·[R + γV(s')]
  • Value Iteration: V ← max over actions
  • Synchronous VI: update all states at once
  • Episodic vs continuing tasks
  • γ≈0: myopic; γ≈1: far-sighted
Very High Frequency
L6: Finite MDP L7: Bellman Equations L3: Value Iteration
Topic 3

Monte Carlo Methods

  • First-visit: only first occurrence in episode
  • G = r₁ + γr₂ + γ²r₃ + … (work backwards)
  • V(s) ← V(s) + (1/n)[G − V(s)]
  • Q update → greedy policy improvement
  • Off-policy: importance sampling ratio ρ
  • Weighted IS: ρ·G / Σρ
Very High Frequency
L9: Monte Carlo
Topic 4

Policy Gradient

  • REINFORCE: θ ← θ + α·G·∇ln π(a|s)
  • Softmax policy: π(a|s) = σ(θᵀφ(s,a))
  • Baseline: reduces variance (subtract b from G)
  • Without baseline: high variance updates
  • A2C: Actor picks actions; Critic estimates V
  • Advantage: A(s,a) = Q(s,a) − V(s)
High Frequency
L8: Dynamic Programming
Topic 5

MAB vs MDP Classification

  • MAB: no state transitions, i.i.d. rewards
  • MDP: current action affects future states
  • Key test: does today's action change tomorrow's context?
  • Episodic: clear start/end (e.g., ICU stays)
  • Continuous: no terminal state
Medium Frequency
L2: MAB vs MDP L1: RL Framework
Topic 6

DQN

  • Experience replay: breaks correlation
  • Target network: separate frozen network for TD target
  • Loss: (r + γ·max Q_target(s') − Q(s,a))²
  • Deadly triad: FA + bootstrapping + off-policy
Medium Frequency
L8: Dynamic Programming
Topic 7

MCTS & AlphaGo/Zero

  • UCB for trees: Q(s,a) + c·√(N(s)/N(s,a))
  • 4 phases: Select → Expand → Simulate → Backprop
  • AlphaGo: trained on human games + self-play
  • AlphaZero: self-play only → more general
Medium Frequency
L5: UCB (foundation)
Topic 8

Imitation Learning

  • BC loss: L = −Σ log π_θ(aᴱ|s)
  • Failure: covariate shift (unseen states at test)
  • DAGGER: aggregate expert labels on policy's own states
  • Advantage: policy learns to recover from mistakes
Lower Frequency
L9: Monte Carlo (related)

3. Key Formulas — Cheat Sheet

UCB Action Selection (MAB)
$$ a^* = \arg\max_a \left[ Q(a) + c \cdot \sqrt{\frac{\ln t}{N(a)}} \right] $$ \(t\) = total time steps so far  ·  \(N(a)\) = times \(a\) was chosen
Exponential Recency-Weighted Average (EWMA)
$$ Q_{n+1}(a) = Q_n(a) + \alpha \left[ R_n - Q_n(a) \right] $$ \(\alpha \in (0,1]\)  ·  \(\alpha = 0.5\) common in exams
Bellman Expectation (Policy Evaluation)
$$ V_\pi(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a)\left[ R(s,a,s') + \gamma V_\pi(s') \right] $$
Bellman Optimality (Value Iteration)
$$ V^*(s) = \max_a \sum_{s'} P(s'|s,a)\left[ R + \gamma V^*(s') \right] $$
Discounted Return (MC)
$$ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots $$ Compute backwards from terminal state
First-Visit MC Update
$$ V(s) \leftarrow V(s) + \frac{1}{n}\left[ G - V(s) \right] $$ \(n\) = number of times \(s\) has been visited so far
Importance Sampling Ratio
$$ \rho = \prod_t \frac{\pi(a_t|s_t)}{b(a_t|s_t)} $$ \(\pi\) = target policy  ·  \(b\) = behaviour policy
Policy Gradient Update (REINFORCE)
$$ \theta \leftarrow \theta + \alpha (G - b)\, \nabla_\theta \ln \pi_\theta(a|s) $$ \(b\) = baseline (often \(V(s)\) or a constant)
Linear Softmax Policy
$$ \pi(a|s) = \frac{\exp(\theta^\top \phi(s,a))}{\sum_{a'} \exp(\theta^\top \phi(s,a'))} $$ \(\phi(s)\) = feature vector of state \(s\)
UCB for MCTS (Tree)
$$ \text{UCB}(s,a) = Q(s,a) + c\sqrt{\frac{\ln N(s)}{N(s,a)}} $$ \(N(s)\) = parent visits  ·  \(N(s,a)\) = child visits
DQN Loss
$$ \mathcal{L}(\theta) = \mathbb{E}\!\left[\left(r + \gamma \max_{a'} Q_{\theta^-}(s',a') - Q_\theta(s,a)\right)^2\right] $$ \(\theta^-\) = frozen target network weights
Behavior Cloning Loss
$$ \mathcal{L}_{\text{BC}}(\theta) = -\frac{1}{N}\sum_i \log \pi_\theta(a_i^E \mid s_i) $$ \(a^E\) = expert action at state \(s\)

(c) How sensitive is the dispatch choice to r? Comment on the effect of α on the agent's memory of past rewards. [2 Marks]
Show Answer — Part (c)
  • UCB selects C over B only if r < −3.92 — the choice is robust to all realistic values of r.
  • EWMA: weight of kth-last reward = αᵏ. At α=0.5 each older reward contributes half as much as the next newer one.
  • α=1 → only the latest reward matters (maximum sensitivity to r). α→0 → old history dominates (insensitive to r).
  • α=0.5 provides moderate sensitivity — recent performance matters but cannot alone override history.

(d) [1 Mark] What happens if baseline b is not used? Explain w.r.t. the AI tutor only.
Show Answer — Part (d)

Without b, every positive return (G > 0) reinforces the chosen action — even marginally positive ones. The tutor could lock onto Hard tasks that gave small positive returns, ignoring that Medium tasks might give much higher returns. This high-variance updating destabilises the policy.

(e) [2 Marks] A2C is applied. State the role of Actor and Critic specifically in the AI tutor scenario.
Show Answer — Part (e)
ComponentIn the AI Tutor
Actor (policy π)Selects which difficulty level (E/M/H) to recommend at each tutoring session
Critic (value V)Estimates the student's expected long-term learning outcome from their current mastery/engagement state — provides a baseline so the actor gets a fair signal
Advantage A\(A = r + \gamma V(s') - V(s)\): tells the actor whether that difficulty choice led to better (+) or worse (−) outcomes than average for this student type

(d-1) [1.5 Marks] Write the behavior cloning loss function for policy π_θ(a|s). State one major failure mode when the policy encounters unseen states.
Show Answer — Part (d-1)
$$ \mathcal{L}_{\text{BC}}(\theta) = -\frac{1}{N}\sum_i \log \pi_\theta(a_i^E \mid s_i) $$

Failure — Covariate Shift: At test time the policy visits states not in the expert's demonstrations. With no training signal there, errors compound: one mistake → unseen state → worse mistake → further unseen states.

(d-2) [1.5 Marks] How does DAGGER modify data collection vs behavior cloning? Why does this help the policy handle its own mistakes?
Show Answer — Part (d-2)
AspectBehavior CloningDAGGER
Data sourceFixed expert demos onlyRun current policy → collect states → ask expert to label them
Training distributionExpert state distributionExpert + policy's own visited states (mixed, growing)
Handles mistakes?No — unseen recovery states cause compounding driftYes — policy sees and learns from states it actually reaches

DAGGER directly solves covariate shift: iteratively adding the policy's own trajectory distribution means it learns to recover from mistakes, not just imitate perfect execution.

Top 3 traps students fall into 1. MCReturn direction: Compute G backwards from terminal, not forwards.
2. First-visit MC: Only use the first time a (state, action) appears in the episode for the update.
3. UCB denominator: Use ln(t) where t = total time steps so far, NOT just the count for that arm.
