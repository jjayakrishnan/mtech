The One Classification Test

The exam gives you a real-world scenario and asks: is this a Multi-Armed Bandit or a Finite MDP? One question decides it — every time.

Does the action I take TODAY change the state or reward I get TOMORROW?
NO → Multi-Armed Bandit
YES → Finite MDP

That's the whole rule. Everything else — the terminology, the formulas — is built on top of this one distinction. The rest of this lesson explains why, with the exact wristband scenario your exam used.

Multi-Armed Bandit (MAB)

Everyday picture

You're at a casino with several slot machines (bandits). Each machine has its own hidden payout rate. Pulling one machine's arm doesn't change how the other machines behave. Your job: figure out which machine pays best, and play it.

In RL language: MAB is a non-associative setting. "Non-associative" means you don't need to associate different actions with different situations — you're always in the same situation (or situations are independent of what you did before).

Formal definition

A k-armed bandit problem has k actions (arms). At each time step you choose one action and receive a reward sampled from that action's reward distribution. Crucially: your choice today has zero effect on tomorrow's reward distributions.

The key properties of MAB

  • No state — or equivalently, there is only one state. You're always in the same situation.
  • No transitions — actions don't move you between states.
  • Actions are independent — pulling arm A doesn't change what arm B will give you next time.
  • Goal: maximise total reward by learning which arm has the highest expected reward — the action value Q(a).
Course terminology — use this exact phrase

Your course calls MAB the "non-associative setting" of RL. Supervised review Q2 asks this directly: "Why do you say Multi-armed bandit is RL problem in non-associative setting?" Answer: because there is no state to associate actions with — every pull is independent of history.

Finite MDP

Everyday picture

Now imagine you're playing chess. The move you make now changes the board state — and that new board state determines what moves are available and what rewards (winning/losing) are possible next. Every action has consequences that ripple forward.

MDP stands for Markov Decision Process. "Finite" means the number of states and actions is finite (countable). It's the full RL setting.

5 components — memorise these for the closed-book exam

An MDP is defined by (S, A, P, R, γ):

  • S — finite set of states
  • A — finite set of actions
  • P — transition probability: p(s′, r | s, a) = probability of landing in state s′ with reward r, given you took action a in state s
  • R — reward function
  • γ — discount factor (0 ≤ γ ≤ 1)

The Markov Property

This is what makes an MDP an MDP. The future depends only on the current state, not on the entire history of how you got there.

\( P[S_{t+1} \mid S_t] = P[S_{t+1} \mid S_1, S_2, \ldots, S_t] \)

In plain words: to predict the future, knowing where you are now is enough. Knowing the path you took to get here adds nothing.

Side-by-Side Comparison

Multi-Armed Bandit

  • Non-associative setting
  • Single state (or independent situations)
  • No state transitions
  • Action today ≠ affect tomorrow
  • Goal: find best arm Q(a)
  • Algorithms: ε-greedy, UCB, weighted avg

Finite MDP

  • Associative / full RL setting
  • Multiple states S
  • Actions cause state transitions P(s′|s,a)
  • Action today shapes tomorrow's state
  • Goal: find optimal policy π*
  • Algorithms: Value Iteration, Policy Iteration, MC

Greedy & ε-Greedy Action Selection

Greedy always picks the action with the highest current estimate:

$$ A_t = \arg\max_a Q_t(a) $$

ε-greedy usually exploits, but with probability ε explores uniformly at random:

$$ A_t = \begin{cases} \arg\max_a Q_t(a) & \text{with prob } 1-\varepsilon \\ \text{random action} & \text{with prob } \varepsilon \end{cases} $$

Probability a specific action is chosen (k total actions):

$$ P(\text{greedy chosen}) = (1-\varepsilon) + \frac{\varepsilon}{k} $$

Worked Example — From Your Actual Exam (EC-2, Q1b)

The wristband scenario had two cases. This is exactly how to write the answer.

Case 1: The user's activity pattern is independent of the modes chosen — choosing Mode A or B today does NOT affect future rewards.
Case 1 Answer — MAB

Classification: Multi-Armed Bandit.
Justification: The user's activity (and therefore future rewards) is independent of the mode chosen. Selecting Mode A or B today has no effect on tomorrow's state or reward distribution. Since there are no state transitions driven by the action, this is a non-associative setting — a MAB problem.

Case 2: If the wristband samples in Mode C for too long, battery temperature rises, reducing future rewards for power-hungry modes.
Case 2 Answer — Finite MDP

Classification: Finite MDP. Yes, the added dependency changes the problem class.
Justification: The action taken today (choosing Mode C) directly affects future states (battery temperature state) and future rewards (power-hungry modes become less rewarding). There is now a state transition driven by the agent's action — this satisfies the MDP structure. The Markov property holds: future rewards depend on the current battery-temperature state, not the full history.

Exam writing tip

Always give the classification then the justification in one tight sentence that mentions either "no state transition" (MAB) or "action changes future state/reward" (MDP). That two-part structure is what earns full marks on 1-mark classification questions.

Episodic vs Continuing Tasks

This was tested in EC-2 Q2c (ICU scenario, 1.5 marks). It's a sub-distinction within MDP tasks.

Episodic Task

  • Has a terminal (absorbing) state
  • Interaction breaks into episodes
  • Each episode ends and resets
  • ICU example: patient stabilises after 4 consecutive Optimal readings → episode ends
  • Other examples: chess game, maze navigation

Continuing Task

  • No terminal state — runs forever
  • Agent must keep making decisions indefinitely
  • Must discount (γ < 1) or returns become infinite
  • Examples: stock trading bot, a server managing requests
Exam-ready answer for EC-2 Q2c (ICU is episodic)

The ICU task is episodic. The problem states a clear terminal condition: once SpO₂ is Optimal for four consecutive recordings, the patient stabilises and the task ends. This natural stopping point defines episodes — the interaction resets for the next patient case. A continuing task would have no such terminal state.

MEMORISE THIS — Closed Book

Write these out from memory before the exam. No formula sheet allowed.

MDP 5 components (in order)

\[ (\mathcal{S},\; \mathcal{A},\; P,\; R,\; \gamma) \]

Transition dynamics function

\[ p(s', r \mid s, a) \;=\; \Pr\{S_t = s',\, R_t = r \mid S_{t-1} = s,\, A_{t-1} = a\} \]

Markov property (state is sufficient summary of history)

\[ \Pr[S_{t+1} \mid S_t] \;=\; \Pr[S_{t+1} \mid S_1, S_2, \ldots, S_t] \]

Discounted Return

\[ G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \]

Classification one-liner (use these exact words)

  • MAB: "Actions do not affect future state or reward — non-associative setting"
  • MDP: "Action in state \(s\) transitions to \(s'\) with probability \(p(s'|s,a)\) — Markov property holds"
  • Episodic: "Has a terminal state; interaction resets into episodes"
  • Continuing: "No terminal state; must use discounting (\(\gamma < 1\))"

Scenario Classifier — Practice

Classify each scenario before revealing the answer. Use the one-question test.

Click MAB or MDP for each scenario, then see why.

Scenario A: A doctor recommends either Drug X or Drug Y to each new patient. Each patient's outcome (recovery or no recovery) is completely independent — giving Drug X to Patient 1 has no effect on what Patient 2 will respond to.

Scenario B: A robot's battery is Low, Medium, or High. If the robot Continues Working, it consumes battery; if it Recharges, it restores battery. Future rewards depend on the battery level at the next step.

Scenario C: A pharma platform recommends one of four interventions (SMS, Teleconsult, Lab reminder, Pharmacist call) to patients. The reward (adherence score) from each patient visit is observed independently. What works for one patient today doesn't change the system's reward distribution for tomorrow.

Scenario D: An ICU ventilator AI adjusts pressure (increase/decrease/maintain) to move a patient's SpO₂ toward Optimal. The patient's current oxygen level (lowO2 / Optimal / highO2) is the state. Each action changes the patient's state with some probability.

Exam-Style Quiz

Same format as the mid-semester paper. Closed-book conditions — no peeking at the lesson above.

Q1  [1 mark]   A stock-trading app recommends Buy, Hold, or Sell each day. The market conditions tomorrow are influenced by the large trades made today. Classify this as MAB or Finite MDP with one justification.
Correct. The tell is "market conditions tomorrow are influenced by trades today" — action → state change → future reward change. That's an MDP.
Q2  [1 mark]   In the pharma platform question (EC-2 Q3), why is the intervention recommendation system correctly modelled as a MAB?
Correct. Number of arms ≠ reason for MAB. Independence (no state transitions) is the reason.
Q3  [1 mark]   What is the Markov property, in one sentence?
Correct. "Current state is a sufficient summary of history" — knowing where you are now is all you need to predict the future.
Q4  [1.5 marks]   The ICU ventilator task ends once the patient stabilises (4 consecutive Optimal readings). Is this episodic or continuing? Justify in ≤30 words.
Correct. Terminal state = episodic. The γ < 1 argument is about why discounting is needed for continuing tasks — it doesn't define episodic.
Q5  [1 mark]   Name the 5 components of a Finite MDP in order (write from memory).
Correct: (S, A, P, R, γ). The P is the dynamics — p(s′, r | s, a). This is the component students most often forget.

+12 Analogy — MAB vs MDP

MAB — the Slot Machine: You're at a casino with 10 slot machines. Each pull gives a random reward. Your position doesn't change — you're always at the same machines. Only question: which machine to pull? That's MAB.

MDP — the Chess Board: Every move you make changes the board. Your future choices depend entirely on the new board state, not on how you got there (Markov property). You must plan ahead through a sequence of states. That's MDP.

Key test: "Does my action change the situation I face next?" — Yes → MDP. No → MAB.

Try It Yourself

S&B Ex 2.1 — ε-Greedy Probability [3 marks]

In ε-greedy action selection with \(\varepsilon = 0.5\) and \(k = 2\) actions, what is the probability that the greedy action is selected on any given step?

Solution:

  • With prob \(1-\varepsilon = 0.5\): the greedy branch is taken → greedy action selected (prob 1).
  • With prob \(\varepsilon = 0.5\): the random branch is taken → greedy action selected with prob \(\tfrac{1}{k} = \tfrac{1}{2}\).
$$ P(\text{greedy}) = (1-\varepsilon) + \varepsilon \cdot \frac{1}{k} = 0.5 + 0.5 \times 0.5 = \mathbf{0.75} $$
▶  Key formula & step-by-step
Probability greedy action is selected (k actions, ε-greedy): $$ P(\text{greedy chosen}) = (1-\varepsilon) + \frac{\varepsilon}{k} $$ With \(\varepsilon=0.5\), \(k=2\): \( P = 0.5 + \tfrac{0.5}{2} = 0.75 \)
  1. Branch 1 — exploit (prob \(1-\varepsilon = 0.5\)): We always pick the greedy action. Contribution to \(P(\text{greedy}) = 0.5\).
  2. Branch 2 — explore (prob \(\varepsilon = 0.5\)): We pick uniformly at random from all \(k=2\) actions. Chance of landing on the greedy action \(= \tfrac{1}{2}\). Contribution \(= 0.5 \times 0.5 = 0.25\).
  3. Add the two branches: \(P(\text{greedy}) = 0.5 + 0.25 = \mathbf{0.75}\). The greedy action gets selected 75% of the time even though we explore 50% of the time.
▶  Key formula & step-by-step
Sample-average update (incremental form): $$ Q_{n+1}(a) = Q_n(a) + \frac{1}{n}\bigl[R_n - Q_n(a)\bigr] $$ \(Q_1(a)=0\) for all arms. Greedy = action with highest \(Q\).
  1. After t=1 (A₁=1, R₁=−1): \(Q(1)=-1\), others \(=0\). Greedy ∈ {2,3,4} — action 2 could be chosen greedily ✓
  2. After t=2 (A₂=2, R₂=+1): \(Q(2)=1\). Greedy = action 2 (highest Q). A₃=2 is greedy ✓
  3. After t=3 (A₃=2, R₃=−2): \(Q(2)=\tfrac{1+(-2)}{2}=-0.5\). Now Q(3)=Q(4)=0 > −0.5. Greedy ∈ {3,4}. A₄=2 is not greedy → ε must have fired here ✗ (definite exploration)
  4. After t=4 (A₄=2, R₄=+2): \(Q(2)=\tfrac{1-2+2}{3}=\tfrac{1}{3}\). Greedy = action 2 again. A₅=3 could be greedy if Q(3)>Q(2)… but Q(3)=0 so A₅=3 is exploratory too (possible, not certain).
  5. Answer: Step t=4 is the only certain exploration — the only time the action taken is definitively not greedy given the Q-values at that moment.
▶  Key formula & step-by-step
Long-run fraction selecting optimal arm (k arms): $$ P(\text{optimal selected}) \approx (1-\varepsilon) + \frac{\varepsilon}{k} $$ With \(k=10\): \(\varepsilon=0.1 \Rightarrow 91\%\); \(\varepsilon=0.01 \Rightarrow 99.1\%\); \(\varepsilon=0 \Rightarrow \approx83\%\) (gets stuck).
  1. ε=0.1: \(P(\text{opt}) = 0.9 + 0.01 = 0.91\). Earns about 91% of optimal average reward in steady state.
  2. ε=0.01: \(P(\text{opt}) = 0.99 + 0.001 = 0.991\). Higher long-run performance — explores less but converges more accurately.
  3. ε=0 (pure greedy): Locks on to the first arm that looks good. Q-estimates for unchosen arms stay at 0. Empirically converges to only ~83% of optimal.
  4. Winner: ε=0.01 wins in the long run (≈8 pp better than ε=0.1). Short-run ε=0.1 learns faster, but ε=0.01 overtakes it once the estimates are accurate.
S&B Ex 2.2 — Bandit Action Trace [5 marks]

In a 4-action bandit (\(k=4\)), ε-greedy is used with \(Q_1(a)=0\) for all actions. The following action–reward sequence is observed:

\(t\)\(A_t\)\(R_t\)
11−1
221
32−2

530

On which time steps was the ε random draw definitely taken (an exploratory action must have occurred)?

Solution — trace the Q estimates:

  • After t=1: \(Q(1)=-1\), all others \(=0\). At t=2, greedy ∈ {2,3,4}; action 2 is consistent with greedy.
  • After t=2: \(Q(2)=1\). At t=3, greedy = action 2; action 2 is consistent with greedy.
  • After t=3: \(Q(2)=\frac{1+(-2)}{2}=-0.5\). At t=4, greedy ∈ {3,4} (both \(Q=0 > -0.5\)); choosing action 2 must have been exploratory — ε definitely fired.
  • After t=4: \(Q(2)=\frac{1-2+2}{3}=\frac{1}{3}\). At t=5, greedy = action 2; action 3 may be exploratory but is not certain.

Answer: Step 4 is the only step where ε must have fired.

S&B Ex 2.3 — Long-Run Best Method [5 marks]

From Figure 2.2 (10-armed testbed), three methods are compared: greedy (\(\varepsilon=0\)), \(\varepsilon=0.01\), and \(\varepsilon=0.1\). Which performs best in the long run? By approximately how much (as a fraction of optimal reward)?

Solution:

\(\varepsilon=0.1\) converges fastest early but levels off at around 91% of optimal — it explores 10% of the time even after finding the best arm.

\(\varepsilon=0.01\) converges more slowly but eventually approaches 99% of optimal (explores only 1% of the time).

Greedy (\(\varepsilon=0\)) gets stuck at around 83% of optimal — it locks in on the first action that looks good and never discovers the true best arm.

Long-run winner: \(\varepsilon=0.01\), by roughly 8 percentage points over \(\varepsilon=0.1\). The greedy method performs worst because it cannot correct early estimation errors.
