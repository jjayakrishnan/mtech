1. Incremental Computation of the Sample Average

The sample-average estimate of an action's value after n rewards is:

\[ Q_n = \frac{R_1 + R_2 + \cdots + R_{n-1}}{n-1} \]

Recomputing this sum every time is wasteful. We want an incremental formula that updates the old estimate with just the new reward.

Plain English

Think of updating your running average grade. You don't re-add all past scores each semester. Instead: "My average was 78. My new score is 90. I've now taken 5 courses. New average = old average + (1/5)(90 - 78) = 78 + 2.4 = 80.4."

Derivation

Step-by-step derivation

\( Q_{n+1} = \frac{1}{n}\sum_{i=1}^{n} R_i \)

\( = \frac{1}{n}\left(R_n + \sum_{i=1}^{n-1} R_i\right) \)

\( = \frac{1}{n}\left(R_n + (n-1)\,Q_n\right) \)

\( = \frac{1}{n}\left(R_n + n\,Q_n - Q_n\right) \)

\( = Q_n + \frac{1}{n}\left(R_n - Q_n\right) \)

Incremental Sample-Average Update
\[ Q_{n+1} = Q_n + \frac{1}{n}\bigl[R_n - Q_n\bigr] \]

Memory cost: store only Q_n and n. No need to keep all past rewards.

2. The General Update Form

The incremental formula is a special case of a universal pattern in RL:

General Update Rule
\[ \text{NewEstimate} \leftarrow \text{OldEstimate} + \text{StepSize}\,\bigl[\text{Target} - \text{OldEstimate}\bigr] \]
Reading the formula

Target - OldEstimate = the "error" in your current estimate.
StepSize = how much you correct toward the target.
You nudge the estimate a fraction of the way toward the new observation.

For sample averages, StepSize = 1/n (shrinks over time). For nonstationary problems, we use a constant step-size instead.

3. Numerical Illustration

Worked example

Given: Q4 = 3.0,   R4 = 7.0,   step-size = 1/n = 1/4

Compute Q5:
Q5 = Q4 + (1/4)[R4 - Q4]
    = 3.0 + 0.25 × [7.0 - 3.0]
    = 3.0 + 0.25 × 4.0
    = 3.0 + 1.0
    = 4.0

Intuition check

The reward (7.0) was above our estimate (3.0), so our estimate moved UP. But only by 1/4 of the gap (not all the way to 7) because we're averaging with 3 prior observations.

Try it yourself: Q_3 = 5.0, R_3 = 2.0, step-size = 1/3. What is Q_4?

Apply: Q4 = Q3 + (1/3)[R3 - Q3]

Q4 = 5.0 + (1/3)[2.0 - 5.0] = 5.0 + (1/3)(-3.0) = 5.0 - 1.0 = 4.0
The reward was below the estimate, so the estimate moved DOWN.

4. Stationary vs Nonstationary Rewards

PropertyStationaryNonstationary
Reward distributionFixed over timeChanges / drifts over time
Real-world exampleSlot machines in a casinoClick-through rates on online ads
Best step-size1/n (gives true average)Constant α (tracks recent rewards)
Old rewardsEqually importantShould be downweighted
Analogy

Stationary: A coin's bias never changes. Average all flips equally.
Nonstationary: A restaurant's quality drifts as chefs change. Last month's reviews matter more than last year's.

Why 1/n fails for nonstationary

With step-size 1/n, after 1000 observations, each new reward only shifts Q by 1/1001. If the true value jumped, you'd barely notice for hundreds of steps. The estimate is "stuck in the past."

5. Constant Step-Size α

Replace 1/n with a fixed constant α (typically 0.1 to 0.3):

Constant Step-Size Update
\[ Q_{n+1} = Q_n + \alpha\bigl[R_n - Q_n\bigr] \]

Equivalent to: \( Q_{n+1} = (1-\alpha)Q_n + \alpha R_n \)

Exponential Recency-Weighted Average

Unrolling the recursion reveals that older rewards get exponentially less weight:

Expanded form
\[ Q_{n+1} = (1-\alpha)^n Q_1 + \sum_{i=1}^{n} \alpha(1-\alpha)^{n-i} R_i \]

The weight on reward Ri is α(1-α)n-i. Recent rewards get weight close to α; old rewards fade exponentially.

Key insight

With α = 0.2, the weight on a reward halves roughly every 3 steps: α(1-α)0 = 0.2, α(1-α)3 = 0.2 × 0.512 = 0.102. Older observations are effectively forgotten.

7. Optimistic Initial Values

A simple trick to encourage exploration without explicit randomness: initialise all Q1(a) to a value much higher than any realistic reward.

Plain English

If you believe every restaurant is a 10/10 before trying it, you'll be "disappointed" by your first visit (actual reward < 10). So you'll keep trying NEW restaurants because the untried ones still look like 10s. Eventually you've sampled everything.

How it works

Set Q1(a) = 5 for all actions (when true values are around 0-2). Initially every action looks great. Whichever action gets tried first receives a real reward (say 1.2), dropping its estimate. Now other actions (still at 5) look better, so the agent explores them next.

Example with 3 arms, Q_1 = 5 for all, alpha = 0.5

Step 1: Pick arm 1 (all tied at 5). R = 1.0 → Q(arm1) = 5 + 0.5(1-5) = 3.0
Step 2: Pick arm 2 or 3 (both at 5 > 3). Say arm 2. R = 0.5 → Q(arm2) = 5 + 0.5(0.5-5) = 2.75
Step 3: Pick arm 3 (still at 5). R = 2.0 → Q(arm3) = 5 + 0.5(2-5) = 3.5
Result: All 3 arms explored within first 3 steps!

Limitation: only temporary

Optimistic initialisation forces early exploration, but the effect fades as estimates converge to true values. It does NOT provide ongoing exploration in later timesteps. It also doesn't help with nonstationary problems — the initial bias washes out long before the environment changes.

8. Upper Confidence Bound (UCB)

The problem with epsilon-greedy: it explores randomly, treating poorly-known actions the same as well-known ones. UCB explores intelligently — it prefers actions we're uncertain about.

UCB Action Selection
\[ A_t = \arg\max_a \left[ Q_t(a) + c\,\sqrt{\frac{\ln t}{N_t(a)}} \right] \]

Qt(a) = current value estimate for action a
Nt(a) = number of times action a has been selected
t = total timesteps so far
c = exploration parameter (controls exploration-exploitation tradeoff)
ln = natural logarithm

UCB Intuition: Two Terms

TermRoleBehavior
Qt(a)ExploitationPrefer actions with high estimated value
\( c\sqrt{\frac{\ln t}{N_t(a)}} \)Exploration bonusPrefer actions that haven't been tried much
Why it works

If Nt(a) is small (rarely tried) → the bonus is large → explore that action.
If Nt(a) is large (tried many times) → the bonus shrinks → rely on Q estimate.
As t grows, ln(t) grows slowly, ensuring eventually all actions get tried, but well-estimated good actions dominate.

9. UCB Worked Examples

Example A: 2 actions, c = 2, t = 10

Given

Action 1: Q10(1) = 3.5,   N10(1) = 7
Action 2: Q10(2) = 2.8,   N10(2) = 3
c = 2,   t = 10,   ln(10) = 2.303

Computation

UCB(1) = 3.5 + 2√(2.303/7) = 3.5 + 2√(0.329) = 3.5 + 2 × 0.574 = 3.5 + 1.147 = 4.647

UCB(2) = 2.8 + 2√(2.303/3) = 2.8 + 2√(0.768) = 2.8 + 2 × 0.876 = 2.8 + 1.753 = 4.553

Select: Action 1 (4.647 > 4.553)

Reading the result

Action 2 has a lower Q-value BUT got a bigger exploration bonus (tried only 3 times vs 7). Still, action 1 wins because its Q advantage outweighs the exploration bonus. Close call though!

Example B: 4 actions, c = 1, t = 20

Given (ln(20) = 3.0)

Action 1: Q = 4.0, N = 8 → UCB = 4.0 + 1√(3.0/8) = 4.0 + 0.612 = 4.612
Action 2: Q = 3.2, N = 6 → UCB = 3.2 + 1√(3.0/6) = 3.2 + 0.707 = 3.907
Action 3: Q = 5.1, N = 5 → UCB = 5.1 + 1√(3.0/5) = 5.1 + 0.775 = 5.875
Action 4: Q = 2.0, N = 1 → UCB = 2.0 + 1√(3.0/1) = 2.0 + 1.732 = 3.732

Select: Action 3 (5.875 is highest)

Notice

Action 4 was tried only once, giving it the biggest exploration bonus (1.732). But action 3's high Q-value (5.1) still dominates. UCB balances both signals — it doesn't blindly explore the least-tried action.

Try it yourself: c = 2, t = 8, ln(8) = 2.08. Arm A: Q=4.0, N=5. Arm B: Q=3.0, N=3. Which arm?

Compute UCB for each arm:

UCB(A) = 4.0 + 2√(2.08/5) = 4.0 + 2√(0.416) = 4.0 + 2(0.645) = 4.0 + 1.29 = 5.29
UCB(B) = 3.0 + 2√(2.08/3) = 3.0 + 2√(0.693) = 3.0 + 2(0.833) = 3.0 + 1.665 = 4.665
Select Arm A (5.29 > 4.665). Arm A wins on exploitation despite B's bigger bonus.
Try it yourself: Same setup but Arm B has N=1 instead of 3. Now which arm wins?

Recalculate UCB(B) with N=1:

UCB(A) = 5.29 (unchanged)
UCB(B) = 3.0 + 2√(2.08/1) = 3.0 + 2√(2.08) = 3.0 + 2(1.442) = 3.0 + 2.884 = 5.884
Now Arm B wins (5.884 > 5.29)! With only 1 observation, the uncertainty bonus is large enough to overcome the Q disadvantage. UCB says: "We don't know enough about B yet — explore it."

When to use what

Stationary + want optimal: UCB (directed exploration, no wasted random pulls).
Nonstationary: ε-greedy with constant α (simple, robust, keeps exploring).

EC-3 Comprehensive, Q1 — Healthcare MAB [10 marks]

A reinforcement learning agent helps a healthcare professional choose between two treatments: Medication (M) and Therapy (T). Reward is +1 (recovery) or −1 (no improvement). Dataset of 8 time steps:

tActionReward
1M+1
2M−1

4T+1
5T−1
6M+1
7T−1
8M\(r\)

Part (a) [4 marks]: UCB with c=1.0 — find max r for which UCB(T) > UCB(M) at t=9

Full Worked Solution

Step 1: Count selections up to t=8: \(N_8(M) = 4\), \(N_8(T) = 4\). Next decision is at \(t = 9\).
Step 2: Compute \(Q_8(T)\) (sample average): rewards for T were +1, +1, −1, −1 → \(Q_8(T) = 0/4 = 0\).
Step 3: Compute \(Q_8(M)\) as a function of \(r\): rewards for M were +1, −1, +1, \(r\) → \(Q_8(M) = (1 + r)/4\).
Step 4: At \(t=9\), both have \(N=4\), so exploration bonuses are equal: \[ c\sqrt{\frac{\ln 9}{N}} = 1 \cdot \sqrt{\frac{2.197}{4}} = \sqrt{0.549} = 0.741 \] Since the bonuses cancel, UCB(T) > UCB(M) iff \(Q_8(T) > Q_8(M)\).
Step 5: Solve \(0 > (1+r)/4\): gives \(r < -1\). Since reward is bounded to \(\{+1, -1\}\), the maximum integer reward \(r = -1\) gives \(Q_8(M) = 0 = Q_8(T)\) → tie, not strictly exceeding. So no valid \(r \in \{+1,-1\}\) makes UCB(T) strictly exceed UCB(M). The maximum \(r\) for which it holds is \(r < -1\) (not achievable with the given reward set).

Part (b) [4 marks]: Nonstationary α=0.5, Q₁(M)=0, compute Q at t=9

Full Worked Solution — Exponential Recency-Weighted Update

Formula: \(Q_{n+1} = Q_n + \alpha(R_n - Q_n)\) with \(\alpha = 0.5\). Only update when action M is taken.
t=1 (M, +1): \(Q = 0 + 0.5(1 - 0) = 0.5\)
t=2 (M, −1): \(Q = 0.5 + 0.5(-1 - 0.5) = 0.5 - 0.75 = -0.25\)
t=3,4,5 (T): No update to Q(M). \(Q = -0.25\)
t=6 (M, +1): \(Q = -0.25 + 0.5(1 - (-0.25)) = -0.25 + 0.625 = 0.375\)
t=7 (T): No update. \(Q = 0.375\)
t=8 (M, r): \(Q = 0.375 + 0.5(r - 0.375) = 0.375 + 0.5r - 0.1875 = 0.1875 + 0.5r\)
Result: \(Q_9(M) = 0.1875 + 0.5r\). If \(r = +1\): Q = 0.6875. If \(r = -1\): Q = −0.3125.
EC-2 Mid-Sem, Q3 — Online Pharma MAB [7.5 marks]

Actions: SMS (S), Teleconsultation (T), Lab Reminder (R), Counselling Call (C).
Observations in order: (1,S,7), (2,T,5), (3,R,6), (4,C,4), (5,S,8), (6,T,6), (7,R,7), (8,C,5).

Part (a) [3 marks]: Exponential recency-weighted update with α=0.5, Q₁=0

Full Worked Solution — All Actions

Action S: t=1: Q=0+0.5(7−0)=3.5. t=5: Q=3.5+0.5(8−3.5)=5.75
Action T: t=2: Q=0+0.5(5−0)=2.5. t=6: Q=2.5+0.5(6−2.5)=4.25
Action R: t=3: Q=0+0.5(6−0)=3.0. t=7: Q=3.0+0.5(7−3.0)=5.0
Action C: t=4: Q=0+0.5(4−0)=2.0. t=8: Q=2.0+0.5(5−2.0)=3.5
Final Q-values: S=5.75, T=4.25, R=5.0, C=3.5. Best action: SMS (S).

MEMORISE THIS — Closed Book

Incremental Update (sample average)
\[ Q_{n+1} = Q_n + \frac{1}{n}\bigl[R_n - Q_n\bigr] \]
Constant Step-Size (nonstationary)
\[ Q_{n+1} = Q_n + \alpha\bigl[R_n - Q_n\bigr] \]
General Update Form
\[ \text{NewEstimate} \leftarrow \text{OldEstimate} + \text{StepSize}\bigl[\text{Target} - \text{OldEstimate}\bigr] \]
UCB Action Selection
\[ A_t = \arg\max_a \left[ Q_t(a) + c\sqrt{\frac{\ln t}{N_t(a)}} \right] \]
Four things to never forget

1. Step-size \(1/n\) gives equal weight to all rewards (stationary). Constant \(\alpha\) gives exponential recency weighting (nonstationary).

2. UCB = exploitation term + exploration bonus. Exploration bonus shrinks as \(N_t(a)\) grows.

3. Optimistic init forces exploration ONLY early on — it's temporary, not a long-term solution.

4. In the UCB formula, \(t\) is in the numerator (under ln) and \(N_t(a)\) is in the denominator. Don't flip them.

Exam-Style Quiz

Closed-book conditions. Cover the page above before attempting.

Q1 [1 mark] Given Q_4 = 3.0 and R_4 = 7.0, what is Q_5 using the sample-average update?
Correct. Step-size is 1/4 (this is the 4th reward). The estimate moves 1/4 of the way from 3.0 toward 7.0.
Q2 [1 mark] Why is constant step-size alpha preferred over 1/n for nonstationary problems?
Correct. The key property is exponential recency weighting: weight on R_i is alpha(1-alpha)^{n-i}, so old rewards fade away naturally.
Q3 [2 marks] UCB: c=2, t=10 (ln10=2.3). Action A: Q=4.0, N=8. Action B: Q=3.5, N=2. Which action is selected?
Correct. Action B's exploration bonus (2.14) overwhelms its Q disadvantage (0.5 less than A). UCB says: "B has only been tried twice — we need more information."
Q4 [1 mark] What is the main limitation of optimistic initial values?
Correct. Optimistic init is a "one-shot" exploration trick. After the initial phase, estimates settle and the optimistic bias disappears. It cannot help with nonstationary environments or late-stage exploration.
Q5 [1 mark] In UCB, what happens to the exploration bonus for an action as N_t(a) increases?
Correct. As you try an action more, its uncertainty decreases, so the exploration bonus shrinks. This is what makes UCB "intelligent" — it focuses exploration on poorly-known actions.
