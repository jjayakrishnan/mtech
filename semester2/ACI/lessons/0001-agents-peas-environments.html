Lesson 1: AI Agents, PEAS & Environments

Lesson 1 of 6 | Purely conceptual — no formulas

1. What is AI? Rational Agents

Artificial Intelligence is the study and design of rational agents — entities that perceive their environment through sensors and act upon it through actuators to maximise a performance measure.

Think of it like this: a thermostat senses temperature and acts by turning the heater on or off. A rational agent does the same, but it picks the action that is expected to produce the best outcome given what it knows.

The Agent-Environment Loop

At each time step the agent receives a percept (sensor reading), consults its internal decision mechanism, and emits an action. The cycle repeats continuously.

2. PEAS Specification

Before building an agent, we specify its task using the PEAS framework:

  • Performance measure — how do we score success?
  • Environment — what world does the agent operate in?
  • Actuators — what actions can the agent take?
  • Sensors — what information can the agent perceive?

3. Environment Properties (6 Pairs)

Every environment can be classified along six independent dimensions. Each pair is a spectrum, not a binary — but for exams, pick the closest match and justify briefly.

PropertyMeaningExample
Fully Observable vs Partially Observable Can the agent see the complete state at every step? Chess (full) vs Poker (partial — hidden cards)
Deterministic vs Stochastic Does the same action from the same state always produce the same next state? Vacuum world (det.) vs Driving with wind gusts (stoch.)
Episodic vs Sequential Is the current decision independent of past decisions? Spam filter (episodic) vs Chess (sequential)
Static vs Dynamic Can the environment change while the agent is deliberating? Crossword (static) vs Real-time traffic (dynamic)
Discrete vs Continuous Are states, time, percepts, and actions finite/countable? Chess (discrete) vs Robot arm movement (continuous)
Single-agent vs Multi-agent Is there one agent or are other entities also optimising? Sudoku solver (single) vs Competitive game (multi)

Quick Mnemonic

F-D-E-S-D-S — "Fully Deterministic Episodes Stay Discrete & Single." Recall the first letters: Observable, Deterministic, Episodic, Static, Discrete, Single-agent — then flip each to get the opposite property.

4. Five Agent Types (Simplest to Most Complex)

1. Simple Reflex Agent

Acts only on the current percept using condition-action rules. No memory, no model of the world.

Percept ──▶ [Condition-Action Rules] ──▶ Action

2. Model-Based Reflex Agent

Maintains an internal state (model of the world) to handle partial observability. Updates the model using percepts, then applies rules.

Percept ──▶ [Update State] ──▶ Internal State ──▶ [Rules] ──▶ Action

3. Goal-Based Agent

Has explicit goals. Considers future consequences of actions (search/planning) to pick actions that achieve the goal.

Percept ──▶ [State] ──▶ [Goal + Search/Plan] ──▶ Action

4. Utility-Based Agent

Uses a utility function to rank states by desirability. Chooses the action that maximises expected utility — handles trade-offs and uncertainty.

Percept ──▶ [State] ──▶ [Utility Function + Maximise EU] ──▶ Action

5. Learning Agent

Has four components: learning element (improves), performance element (selects actions), critic (evaluates against a standard), and problem generator (suggests exploratory actions).

Ordering Tip

Each agent type adds one capability over the previous: Rules → +State → +Goals → +Utility → +Learning. If an exam question asks "which agent type?", pick the simplest one that can handle the described task.

5. Problem-Solving Agents

A goal-based agent that uses search to find a sequence of actions is called a problem-solving agent. It formulates the problem with five components:

  1. Initial State — where the agent starts (e.g., current board configuration)
  2. Actions — what the agent can do in each state (e.g., slide or hop a tile)
  3. Transition Model — the result of applying an action to a state: Result(s, a) → s'
  4. Goal Test — is this state a goal? (e.g., 3 matching tiles in a row)
  5. Path Cost — numeric cost of a path (often: number of moves, or weighted sum)

Applied to the Sliding-Tile Game

  • Initial State: A B A _ B A B _ (or any given starting configuration)
  • Actions: Move#1 (slide adjacent tile into empty), Move#2 (hop over one tile into empty)
  • Transition Model: Applying a move swaps the tile with the empty space
  • Goal Test: Three same-letter tiles are adjacent
  • Path Cost: Number of moves made

Try It Yourself

Write the PEAS specification for a Crop Irrigation AI Agent

Scenario: An AI agent manages water irrigation for a farm. It decides when and how much to water each zone based on soil moisture, weather forecasts, and crop type.

Sample Answer

PEASYour Specification
PerformanceMaximize crop yield; minimize water usage; maintain optimal soil moisture
EnvironmentFarm fields divided into zones; weather conditions; soil types; crop growth stages
ActuatorsValves (open/close/adjust flow), sprinkler heads, drip lines, pump controls
SensorsSoil moisture sensors, weather station (temperature, humidity, rain), flow meters, crop cameras

Environment Properties

  • Partially observable — cannot sense every root's moisture level
  • Stochastic — weather is unpredictable
  • Sequential — today's watering affects tomorrow's soil state
  • Dynamic — weather and evaporation change while deciding
  • Continuous — moisture levels, time, and flow rates are continuous
  • Single-agent — no adversary (unless we model pests as agents)
