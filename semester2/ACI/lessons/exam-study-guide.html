2. Closed-Book Cheat Sheet

A* Evaluation

$$f(n) = g(n) + h(n)$$

\(g(n)\) = actual cost from start • \(h(n)\) = heuristic estimate to goal • \(f(n)\) = total estimated cost

Admissibility

$$h(n) \leq h^*(n)$$

Never overestimates true cost. Guarantees A* optimality. Key: relaxed problem ⟹ admissible h.

Consistency (Monotonicity)

$$h(n) \leq c(n, a, n') + h(n')$$

Triangle inequality. Consistent ⟹ admissible. A* with consistent h never re-expands nodes.

Alpha-Beta Prune Condition

$$\text{Prune when } \beta \leq \alpha$$

\(\alpha\) = MAX's best so far (starts \(-\infty\)) • \(\beta\) = MIN's best so far (starts \(+\infty\))

Alpha-Beta Complexity

$$O(b^{m/2}) \text{ (best case)}$$

Minimax: \(O(b^m)\). Perfect ordering halves depth. Doubles search depth for same computation.

MCTS UCB Formula

$$\text{UCB}(i) = v_i + C\sqrt{\frac{\ln N}{n_i}}$$

\(v_i\) = win rate • \(C\) = exploration constant • \(N\) = parent visits • \(n_i\) = node visits

ACO Pheromone Update

$$\tau_{ij} \leftarrow (1-\rho)\,\tau_{ij} + \Delta\tau_{ij}$$

\(\rho\) = evaporation rate • \(\Delta\tau_{ij}\) = quality of solutions using edge \((i,j)\)

NEAT Fitness Sharing

$$f'_i = \frac{f_i}{\sum_j \text{sh}(d_{ij})}$$

Shared fitness prevents one species dominating. \(\text{sh}(d)=1\) if \(d < \delta_t\), else 0.

Simulated Annealing Accept

$$P(\text{accept}) = e^{\Delta E / T}$$

Accept worse move with probability \(e^{\Delta E/T}\). As \(T \to 0\), behaves like hill climbing.

Question 2 — Informed Search & Heuristics [6 Marks]

(a) In an 8-puzzle, define two admissible heuristics \(h_1\) (misplaced tiles) and \(h_2\) (Manhattan distance). For the state below, compute both. State which dominates and why. [3 Marks]

  Current state:     Goal state:

Step 2 — \(h_2\): Sum of Manhattan distances

Manhattan distance = |row difference| + |col difference| for each tile:

TileCurrent (r,c)Goal (r,c)Distance

Step 3 — Dominance

\(h_2\) dominates \(h_1\) because \(h_2(n) \geq h_1(n)\) for every node. Dominance means A* with \(h_2\) expands fewer nodes and finds the solution faster. Both are admissible: \(h_2\) comes from a relaxed problem where tiles can slide without blocking.

Exam tip: To prove admissibility, state the relaxed problem. \(h_2\) relaxes "a tile cannot move through another tile." \(h_1\) relaxes both "blocking" AND "only adjacent moves." This is why \(h_2\) is tighter (less relaxed ⟹ more accurate ⟹ dominates).

Show Answer — Part (b)
CriterionA*Greedy Best-First Search

Exam tip: The professor explicitly said "Greedy Best-First is NOT optimal — it ignores g(n)." This distinction almost always appears in the exam.

Question 3 — Local Search & Evolutionary Algorithms [6 Marks]

(a) Given the following fitness landscape (x-axis = state, y-axis = objective value), trace steepest-ascent Hill Climbing starting from state D. Mark where it gets stuck and explain why. [2 Marks]

  State:  A   B   C   D   E   F   G   H
  Value:  2   5   3   6   8   4   9   3
  Neighbours: A↔B, B↔C, C↔D, D↔E, E↔F, F↔G, G↔H
Show Answer — Part (a)

Trace: Steepest-Ascent Hill Climbing from D

  1. Start at D (value 6). Neighbours: C (value 3), E (value 8). Best neighbour = E (8 > 6). Move to E.
  2. At E (value 8). Neighbours: D (value 6), F (value 4). Best neighbour = D (6), but 6 < 8 (current). No neighbour beats E. Stop — local maximum at E.

Why stuck? E is a local maximum: all its neighbours (D=6, F=4) have lower values. Hill Climbing has no mechanism to escape — it only accepts strictly better moves. The global maximum G (value 9) is unreachable without first going through F (value 4), which is worse than E.

Exam tip: A local maximum is not the global maximum. To escape local maxima: (1) random restart — try from multiple starting points; (2) simulated annealing — accept worse moves with probability \(e^{\Delta E/T}\); (3) beam search — maintain k states simultaneously.

Exam tip: Single-point crossover splits at one position. Multi-point crossover uses two or more split points. Uniform crossover randomly picks each bit from either parent. Mutation rate is kept low (≈1%) to preserve good solutions.

(c) Distinguish between Generational GA and Steady-State GA. Which is better for maintaining diversity and why? [2 Marks]

Show Answer — Part (c)
AspectGenerational GASteady-State GA
ReplacementEntire population replaced each generation by new offspringOnly 1–2 individuals replaced per iteration (worst replaced by best child)
Convergence speedFast — large population turnoverSlower — incremental change
DiversityLower — good solutions can be lostHigher — elites persist longer
MemoryTwo populations needed (current + next)One population, in-place updates

Steady-State GA is better for diversity because fit individuals survive for many generations, preventing loss of good genetic material (genetic drift). Combined with crowding/niching (replacing the most similar individual rather than the worst), it further preserves diversity across the fitness landscape.

Common trap: "Generational GA replaces the best individuals" — WRONG. It replaces the entire population, which may accidentally discard the current best (unless elitism is used to preserve the top k individuals).

Question 4 — NEAT & Neural Architecture Search [4 Marks]

(a) Explain the "competing conventions problem" in neuroevolution. How does NEAT's solution address it? [2 Marks]

Show Answer — Part (a)

Competing Conventions Problem

When evolving neural networks with a standard GA, two networks that compute the same function may encode it differently — for example, the same two hidden neurons may be connected in opposite orders. When these networks crossover, the offspring inherits a scrambled, non-functional network. This is the competing conventions (or permutation) problem.

Analogy: Imagine combining two correct jigsaw puzzles of the same image, but one is built left-to-right and the other right-to-left — the merged result is noise, not a picture.

NEAT's Solution: Historical Markings + Speciation

  • Historical markings: Every new gene (node or connection) is assigned a unique innovation number when first created. Crossover only pairs genes with matching innovation numbers — preventing mismatched recombination.
  • Speciation: Networks are grouped into species based on structural similarity. Crossover only happens within a species. This protects novel structures (which initially perform poorly) long enough to allow them to improve before competing globally.

Exam tip: The professor specifically named NEAT and its three key innovations: (1) historical markings, (2) speciation with fitness sharing, (3) structural mutations that grow the network incrementally. All three may be asked separately.

(b) What is Neural Architecture Search (NAS)? How does the DARTS approach differ from evolutionary NAS? [2 Marks]

Show Answer — Part (b)

Neural Architecture Search (NAS)

NAS automates the design of neural network architectures. Instead of human experts hand-crafting layer types, connectivity patterns, and hyperparameters, NAS algorithms search the architecture space to find high-performing designs for a given task.

AspectEvolutionary NAS (e.g., DeepNEAT)DARTS (Differentiable Architecture Search)
Search methodGenetic operators — mutate and crossover architectures, select by fitnessGradient descent — architecture parameters are continuous; optimise jointly with weights
Search spaceDiscrete — each architecture is a separate evaluationContinuous relaxation — all operations mixed, weights learned
CostVery high — train each candidate architecture fullyMuch lower — one supernet trained, then discretised
ExplorationGood global exploration via population diversityEfficient local search; can miss global optima

Exam tip: DARTS is the gradient-based shortcut — instead of training thousands of separate networks (like evolutionary NAS), it trains one "supernet" where each edge is a weighted mixture of candidate operations, then prunes to the strongest operation per edge.

(b) Apply Alpha-Beta pruning to the same tree (left-to-right traversal). List every node/leaf that is pruned and explain the α/β values that caused each pruning. [3 Marks]

Show Answer — Part (b)

Initialise root: α = −∞, β = +∞

Left subtree (N1 MIN, then N3 MAX, then N4 MAX)

  1. N3 (MAX): Visit leaf 3 → αN3 = 3. Visit leaf 5 → αN3 = 5. N3 returns 5.
  2. N1 (MIN): first child N3 = 5 → βN1 = 5. Root α = −∞ < β = 5: no prune. Visit N4.
  3. N4 (MAX): Visit leaf 2 → αN4 = 2. Check: parent N1 has β = 5. Visit leaf 9 → αN4 = 9. N4 returns 9. (No prune at N4 — we must visit both leaves to determine N4's value.)
  4. N1 = min(5, 9) = 5. Root α = max(−∞, 5) = 5.

Right subtree (N2 MIN, then N5 MAX, then N6 MAX)

  1. N5 (MAX): Visit leaf 1 → αN5 = 1. Visit leaf 7 → αN5 = 7. N5 returns 7.
  2. N2 (MIN): first child N5 = 7 → βN2 = 7. Check: root α = 5 < β = 7: no prune. Visit N6.
  3. N6 (MAX): Visit leaf 6 → αN6 = 6. Check: parent N2 has βN2 = 7. Since 6 < 7, continue. Visit leaf 4 → αN6 remains 6. N6 returns 6.
  4. N2 = min(7, 6) = 6. Root = max(5, 6) = 6.
Result: No nodes pruned in this tree. All 8 leaves are evaluated. This illustrates worst-case ordering for alpha-beta — the bounds set by earlier nodes never become tight enough to cut off later branches.

Exam tip: This is an example of leaves arranged in non-decreasing order at MAX nodes and non-increasing at MIN nodes — exactly the worst case for alpha-beta. The professor asked in the regular mid-semester: "Arrange leaf values so no pruning occurs." The answer is this ordering pattern.

(c) Describe the four phases of Monte Carlo Tree Search (MCTS). For a node with 3 wins in 8 visits, parent has 20 visits, and exploration constant \(C = \sqrt{2}\), compute the UCB score. [2 Marks]

Show Answer — Part (c)

MCTS Four Phases

PhaseWhat happens
1. SelectionFrom root, follow UCB scores down the tree until reaching a node with unexplored children
2. ExpansionAdd one new child node for an unexplored move from the selected node
3. Simulation (Rollout)From the new node, play randomly to a terminal state — record win/loss result
4. BackpropagationUpdate visit count and win count for all nodes on the path from new node to root

UCB Calculation

Given: \(v_i = 3/8 = 0.375\), \(N = 20\), \(n_i = 8\), \(C = \sqrt{2} \approx 1.414\)

$$\text{UCB}(i) = v_i + C\sqrt{\frac{\ln N}{n_i}} = 0.375 + 1.414 \times \sqrt{\frac{\ln 20}{8}}$$ $$= 0.375 + 1.414 \times \sqrt{\frac{2.996}{8}} = 0.375 + 1.414 \times \sqrt{0.3745}$$ $$= 0.375 + 1.414 \times 0.612 = 0.375 + 0.865 = \mathbf{1.240}$$

Exam tip: When \(n_i\) is very small (node rarely visited), the exploration term dominates — UCB is high. When \(n_i\) is large, the exploitation term (win rate) dominates. MCTS balances both automatically without needing a static evaluation function — this is why it works for Go where evaluation is hard.

4. Top Exam Traps

Top 5 mistakes students make in this exam
  1. Confusing steady-state vs generational GA — generational replaces the ENTIRE population; steady-state replaces only 1–2 individuals. Confirmed error in your quiz history.
  2. Pruning condition written wrong — prune when β ≤ α (or α ≥ β). NOT when α > β alone. Equal also triggers prune.
  3. GBFS treated as optimal — it is NOT optimal because it ignores g(n). A* is optimal (with admissible h), GBFS is just fast.
  4. Admissibility vs consistency confused — admissible = never overestimates (\(h \leq h^*\)). Consistent = triangle inequality (\(h(n) \leq c(n,a,n') + h(n')\)). Consistent ⟹ admissible, but NOT the other way round.
  5. MCTS phases in wrong order — it is always: Selection → Expansion → Simulation → Backpropagation. Never simulate before expanding.
